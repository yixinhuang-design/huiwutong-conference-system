# 座位布局图编辑与过道显示修复说明

**修复日期**: 2026-02-10
**问题**: 座位布局图内无法编辑修改已有座位，且过道无法正确显示
**状态**: ✅ 已修复

---

## 📋 问题描述

在智能排座页面中，存在两个主要问题：

1. **座位编辑问题**：
   - 已占用座位选择后，操作选项不够完整
   - 空座位的编辑流程不够友好
   - 缺少"选择座位用于后续操作"的功能

2. **过道显示问题**：
   - 过道座位的视觉区分不够明显
   - 过道座位号显示不当
   - 过道座位的样式与普通空座位的区别不够清晰

---

## 🔧 修复方案

### 修改1: 增强过道座位的视觉显示

**文件**: `seating-mgr.html` (行 822-833)

**修改内容**：

```css
/* 修改前 */
.seat.aisle {
    background: transparent;
    border: 2px dashed var(--gray-300);
    cursor: not-allowed;
    opacity: 0.4;
}

/* 修改后 ✅ */
.seat.aisle {
    background: rgba(128, 128, 128, 0.15);  /* ✅ 半透明背景 */
    border: 2px dashed rgba(128, 128, 128, 0.5);  /* ✅ 更深的虚线 */
    cursor: not-allowed;
    opacity: 0.6;  /* ✅ 提高不透明度 */
}

.aisle-marker {  /* ✅ 新增：过道标记 */
    display: block;
    width: 3px;
    height: 3px;
    background: rgba(128, 128, 128, 0.6);
    border-radius: 50%;
}
```

**优势**：
- 过道座位有灰色半透明背景，更易区分
- 虚线边框更明显
- 中心小点标记，表示过道状态

### 修改2: 优化座位模板显示

**文件**: `seating-mgr.html` (行 2240-2250)

**修改内容**：

```html

    {{ seat.occupant.name }}
</span>
<span v-else class="seat-number">{{ seat.number }}</span>

<!-- 修改后 ✅ -->

```

**改进**：
- 过道座位显示为小点标记，而不是座位号
- 逻辑更清晰：首先检查过道，再检查占用状态

## 🎯 预期效果

### 过道显示效果

1. **视觉区分**：
   - 过道座位：半透明灰色背景 + 虚线边框 + 中心小点
   - 普通空座位：纯白背景 + 实线边框 + 座位号
   - 已占用座位：彩色渐变背景 + 参会者名字

2. **交互行为**：
   - 过道座位：无法点击（cursor: not-allowed）
   - 普通座位：支持各种操作

### 座位编辑流程

1. **已占用座位操作**：
   - 点击已占用座位 → 选中 → 可与空座位交换或与其他已占用座位交换

2. **空座位操作**：
   - 点击空座位 → 选择操作：
     - 输入参会者名字 → 分配参会者
     - 输入 "select" → 选择该座位用于后续操作
     - 输入 "clear" → 清空（提示已是空座位）

3. **多步骤编辑**：
   - 选择空座位 → 点击已占用座位 → 交换参会者

## 💡 技术要点

1. **过道标记实现**：
   - 使用 `v-if="seat.isAisle"` 判断是否为过道
   - 显示简单的小点标记，而不是座位号
   - CSS类名 `.aisle` 自动应用基础样式

2. **座位编辑流程**：
   - 使用 `selectedSeats` 数组跟踪选中的座位
   - 支持二次点击进行操作选择
   - 通过 `prompt()` 获取用户操作指令

3. **交互优化**：
   - 空座位的操作选项更加明确
   - 支持"选择座位"操作用于复杂的调整场景
   - 所有操作都需要用户确认
