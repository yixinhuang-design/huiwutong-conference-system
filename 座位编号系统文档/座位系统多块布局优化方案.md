### 1.3 关键布局特性

| 特性 | 说明 | 影响 |
|------|------|------|
| **多个独立区块** | 5-7个座位区域 | 需要独立配置和管理 |
| **横向对称布局** | 左右两侧对称排列 | 可简化配置（左侧镜像右侧） |
| **不同尺寸区块** | 区块尺寸差异大（1×8 至 3×15） | 需要灵活的尺寸设定 |
| **层级布局** | 上下分层（上层小块、下层大块） | 需要位置坐标或分组信息 |
| **编号独立性** | 每个区块有独立的编号方式 | 需要区块级的编号配置 |
| **分组/颜色区分** | 按机构/部门使用不同颜色 | 需要样式和分组映射 |

## 2. 现状与问题

### 2.1 现有系统 (`createSeatingArea()` 函数)

```javascript
function createSeatingArea(id, name, type, rows, cols, startNumber) {
    return {
        id,           // 区块ID
        name,         // 区块名称
        type,         // 区块类型: 'stage', 'vip', 'regular'
        rows,         // 行数
        cols,         // 列数
        startNumber,  // 起始座位号
        seating: []   // 座位数据
    };
}
```

**优点**：
- ✅ 简洁直观
- ✅ 易于理解
- ✅ 适合简单布局

**局限性**：
- ❌ **无布局位置信息** - 无法指定区块在页面中的位置
- ❌ **缺乏分组机制** - 无法将多个区块分组显示
- ❌ **编号方式固定** - 所有区块用同一编号模式
- ❌ **缺乏响应式布局** - 不支持栅格化布局
- ❌ **无样式扩展** - 无法指定独立的颜色/样式

### 2.2 需要解决的问题

| 问题 | 原因 | 影响范围 |
|------|------|---------|
| 多区块位置无法指定 | 现有系统缺乏坐标/栅格信息 | 无法实现图1的复杂布局 |
| 区块间无依赖关系 | 平面化设计，无层级 | 难以表达上下分层结构 |
| 编号方式无法差异化 | 使用全局 startNumber | 不同区块需要不同的编号起点 |
| 响应式适应困难 | 硬编码的行列数 | 小屏幕时会错位或溢出 |
| 样式管理分散 | 通过 type 判断 | 颜色/样式定制困难 |

## 3. 技术实现方案

### 3.1 方案对比

#### 方案A：基于CSS Grid的位置方案 ✅ 推荐

**核心思路**：
- 定义一个全局的网格容器（如 20列×10行）
- 每个座位区块通过 grid-column 和 grid-row 指定位置
- 座位区块的内部通过 Flexbox 排列

**优点**：
- ✅ 布局灵活，位置精确
- ✅ CSS原生支持，性能好
- ✅ 响应式设计相对简单
- ✅ 与现有HTML结构兼容

**缺点**：
- ❌ 需要计算栅格坐标

**复杂度**：⭐⭐⭐

#### 方案B：基于坐标的绝对定位方案

**核心思路**：
- 为每个座位区块定义 x, y, width, height
- 使用 CSS position 和 transform 进行定位

**优点**：
- ✅ 位置精确，易于计算
- ✅ 实现简单

**缺点**：
- ❌ 响应式设计困难
- ❌ 需要手动计算坐标

**复杂度**：⭐⭐⭐⭐

#### 方案C：基于分组的响应式方案

**核心思路**：
- 定义多个区域容器（上方、中央、下方）
- 每个区域内放置多个座位区块
- 区域本身响应式布局

**优点**：
- ✅ 层级清晰，结构简洁
- ✅ 响应式友好
- ✅ 易于维护

**缺点**：
- ❌ 对称性难以表达

**复杂度**：⭐⭐

**推荐指数**：⭐⭐⭐⭐⭐

### 3.2 推荐方案：方案A + 方案C 混合

**最终设计**：
1. **顶层**：使用CSS Grid定义全局布局
2. **区域层**：将座位区块分组到各个区域（上方、中央、下方）
3. **区块层**：每个座位区块通过栅格坐标定位
4. **座位层**：区块内通过Flexbox排列座位

```
┌─────────────────────────────────────┐
│   全局容器（Grid 20列×10行）        │
├──────────┬──────────┬──────────┐
│ 上方区   │ 中央区   │ 下方区   │
│ (Grid)   │ (Grid)   │ (Grid)   │
└──────────┴──────────┴──────────┘
    ↓         ↓         ↓
 各座位区块  各座位区块  各座位区块
```

```javascript
        // ========== 基础信息 ==========
        id:              config.id,            // 区块唯一ID
        name:            config.name,          // 区块名称
        type:            config.type,          // 类型: 'stage', 'vip', 'regular', 'banner', 'enterprise'
        
        // ========== 尺寸配置 ==========
        rows:            config.rows,          // 行数
        cols:            config.cols,          // 列数
        
        // ========== 编号配置 ==========
        numberingMode:   config.numberingMode || 'leftToRight',  // 编号方式
        startNumber:     config.startNumber || 0,                // 起始编号
        
        // ========== 布局定位 (新增) ==========
        gridPosition: {
            column: config.gridPosition?.column || 'auto',  // CSS Grid column
            row:    config.gridPosition?.row    || 'auto',  // CSS Grid row
            width:  config.gridPosition?.width  || 'auto',  // 跨度列数
            height: config.gridPosition?.height || 'auto'   // 跨度行数
        },
        
        zone:            config.zone || null,                    // 区域分组: 'top', 'center', 'bottom'
        order:           config.order || 0,                      // 区域内顺序
        
        // ========== 样式配置 (新增) ==========
        styles: {
            backgroundColor:  config.styles?.backgroundColor || null,
            borderColor:      config.styles?.borderColor || null,
            textColor:        config.styles?.textColor || null,
            cssClass:         config.styles?.cssClass || null,
            customCSS:        config.styles?.customCSS || null
        },
        
        // ========== 功能配置 (新增) ==========
        isSelectable:    config.isSelectable !== false,      // 是否可选座位
        isEditable:      config.isEditable !== false,        // 是否可编辑
        canAssignPeople: config.canAssignPeople !== false,   // 是否可分配人员
        
        // ========== 数据 ==========
        seating: []                                           // 座位数据
    };
}
```

### 5.3 Vue 3 代码实现（关键片段）

```javascript
import { ref, computed, reactive } from 'vue'

export default {
    setup() {
        // 当前布局配置
        const layoutConfig = ref(multiBlockLayoutConfigs.conferenceHall)
        
        // 所有座位区块
        const seatingAreas = ref([])
        
        // 初始化多块布局
        function initializeMultiBlockLayout(layoutName = 'conferenceHall') {
            const config = multiBlockLayoutConfigs[layoutName]
            if (!config) {
                console.error(`Layout ${layoutName} not found`)
                return
            }
            
            seatingAreas.value = []
            
            // 遍历所有区域
            Object.entries(config.areas).forEach(([zoneName, zoneConfig]) => {
                zoneConfig.regions.forEach((areaConfig, index) => {
                    // 创建座位区块
                    const area = createSeatingArea({
                        ...areaConfig,
                        zone: zoneName,
                        order: index
                    })
                    
                    // 生成座位
                    generateSeats(area)
                    
                    seatingAreas.value.push(area)
                })
            })
        }
        
        // 生成座位
        function generateSeats(area) {
            const { rows, cols, startNumber, numberingMode } = area
            
            area.seating = []
            
            for (let row = 0; row < rows; row++) {
                for (let col = 0; col < cols; col++) {
                    // 使用座位编号系统
                    const seatNumber = SeatNumberGenerator.generateSeatNumber({
                        mode: numberingMode,
                        row,
                        col,
                        rows,
                        cols,
                        baseNum: startNumber
                    })
                    
                    area.seating.push({
                        number: seatNumber,
                        row,
                        col,
                        status: 'empty',  // empty, occupied, selected
                        person: null
                    })
                }
            }
        }
        
        // 获取区域内的座位区块（支持排序）
        function getAreaBlocks(zoneName) {
            return seatingAreas.value
                .filter(area => area.zone === zoneName)
                .sort((a, b) => a.order - b.order)
        }
        
        // 获取座位区块的CSS Grid定位属性
        function getGridPositionStyle(area) {
            const { column, row } = area.gridPosition
            
            return {
                gridColumn: column,
                gridRow: row
            }
        }
        
        return {
            layoutConfig,
            seatingAreas,
            initializeMultiBlockLayout,
            generateSeats,
            getAreaBlocks,
            getGridPositionStyle
        }
    }
}
```

## 7. 常见问题

### Q1：CSS Grid 与 Flexbox 的选择？
**A**: 使用 CSS Grid 用于全局布局定位，使用 Flexbox 用于座位区块内部的行列排列。两者配合能实现最灵活的布局。

### Q2：如何支持不同的屏幕尺寸？
**A**: 通过 CSS Media Queries 调整网格列数：
- 1920px: 20列
- 1024px: 16列
- 768px: 12列
- 480px: 8列

### Q3：如何添加新的座位区块？
**A**: 直接在 `layoutConfigs` 中添加新的区块配置，或通过函数动态添加：
```javascript
const newArea = createSeatingArea({
    id: 'area_new',
    name: '新区域',
    // ... 其他配置
})
seatingAreas.value.push(newArea)
```

### Q4：如何改变布局？
**A**: 支持多个预定义布局，切换时调用：
```javascript
initializeMultiBlockLayout('differentLayout')
```

### Q5：座位编号如何与多块布局配合？
**A**: 每个座位区块有独立的 `numberingMode` 和 `startNumber`，支持5种编号方式的任意组合。
