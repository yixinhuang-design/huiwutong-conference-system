# 排座功能 - UI按钮调整座位编号 (新增功能)

**更新日期**: 2026年2月28日  
**功能类型**: UI界面增强  
**复杂度**: ⭐ 简单  
**改动影响**: 用户体验优化  

---

## 🎉 新增功能概览

现在用户可以**直接通过UI界面调整座位编号**，而不需要修改代码！

### ✨ 核心改进

**之前**: 修改座位编号方式需要修改代码
```javascript
// 需要找到代码文件修改
numberingMode: 'leftToRight'   // ← 必须改代码
```

**现在**: 直接通过UI界面调整 ✅
```
1. 点击座位区的编辑按钮 (🖊️)
2. 在配置窗口中选择编号方式
3. 设置起始编号
4. 点击保存 ✓
```

---

## 🎯 如何使用

### 3个简单步骤

**步骤1**: 点击座位区的**编辑按钮** (🖊️ 铅笔图标)

**步骤2**: 在弹出的配置窗口中找到**"排座方式"**部分

**步骤3**: 
- 选择**编号方式**: 左→右、右→左、中心扩展、蛇形①、蛇形②
- 设置**起始编号**: 默认为1，可改为任意数字（100、1001等）
- 点击**保存**

完成！座位号会立即按新方式显示。

## 📝 改动详情

### 代码层面的修改

**修改文件**: `seating-mgr.html`

**改动1**: 座位区配置弹窗中添加编号方式选择器

```html
<!-- 新增：座位编号方式选择 -->
<div class="form-group">
    <label>座位编号方式</label>
    <select v-model="areaConfig.numberingMode">
        <option value="leftToRight">左→右 (标准线性编号)</option>
        <option value="rightToLeft">右→左 (镜像编号)</option>
        <option value="centerOutward">中心扩展 (VIP优先)</option>
        <option value="alternatingLTRStart">蛇形① (奇偶行交替)</option>
        <option value="alternatingRTLStart">蛇形② (逆向交替)</option>
    </select>
</div>

<!-- 新增：起始编号设置 -->
<div class="form-group">
    <label>起始编号</label>
    <div class="number-input">
        <button @click="areaConfig.startNumber = Math.max(1, (areaConfig.startNumber || 1) - 10)">-10</button>
        <input type="number" v-model.number="areaConfig.startNumber" min="1" placeholder="默认: 1">
        <button @click="areaConfig.startNumber = (areaConfig.startNumber || 1) + 10">+10</button>
    </div>
    <small>座位编号将从此数字开始</small>
</div>

<!-- 新增：预览信息 -->
<div class="numbering-preview">
    <small><strong>预览:</strong> 当前设置为 "{{ getNumberingModeLabel(areaConfig.numberingMode) }}" 编号，从 {{ areaConfig.startNumber || 1 }} 开始</small>
</div>
```

**改动2**: 初始化 `areaConfig` 数据对象

```javascript
const areaConfig = reactive({
    // ... 其他字段 ...
    numberingMode: 'leftToRight',  // 编号方式
    startNumber: 1,                // 起始编号
    // ... 其他字段 ...
});
```

**改动3**: 编辑座位区时加载编号配置

```javascript
const editArea = (area) => {
    // ... 其他代码 ...
    
    // 加载编号方式和起始编号
    Object.assign(areaConfig, {
        // ... 其他字段 ...
        numberingMode: area.numberingMode || 'leftToRight',
        startNumber: area.startNumber || 1,
        // ... 其他字段 ...
    });
    
    showAreaConfigModal.value = true;
};
```

**改动4**: 保存时传递编号参数

```javascript
const newArea = createSeatingArea(
    id,
    areaConfig.name,
    areaConfig.type,
    areaConfig.rows,
    areaConfig.cols,
    areaConfig.angle,
    areaConfig.venueType,
    areaConfig.layoutDirection,
    aisleConfig,
    areaConfig.hasFixedSeats ? areaConfig.fixedSeatsPattern : null,
    areaConfig.numberingMode,    // ← 新增参数
    areaConfig.startNumber       // ← 新增参数
);
```

## 🎓 使用示例

### 示例1: 标准会议配置

**需求**: 会议有2个会场，座位号要区分

**配置方案**:
```
会场A: 编号方式=左→右, 起始编号=1
会场B: 编号方式=左→右, 起始编号=101
```

**步骤**:
1. 编辑会场A，设置编号方式为"左→右"，起始编号为1，保存
2. 编辑会场B，设置编号方式为"左→右"，起始编号为101，保存

**结果**: 
- 会场A座位号: 1-100
- 会场B座位号: 101-200

---

### 示例2: VIP座位配置

**需求**: VIP座位从中心开始编号，号码从1001开始

**配置方案**:
```
VIP区: 编号方式=中心扩展, 起始编号=1001
```

**步骤**:
1. 编辑VIP座位区
2. 选择编号方式为"中心扩展"
3. 设置起始编号为1001
4. 保存

**结果**: 中间座位号最小（1001, 1002），向外依次增大

---

### 示例3: 走道设计配置

**需求**: 中间有走道，左右两侧座位编号避免跨越走道

**配置方案**:
```
编号方式: 蛇形①
起始编号: 1
```

**步骤**:
1. 编辑座位区
2. 选择编号方式为"蛇形①"
3. 起始编号保持默认1
4. 保存

**结果**: 奇偶行交替编号，适合有走道的布局

## 💡 最佳实践

### ✅ 推荐做法

1. **提前规划**: 编辑前先确定编号方式和起始号
2. **统一风格**: 同一会议的座位区使用相同方式
3. **避免重号**: 不同座位区设置不同起始号
4. **查看预览**: 保存前查看预览信息确认无误
5. **保存确认**: 等待保存完成再关闭窗口

### ❌ 避免做法

1. **未保存就关闭**: 修改后一定要点保存
2. **同座位区不同方式**: 避免混乱
3. **频繁改动**: 编号确定后避免频繁修改
4. **座位号冲突**: 确保不同座位区编号不重叠
5. **未验证效果**: 保存前查看预览

## ❓ 常见问题

**Q: 是否需要刷新页面才能看到变化？**
A: 不需要。修改立即生效，无需刷新。

**Q: 改变编号方式后座位位置会变吗？**
A: 不会。只改变座位号显示，位置不变。

**Q: 可以恢复到之前的编号方式吗？**
A: 可以。重新编辑座位区，选择之前的方式再保存即可。

**Q: 多个座位区可以有相同的编号方式吗？**
A: 可以，但起始号要不同，避免编号重复。

**Q: 编号方式改变会影响现有的座位分配吗？**
A: 不会。座位号只是显示，不影响座位数据和人员分配。
