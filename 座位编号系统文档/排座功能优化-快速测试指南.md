## 🔍 进阶验证

### 验证编号系统的数学正确性

**概念**: 每个座位区可有不同的行列数

**测试方案**: 创建一个简单的座位区（建议 2行×3列 = 6个座位）

**验证公式**:
```javascript
// leftToRight (从左到右)
编号 = (行号 - 1) × 列数 + 列号
例: 第2行第1列 = (2-1) × 3 + 1 = 4 ✓

// rightToLeft (从右到左)
编号 = (行号 - 1) × 列数 + (列数 - 列号 + 1)
例: 第2行第1列 = (2-1) × 3 + (3-1+1) = 6 ✓

// centerOutward (中心向外)
需要计算到中心的距离，然后按距离排序
例: 2×3 座位的中心约在 (1.5, 1.5)
离中心最近的是中间两个座位，给号 1 和 2
```

**工具**: 使用浏览器开发者工具
```javascript
// 在控制台中手动验证
const seat = {row: 2, col: 1, number: 4};  // 假设该座位
const area = {rows: 2, cols: 3, numberingMode: 'leftToRight'};
console.log(generateOptimizedSeatNumber(seat, area));  // 应显示 4
```

## 🐛 常见问题与解决

### Q1: 座位区样式没有显示
**A**: 
1. 检查CSS是否加载正确
2. 清除浏览器缓存: `Ctrl+Shift+Delete`
3. 硬刷新: `Ctrl+Shift+R`

### Q2: 编号显示错误
**A**:
1. 确认该座位区的 `numberingMode` 字段是否设置
2. 检查浏览器控制台: `console.log(area)`
3. 查看是否有JS错误

### Q3: 标签显示为 "undefined"
**A**:
1. 编号方式可能设置了不支持的值
2. 检查支持的模式: leftToRight, rightToLeft, centerOutward, alternatingLTRStart, alternatingRTLStart
3. 或者编号方式未设置（此时不显示标签）

### Q4: 交互功能出现故障
**A**:
1. 这些改动不涉及交互逻辑，应该不会影响
2. 检查浏览器控制台错误
3. 尝试刷新或清除缓存

### Q5: 跨浏览器显示不一致
**A**:
1. CSS渐进增强设计，功能优先于美观
2. 基础功能在所有现代浏览器中都可用
3. 样式差异是正常的
