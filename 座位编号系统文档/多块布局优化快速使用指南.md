## 🎯 Grid多块布局详解

### 布局特点
- 使用 **20×10 的网格系统**
- 自动响应式（5个屏幕尺寸断点）
- 座位区可自定义Grid定位

### 预设定位规则

| 座位区类型 | Grid位置 | 说明 |
|-----------|---------|------|
| stage（主席台）| col: 7-14, row: 1 | 顶部中央 |
| vip（VIP区） | col: 7-12, row: 2-6 | 中央主要位置 |
| regular（普通）| col: 2-19, row: 8-10 | 下方全宽 |

### 响应式适配

| 屏幕宽度 | 表现 | 说明 |
|---------|------|------|
| ≥1920px | 20列Grid | 完整多块布局 |
| <1440px | 14列Grid | 缩小版Grid |
| <1024px | 12列Grid | 平板适配 |
| <768px | 8列Grid | 手机竖屏 |
| <480px | 流式布局 | 小屏幕自动降级 |

## 💡 常见使用场景

### 场景1: 展示复杂会议布局

```javascript
// 在页面加载完成后，自动切换到Grid布局
onMounted(() => {
    initializeDefaultData();  // 初始化座位区
    // 延迟切换，让数据先加载
    setTimeout(() => {
        switchLayout('conferenceHall');
    }, 500);
});
```

### 场景2: 添加布局选择器

```html
<!-- 在座位图上方添加布局切换按钮 -->
<div class="layout-switcher">
    <button 
        @click="switchLayout('flowLayout')"
        :class="{active: currentLayoutMode === 'flowLayout'}">
        流式布局
    </button>
    <button 
        @click="switchLayout('conferenceHall')"
        :class="{active: currentLayoutMode === 'conferenceHall'}">
        多块布局
    </button>
</div>
```

### 场景3: 根据窗口大小自动调整布局

```javascript
// 监听窗口大小变化
const handleWindowResize = () => {
    if (window.innerWidth < 768) {
        switchLayout('flowLayout');  // 小屏自动改为流式
    }
};

window.addEventListener('resize', handleWindowResize);
```

### 场景4: 打印预览（使用Grid布局）

```javascript
const printSeating = () => {
    // 临时切换到Grid布局，便于打印
    const originalMode = currentLayoutMode.value;
    switchLayout('conferenceHall');
    
    setTimeout(() => {
        window.print();
        // 打印后恢复原布局
        switchLayout(originalMode);
    }, 500);
};
```

## 📊 数据结构参考

### 座位区的gridLayout对象


## 🐛 故障排除

### 问题1: 切换布局后没有效果

**可能原因**:
- currentLayoutMode.value 没有正确更新
- 座位区的 gridLayout 对象未初始化

**解决方案**:
```javascript
// 检查当前模式
console.log(currentLayoutMode.value);

// 检查座位区数据
console.log(seatingAreas.value[0].gridLayout);

// 手动重新初始化
seatingAreas.value.forEach((area, index) => {
    if (!area.gridLayout) {
        area.gridLayout = {
            gridColumn: 'auto',
            gridRow: 'auto',
            zoneGroup: null,
            order: index,
            displayLayout: 'flow'
        };
    }
});

// 重新应用布局
applyLayoutConfig('conferenceHall');
```

### 问题2: Grid布局排版混乱

**可能原因**:
- 浏览器不支持CSS Grid（极旧的浏览器）
- CSS样式被其他样式覆盖

**解决方案**:
```javascript
// 检查浏览器支持
console.log(CSS.supports('display', 'grid'));

// 强制刷新页面
location.reload();

// 或清空浏览器缓存后重新加载
```

### 问题3: 座位区显示不完整

**可能原因**:
- 座位区尺寸过大超过Grid范围
- 响应式断点处理不当

**解决方案**:
```javascript
// 调整Grid列数和行数
layoutConfigs.conferenceHall.gridTemplate = {
    columns: 24,    // 增加列数
    rows: 12,       // 增加行数
    gap: 15
};

// 重新应用布局
applyLayoutConfig('conferenceHall');
```

## 📞 支持

### 常见问题（FAQ）

**Q: 为什么默认不是Grid布局？**  
A: 为了保持向后兼容，默认使用流式布局。Grid布局是可选功能，需要显式启用。

**Q: 能同时使用多个布局吗？**  
A: 不能。系统中只有一个活跃的布局模式（currentLayoutMode）。切换需要调用 switchLayout() 。

**Q: 如何保存用户选择的布局？**  
A: 当前系统未实现持久化。可以添加到localStorage：
```javascript
// 保存
localStorage.setItem('preferredLayout', currentLayoutMode.value);

// 加载
const saved = localStorage.getItem('preferredLayout');
if (saved) switchLayout(saved);
```

**Q: Grid布局支持拖拽调整位置吗？**  
A: 当前不支持。可以通过修改 assignGridPosition() 函数实现。
