#### 4. 过滤和排序逻辑（第1495-1515行）

```javascript
if (typeKey === 'pending') {
    records = records.filter(item => item.status === 0);    // ← 待审核
} else if (typeKey === 'approved') {
    records = records.filter(item => item.status === 1);    // ← 已通过
} else if (typeKey === 'registered') {
    records.sort((a, b) => {
        if (a.status === 0 && b.status !== 0) return -1;    // ← 待审核优先
        if (a.status !== 0 && b.status === 0) return 1;
        return new Date(b.createdTime || b.registrationTime) - 
               new Date(a.createdTime || a.registrationTime);
    });
}
```

**Q: 为什么不修改后端？**  
A: 当前数据量只有42条，前端过滤足够。后端修改需要编译部署，成本高。

**Q: 搜索功能还能用吗？**  
A: 可以。搜索在过滤后的数据上进行。

**Q: 分页会不会有问题？**  
A: 不会。分页基于过滤后的记录数计算。

**Q: 数据量很大时怎么办？**  
A: 建议修改后端添加status参数支持，进行服务端过滤。
