## 二（补充）：各模块深度代码实现分析

> 以下分析基于对全部267个Java文件和23个HTML前端页面的逐文件代码阅读，所有结论均有代码依据。

### M1. conference-common — 公共基础库（14个Java文件）

#### 核心组件实现详情

| 组件 | 文件 | 行数 | 核心机制 |
|------|------|------|----------|
| **租户过滤器** | TenantFilter.java | 109 | Jakarta Servlet Filter，从JWT提取tenantId/userId设入ThreadLocal |
| **SQL拦截器** | TenantInterceptor.java | 173 | MyBatis InnerInterceptor，在SELECT语句WHERE中自动注入`tenant_id=?` |
| **上下文持有者** | TenantContextHolder.java | 62 | 3个ThreadLocal变量（tenantId/userId/username） |
| **全局异常** | GlobalExceptionHandler.java | 131 | @ControllerAdvice，处理10种异常类型 |
| **JWT工具(实例)** | util/JwtUtils.java | 133 | @Component，Hutool JWT库，HS384算法 |
| **JWT工具(静态)** | utils/JwtUtils.java | 130 | 静态方法，JJWT库，HMAC-SHA算法 |
| **统一响应** | Result.java | 108 | 泛型R<T>封装，含code/message/data/timestamp |
| **错误码** | ResultCode.java | 66 | 枚举定义，200-5004共28个错误码 |
| **基础实体** | BaseEntity.java | 30 | createTime/updateTime/createBy/updateBy + delFlag(⚠️ @TableField(exist=false)不映射DB) |
| **API监控** | ConferenceDataMonitor.java | 108 | Filter拦截/api/*请求，异步POST访问日志到:8088 |
| **监控配置** | MonitorConfig.java | 40 | @ConditionalOnProperty注册Filter，排除自身日志接口 |

#### TenantFilter 完整处理流程
```
请求进入 → OPTIONS预检? → 直接放行
         → 非/api/路径? → 直接放行
         → 读取Authorization header → 提取Token(支持"Bearer xxx"和裸token)
         → jwtUtils.validateToken() → 成功: 设tenantId+userId到ThreadLocal
                                     → 失败: ⚠️ 请求仍然继续(不做认证拦截!)
         → filterChain.doFilter() → finally清除ThreadLocal
```

#### TenantInterceptor SQL拦截详情
- **仅拦截SELECT** — INSERT/UPDATE/DELETE不注入tenant_id条件
- **白名单表**(不拦截): sys_tenant, sys_user, sys_role, sys_permission, audit_log, chat_group_member, task_assignee, task_log, data_service_health, data_system_metrics
- **注入方式**: 反射修改BoundSql.sql字段，在WHERE/GROUP BY/ORDER BY之间插入`AND tenant_id=X`
- **JOIN处理**: 正则提取第一个表别名，生成`别名.tenant_id=X`

#### 两份JwtUtils关键差异

| 对比项 | util/JwtUtils (实例版) | utils/JwtUtils (静态版) |
|--------|----------------------|------------------------|
| Spring管理 | ✅ @Component | ❌ 普通类 |
| JWT库 | Hutool JWT | JJWT |
| 签名算法 | HS384 | HMAC-SHA (由密钥长度决定) |
| tenantId存储 | payload中的`tenantId`字段 | payload中的`tenantId`字段 |
| userId存储 | payload中的`userId`字段 | JWT的`subject`字段 |
| 被使用者 | TenantFilter注入使用 | 可能被auth模块使用 |

### M2. conference-gateway — API网关（2个Java文件）

#### AuthGlobalFilter 完整逻辑链
```
请求 → getToken(Authorization header) → isWhitePath?
       → YES: 直接放行(⚠️ 不清除已有X-User-Id等header，可被伪造)
       → NO + 无Token: 返回401 "未提供认证Token"
       → NO + 有Token: parseToken(JJWT库验证签名+过期)
           → 过期: 401 "Token已过期"
           → 无效: 401 "Token无效"
           → 成功: mutate请求头注入 X-User-Id/X-Tenant-Id/X-Username → 转发下游
```

#### 白名单路径（无需Token）
`/api/v1/auth/login`, `/api/v1/auth/register`(❌未实现), `/api/v1/auth/captcha`, `/api/v1/auth/sms-code`, **`/api/v1/tenant/**`**(⚠️全部租户管理无认证!), `/actuator/**`, `/doc.html`, `/swagger-resources/**`, `/v3/api-docs/**`, `/webjars/**`

### 4.3 核心架构决策差异

| 决策点 | 路线图方案 | 实际方案 | 原因分析 |
|--------|-----------|---------|---------|
| 数据隔离 | 数据库级隔离 | 行级(tenant_id)隔离 | 单租户场景下更简单高效 |
| 服务调用 | OpenFeign + Hystrix | JdbcTemplate跨库 | 避免服务启动依赖 |
| 流处理 | Flink | 定时轮询(ScheduledTask) | 数据量级暂不需要Flink |
| 分布式事务 | Seata | 无 | 当前单体运行无需 |
| 前端框架 | Vue 3 SPA | Vue 3 CDN + MPA | 保持原有架构风格 |
| 部署方式 | Docker + K8s | 本地IDEA直接运行 | 开发阶段 |
