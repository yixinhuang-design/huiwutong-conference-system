## 一、系统概述

智能会务系统是一套覆盖**会议全生命周期**的管理平台，包含以下六大阶段：

```
阶段一：会议创建 → 阶段二：报名审核 → 阶段三：会前准备(日程/排座/名册)
    → 阶段四：通知发送 → 阶段五：会议进行(群组/任务/AI) → 阶段六：总结归档
```

### 系统架构

| 组件 | 说明 | 端口 |
|------|------|------|
| 管理端PC | Vue 3 多页应用 | 8000 |
| API网关 | Spring Cloud Gateway | 8080 |
| 认证服务 | JWT认证+租户管理 | 8081 |
| 报名服务 | 报名/审核/分组/手册 | 8082 |
| 通知服务 | 通知+预警 | 8083 |
| 会议服务 | 会议/日程/附件/归档 | 8084 |
| AI服务 | 智能问答/知识库 | 8085 |
| 排座服务 | 座位/餐住行 | 8086 |
| 数据服务 | 监控/统计/大屏 | 8088 |
| 协同服务 | 群组/任务/聊天 | 8089 |

## 二、环境准备与启动

### 2.1 前置要求

| 软件 | 版本 | 说明 |
|------|------|------|
| JDK | 21+ | 后端运行环境 |
| MySQL | 9.6 | 端口3308, 密码Hnhx@123 |
| Node.js | 18+ | 前端静态服务器 |
| IDEA | 2024+ | 推荐开发环境 |

### 2.2 数据库初始化

```sql
-- 连接MySQL
mysql -h 127.0.0.1 -P 3308 -u root -pHnhx@123

-- 创建数据库(如未创建)
CREATE DATABASE IF NOT EXISTS conference_auth DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE IF NOT EXISTS conference_registration DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE IF NOT EXISTS conference_notification DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE IF NOT EXISTS conference_seating DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE IF NOT EXISTS conference_collaboration DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE IF NOT EXISTS conference_db DEFAULT CHARACTER SET utf8mb4;

-- 依次执行建表脚本
SOURCE 1_create_database.sql;
SOURCE 2_create_tables.sql;
SOURCE 3_create_indexes.sql;
```

### 2.3 启动后端服务

**方式一：IDEA启动（推荐开发环境）**

1. IDEA打开 `backend/conference-backend/` 目录
2. 等待Maven依赖下载完成
3. 按需启动各模块的Application类：
   - `ConferenceAuthApplication` (8081) — **必须首先启动**
   - `ConferenceMeetingApplication` (8084)
   - 其他服务按需启动

**方式二：命令行启动**

```bash
cd backend/conference-backend

# 编译
mvn clean package -DskipTests

# 启动(示例:会议服务)
java -jar conference-meeting/target/conference-meeting-1.0.0.jar
```

### 2.4 启动前端

```bash
cd admin-pc/conference-admin-pc
node server.js
# 服务启动在 http://localhost:8000
```

浏览器访问 `http://localhost:8000` 即可进入系统。

## 四、会议管理

### 4.1 查看会议列表

首页显示所有会议，支持：
- **状态筛选**: 筹备中/进行中/已结束
- **时间筛选**: 按日期范围
- **关键词搜索**: 会议名称模糊搜索

### 4.2 创建新会议

1. 点击首页右上角 **"+ 创建会议"**
2. 填写基本信息：
   - 会议名称（必填）
   - 开始/结束时间（必填）
   - 地点
   - 描述
   - 封面图片（可上传）
3. 配置报名方式、签到方式
4. 设置工作人员
5. 点击 **"提交"**

### 4.3 会议管理中心

点击会议卡片进入**会议详情(conference-detail)**，这是整个系统的核心入口：

```
会议管理中心
├── 六阶段检查清单 (CheckList)
│   ├── 阶段一：会议创建 ✓
│   ├── 阶段二：报名与审核
│   ├── 阶段三：会前准备
│   ├── 阶段四：通知发送
│   ├── 阶段五：会议进行
│   └── 阶段六：总结归档
├── 统计概览 (报名数/任务数/群组数/签到数)
├── 快捷操作入口
│   ├── 报名管理 → registration-mgr.html
│   ├── 日程管理 → schedule-mgr.html
│   ├── 智能排座 → seating-mgr.html
│   ├── 名册生成 → roster-generation.html
│   ├── 精准通知 → notification-mgr.html
│   ├── 群组管理 → group-management.html
│   ├── 任务分派 → task-management.html
│   ├── AI咨询 → ai-assistant-console.html
│   └── 会议归档 → archive-management.html
└── 操作日志
```

## 七、智能排座

### 7.1 进入排座管理

从会议详情页点击 **"智能排座"**

> ⚠️ 这是系统中最复杂的页面(11000+行代码)

### 7.2 创建会场

1. 点击 **"+ 新建会场"**
2. 设置会场名称、行数、列数
3. 系统自动生成座位网格

### 7.3 座位操作

| 操作 | 说明 |
|------|------|
| 自动分配 | 根据算法自动分配座位 |
| 手动调整 | 拖拽座位或点击交换 |
| 插入过道 | 在指定位置插入过道 |
| 导入参会者 | 从报名数据导入 |
| 座位编号 | 自定义编号规则 |

### 7.4 智能分配算法

系统提供3种分配策略：
1. **顺序分配** — 按报名顺序依次分配
2. **分组分配** — 同部门/同组坐在一起
3. **优先级分配** — VIP/领导优先前排

> ⚠️ **当前状态**: 以上 3 种算法均为**空壳实现**，调用后返回空对象，实际不会分配座位。当前只能使用**手动拖拽**方式分配座位。

### 7.5 餐住行安排

在排座页面的附加Tab中：
- **用餐安排**: 设置用餐时间、地点、桌号
- **住宿安排**: 设置房间分配、楼层
- **交通安排**: 设置车辆、路线、上车点

## 八、精准通知

### 8.1 发送通知

1. 进入 **"精准通知"** 页面
2. 选择通知类型（短信/邮件/系统通知）
3. 选择接收人（全部/分组/个人）
4. 编辑通知内容（支持模板变量）
5. 选择发送时间（立即/定时）
6. 发送

### 8.2 通知模板

- 系统预置常用模板
- 支持自定义模板
- 模板变量：{姓名}、{会议名称}、{时间} 等

### 8.3 催办提醒

- 查看未报名单位/个人
- 一键发送催报通知
- 查看催办效果统计

> ⚠️ **当前状态**:
> - 所有发送渠道（短信/邮件/微信/APP推送）均为**模拟实现**，仅输出日志，不会真实发送
> - 定时发送功能不生效（无调度器执行延时任务）
> - 模板变量 `{{name}}` 等不会被实际替换

## 九、群组与任务协同

### 9.1 群组管理

1. 进入 **"群组管理"** 页面
2. **创建群组**: 设置群名、类型
3. **管理成员**: 添加/移除/设置角色
4. **群组聊天**: 实时消息（WebSocket）
5. **群组资料**: 上传/下载共享文件

### 9.2 任务分派

1. 进入 **"任务分派"** 页面
2. **创建任务**: 标题、描述、优先级、截止时间
3. **分配执行人**: 选择一个或多个执行人
4. **跟踪进度**: 待处理→进行中→已提交→已完成
5. **催办**: 对超期任务发送催办

### 9.3 任务生命周期

```
创建 → 分配 → 执行中 → 提交 → 完成
                ↘ 催办 ↗
```

## 附录B：常见问题排查

### Q1: 登录失败

**现象**: 提交登录后无响应或报错

**排查步骤**:
1. 确认认证服务(8081)已启动
2. 检查MySQL连接(3308端口)
3. 浏览器F12查看网络请求是否到达后端
4. 确认`conference_auth`数据库中有用户数据

### Q2: 页面加载显示模拟数据

**现象**: 页面正常显示但数据明显是假的

**原因**: 前端有fallback机制，API调用失败时自动加载mock数据

**排查步骤**:
1. F12查看Console是否有API调用错误
2. 确认对应后端服务已启动
3. 确认端口号正确(检查前端代码中硬编码的端口)

### Q3: 归档同步后数据为0

**现象**: 调用sync后返回全为0

**排查步骤**:
1. 确认会议ID正确且有真实数据
2. 检查`conf_schedule_attachment`表是否有该会议的附件
3. 检查`conf_registration`表是否有该会议的报名数据
4. 检查`conference_collaboration.chat_group`表是否有该会议的群组

### Q4: 跨域错误

**现象**: 浏览器控制台报CORS错误

**排查步骤**:
1. 确认后端CorsConfig已配置允许localhost:8000
2. 或通过API网关(8080)访问，Gateway已配置CORS
3. 检查请求头中的Origin

### Q5: 端口被占用

**现象**: 服务启动失败，端口已被占用

**解决**:
```bash
# Windows - 查找并终止占用端口的进程
netstat -ano | findstr :8084
taskkill /PID <进程ID> /F

# PowerShell
Get-Process -Id (Get-NetTCPConnection -LocalPort 8084).OwningProcess | Stop-Process -Force
```

### Q6: 数据库连接失败

**现象**: 服务启动报数据库连接错误

**排查步骤**:

## 附录C：已知功能限制清单（补充）

> 以下限制基于代码深度扫描结果，操作时需注意

### 认证与安全

| 功能 | 限制说明 |
|------|---------|
| 退出登录 | 空实现，Token退出后仍然有效至过期 |
| Token刷新 | 空实现，返回null |
| 租户管理 | 无需认证即可访问（在Gateway白名单中） |
| 密码安全 | 支持MD5密码（不安全），无账户锁定机制 |

### 通知服务

| 功能 | 限制说明 |
|------|---------|
| 短信/邮件/微信发送 | 纯模拟，仅输出日志 |
| 定时发送 | 无调度器，不会自动触发 |
| 模板变量 | `{{xxx}}`占位符不会被替换 |

### 排座服务

| 功能 | 限制说明 |
|------|---------|
| 智能分配（3种算法） | 空壳实现，return new Object() |
| 人员分配记录 | 仅更新计数，不记录具体人员 |
| Entity字段 | 与数据库Schema存在6+处不匹配 |

### AI服务

| 功能 | 限制说明 |
|------|---------|
| 智能对话 | 关键词匹配+模板，无LLM |
| OCR识别 | 返回固定字符串"OCR识别结果" |
| 语音识别 | 返回固定字符串"语音识别结果" |
| 文档解析 | 仅支持.txt，PDF/DOCX返回空 |

### 报名服务

| 功能 | 限制说明 |
|------|---------|
| 花名册PDF | 返回空byte数组 |
| Excel导出 | 大数据量时内存占用高（非流式） |
| 分组管理 | 直接使用JdbcTemplate，存在N+1查询 |

### 协同服务

| 功能 | 限制说明 |
|------|---------|
| WebSocket聊天 | userId无Token验证，可伪造身份 |
| 未读消息计数 | 始终返回0 |
| 任务逾期检测 | 未实现自动检测 |

### 数据服务

| 功能 | 限制说明 |
|------|---------|
| 网络监控 | Math.random()生成虚假数据 |
| PDF导出 | 返回空响应 |
