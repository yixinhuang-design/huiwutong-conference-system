### 4️⃣ 验证 CSS 文件加载 (2 分钟)

**方法 1: 使用 Network 标签**

1. 点击 DevTools 中的 **Network** 标签
2. 刷新页面 (F5)
3. 在搜索框中输入: `seating-function-modal`
4. 应该看到:
   ```
   seating-function-modal.css   Status: 200   Size: XX KB
   ```

✅ 状态码是 **200** = CSS 文件加载成功  
❌ 状态码是 **404** = CSS 文件未找到 (检查路径)

**方法 2: 使用 Console 控制台**

1. 点击 DevTools 中的 **Console** 标签
2. 在控制台输入：
   ```javascript
   document.styleSheets.length
   ```
3. 应该返回一个数字（例如：10）
4. 再输入：
   ```javascript
   Array.from(document.styleSheets).find(s => s.href.includes('seating-function-modal'))
   ```
5. 应该返回一个对象（不是 undefined）

✅ 返回对象 = CSS 文件加载成功  
❌ 返回 undefined = CSS 文件未加载

## 🐛 常见问题快速修复

### ❌ 问题 1: CSS 文件返回 404

```
症状: Network 标签中看到红色的 404 错误

原因: CSS 文件路径不正确或文件不存在

快速修复:
1. 检查文件是否存在:
   ls g:\huiwutong新版合集\admin-pc\conference-admin-pc\css\seating-function-modal.css
   
2. 如果不存在，重新创建 CSS 文件

3. 检查 seating-mgr.html 中的路径是否正确:
   应该是: ../css/seating-function-modal.css
   不要是: ./css/seating-function-modal.css (错)
```

### ❌ 问题 2: 弹窗没有样式 (显示为默认样式)

```
症状: 弹窗打开了，但没有蓝色头部、没有玻璃效果

原因: CSS 文件加载成功但规则未应用

快速修复:
1. 强制刷新: Ctrl+Shift+R
2. 清除浏览器缓存
3. 在 Console 中检查:
   document.querySelector('.modal-header')
   window.getComputedStyle(document.querySelector('.modal-header')).backgroundColor
   
4. 如果返回的不是蓝色 (#667eea)，说明样式未应用
```

### ❌ 问题 3: 弹窗颜色看起来不对

```
症状: 颜色不是期望的蓝色或白色

原因: CSS 变量未定义或被覆盖

快速修复:
1. 检查 conference-theme.css 是否包含:
   --primary-color: #667eea
   
2. 在 Console 中验证:
   window.getComputedStyle(document.documentElement).getPropertyValue('--primary-color')
   应该返回: #667eea 或 rgb(102, 126, 234)
```

### ❌ 问题 4: 弹窗超出屏幕

```
症状: 弹窗太大，按钮或内容被切掉

原因: 弹窗宽度或高度设置不当

快速修复:
1. 检查 CSS 中的限制:
   max-width: 95vw;      (应该有)
   max-height: 90vh;     (应该有)
   
2. 检查 modal-body 的滚动:
   overflow-y: auto;     (应该有)
   max-height: 60vh;     (应该有)
   
3. 如果没有这些属性，手动添加到 CSS
```
