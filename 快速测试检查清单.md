## 🐛 常见问题排查

### 问题1：页面打开后日程为空
```
原因可能：localStorage 中有旧数据冲突
解决：
  1. F12 → Console
  2. 输入：localStorage.clear()
  3. 按 Enter
  4. 刷新页面 F5
```

### 问题2：切换日程后座位没变
```
原因可能：数据隔离未生效
解决：
  1. 打开 F12 → Application → LocalStorage
  2. 查找 schedule_seating_* 的键
  3. 检查不同日程的数据是否不同
  4. 如果相同，说明隔离有问题
```

### 问题3：排座后座位还是空的
```
原因可能：排座算法没有执行
解决：
  1. 打开 F12 → Console
  2. 查看是否有错误消息
  3. 确认是否执行了 executeAutoArrange()
  4. 检查参会人员数据是否存在
```

### 问题4：Modal 同时显示
```
原因可能：日程选择流程未修复
解决：
  1. 点击"切换日程"
  2. 确认只显示日程列表 modal（不显示确认 modal）
  3. 选择日程后，列表应该关闭，确认 modal 显示
  4. 如果两个同时显示，说明修改未生效
```
