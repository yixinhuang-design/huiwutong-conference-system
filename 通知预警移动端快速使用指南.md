# 通知预警移动端 - 快速使用指南

## 📱 三个新增/优化页面

### 1. 通知管理移动版（协管员）
**文件**：`app/staff/notice-mobile.html`  
**用途**：快速发送各类通知  
**访问**：`首页` → `通知` 标签

**快捷功能**：
- ⚡ **一键发送**：6个常用通知类型，点击即发
- 📋 **模板库**：5个预设模板，可折叠展开
- 📊 **快速统计**：总参会、待发、已发一目了然
- 🔍 **状态筛选**：全部、待发送、已发送、草稿

**使用场景**：
```
外出时需要快速发送座位通知？
1. 打开 notice-mobile.html
2. 点击快捷发送中的"座位通知"
3. 点击"立即发送"
✅ 完成！（15秒搞定）
```

---

### 2. 预警管理移动版（协管员）
**文件**：`app/staff/alert-mobile.html`  
**用途**：快速处理预警事件、配置规则  
**访问**：`首页` → `预警` 标签

**快捷功能**：
- 🚨 **实时概览**：待处理/处理中/已处理一屏显示
- 🎨 **优先级可视化**：颜色和图标快速识别等级
- ⚡ **快速筛选**：4个按钮快速切换视图
- 📝 **快速处理**：无需进入详情页，直接处理

**使用场景**：
```
开会时看到系统预警，需要快速确认？
1. 打开 alert-mobile.html
2. 看到待处理预警（红色高亮）
3. 点击"处理"按钮
✅ 完成！（10秒搞定）
```

---

### 3. 学员通知中心优化
**文件**：`app/learner/notifications.html`  
**用途**：学员查看各类通知  
**访问**：`首页` → `通知` 标签

**新增功能**：
- 📌 **未读计数**：标题显示未读数量
- ✅ **全部已读**：一键标记所有通知已读
- 🎨 **自动标记**：点击通知自动变为已读

## ⚙️ 设置与自定义

### 通知模板自定义
```
1. 打开模板库（点击"模板库"卡片展开）
2. 选择要使用的模板
3. 系统自动加载模板内容
4. 编辑内容（支持变量：{姓名}、{座位号}等）
5. 选择发送对象/渠道
6. 发送
```

### 预警规则配置
```
1. 点击右上角齿轮按钮（⚙️）
2. 选择新建规则
3. 输入规则名称
4. 选择预警指标（报名率、签到率等）
5. 设置阈值
6. 选择通知方式（短信、系统通知等）
7. 保存规则
```

## 🔍 常见问题

### Q1: 如何快速发送通知到特定人员？
```
A: 使用快捷按钮后，进入"编辑"模式可以修改发送对象。
   默认是全部人员，可改为按分组/状态筛选。
```

### Q2: 预警处理后如何标记为已完成？
```
A: 有两种方式：
   1. 系统自动检测（当指标恢复正常时自动标为已处理）
   2. 手动标记（在处理中的预警点"更新"后可标为已处理）
```

### Q3: 如何查看发送失败的原因？
```
A: 点击已发送的通知项，查看详情中的"失败原因"说明。
   然后可以点击"重新发送"仅对失败目标重发。
```

### Q4: 学员如何快速查看所有未读通知？
```
A: 进入学员通知中心（notifications.html）
   - 标题显示未读计数（如"3个未读通知"）
   - 未读项标记为"未读"状态
   - 点击查看详情，自动标记已读
```

### Q5: 如何一键标记所有通知已读？
```
A: 在学员通知中心，点击右上角"全部已读"按钮，
   所有未读通知立即标记为已读。
```

## 📝 使用建议

### 最佳实践

#### 发送通知
✅ **DO:**
- 使用快捷发送按钮快速操作
- 选择合适的发送渠道（短信/APP）
- 定时发送重要通知（避免打扰）
- 保存常用模板

❌ **DON'T:**
- 在不确定内容的情况下发送
- 频繁发送相同内容
- 使用错误的通知类型
- 忘记预览发送对象

#### 处理预警
✅ **DO:**
- 优先处理高优先级预警
- 及时更新处理进度
- 添加处理备注便于追踪
- 定期检查已处理预警

❌ **DON'T:**
- 忽视低优先级预警
- 处理后不备注原因
- 重复处理同一预警
- 无故删除规则

## 📞 技术支持

### 常见问题排查

**页面加载异常？**
```
检查清单：
1. 确认文件路径正确：app/staff/notice-mobile.html
2. 清除浏览器缓存
3. 检查网络连接
4. 尝试在不同浏览器打开
```

**按钮无响应？**
```
检查清单：
1. 确认已连接网络
2. 检查浏览器控制台是否有错误（F12）
3. 确认后端API是否在线
4. 刷新页面重试
```

**数据显示不完整？**
```
检查清单：
1. 检查网络延迟（超过10秒可能数据未加载）
2. 刷新页面
3. 检查后端是否返回数据
4. 联系技术人员
```
