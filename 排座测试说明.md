# 排座功能测试说明

## 问题诊断

用户反馈"执行排座不生效"，现已进行以下修复：

### 1. **根本原因**
- 页面初始加载时，如果没有从localStorage读取数据，initializeDefaultData()只初始化了座位区，但**没有初始化参会人员数据**
- executeAutoArrange函数检查attendees.value.length是否为0，如果为空则直接返回并提示"请先导入参会人员"

### 2. **已实施的修复**

#### 修复1：添加初始化测试数据
修改`initializeDefaultData()`函数，添加15名测试参会人员：
- 2名VIP（书记、副局长）
- 13名普通参会人员（分属多个部门）

#### 修复2：添加座位区存在性检查
在`executeAutoArrange()`函数中添加检查：
```javascript
if (!seatingAreas.value || seatingAreas.value.length === 0) {
    toast.warning('请先创建座位区');
    return;
}
```

#### 修复3：添加详细的控制台日志
在以下函数中添加日志用于诊断：
- `executeSmartAlgorithm()`：输出参会人员和座位区统计
- `assignVipSeats()`：输出VIP座位分配日志
- `assignByDepartmentWithPosition()`：输出部门分组和区域分配日志
- `assignAttendeesToArea()`：输出每个座位的分配信息
- `assignRegularSeats()`：输出普通座位分配进度

## 测试步骤

1. **打开排座页面**
   - URL: http://127.0.0.1:8080/pages/seating-mgr.html

2. **验证初始数据**
   - 页面应显示3个座位区（A区、B区、C区）
   - 统计信息应显示15名参会人员

3. **执行排座**
   - 点击"执行排座"按钮
   - 选择排座算法（默认为"智能推荐"）
   - 确认对话框

4. **检查排座结果**
   - 打开浏览器开发者工具（F12）
   - 查看Console标签
   - 应该看到详细的排座日志输出，如：
     ```
     执行智能排座算法，参会人员总数: 15
     座位区数量: 3
     VIP参会人员: 2 普通参会人员: 13
     分配VIP座位，VIP人员数: 2
     使用第一个区域作为VIP区
     分配VIP座位: 1排1号 -> 李明
     分配VIP座位: 1排2号 -> 王红
     ...
     ```

5. **验证视觉效果**
   - 座位应显示参会人员姓名
   - VIP座位应有不同的视觉标识（颜色）
   - 统计信息应更新

## 如果仍然不生效

### 检查清单

1. **浏览器缓存**
   - 清空浏览器缓存或使用无痕模式
   - 进行Ctrl+Shift+R硬刷新

2. **localStorage数据**
   - 打开开发者工具 → Application → LocalStorage
   - 查找键 `seating_conf_20240615_001`
   - 如果存在旧数据，删除它
   - 或者手动设置 `seating_test` localStorage项为空

3. **控制台错误**
   - 查看F12开发者工具的Console标签
   - 记录任何JavaScript错误信息
   - 检查是否有`ReferenceError: progress is not defined`等错误

4. **参会人员数据**
   - 在Console中执行: `console.log(attendees.value)`
   - 应该看到15个对象数组
   - 检查是否都有`isVip`属性

5. **座位区数据**
   - 在Console中执行: `console.log(seatingAreas.value)`
   - 应该看到3个座位区
   - 每个区应有`rows`数组和`columnAisles`数组

## 预期结果

执行排座后：
- VIP座位应在A区前几排分配给VIP参会人员
- 普通参会人员按部门分组分配到B区和C区
- 每个参会人员应有对应的座位
- 统计信息显示"已占用座位：15 / 计划座位：150"

## 可能的问题及解决方案

| 问题 | 原因 | 解决方案 |
|------|------|--------|
| 点击"执行排座"后无反应 | 参会人员数为0 | 检查localStorage或使用初始化的测试数据 |
| 提示"请先创建座位区" | 座位区为空 | 页面会自动初始化3个默认座位区 |
| 排座完成但座位没有人 | seats的status检查失败 | 检查是否有aisles标记的座位被误认为不可用 |
| 找不到VIP区域 | 没有type='vip'的区域 | 使用第一个区域作为VIP区（已在代码中处理） |
| 座位不足以分配所有人员 | 座位总数少于参会人员数 | 增加座位行数或减少测试参会人员数 |
