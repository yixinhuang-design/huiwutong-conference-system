## 📌 按依赖关系排序

### 必须先做的（前置条件）
```
1. login.html (所有功能的入口)
   ↓
2. training-create.html (定义培训)
   ├─ registration-manage.html (处理报名)
   │  ├─ seat-manage.html (分配座位)
   │  └─ grouping-manage.html (分组)
   │
   ├─ schedule.html (日程)
   │  └─ checkin.html (签到)
   │
   └─ notice-manage.html (通知)
```

### 可以并行开发的
- chat-room.html (learner/staff) - 可并行
- guestbook.html - 不依赖前置功能
- feedback.html/evaluation.html - 在注册完成后

### 后续优化的
- dashboard.html - 依赖前面所有数据
- data-analysis.html - 依赖足够的数据积累
- alert-config.html - 依赖运行监控
