## 📌 5 分钟快速了解

### 这个项目是什么？
一个完整的日程管理系统，包括：
- **后端**: 17 个 API 接口，基于 Spring Boot
- **数据库**: 8 个表，存储日程相关数据
- **前端**: Web 管理后台 + 两个 App（学员端和管理端）

### 这个项目有什么用？
**用户可以**:
- 创建、编辑、删除日程
- 发布日程供学员查看
- 设置签到、报到、提醒等功能
- 在 App 和 Web 上管理日程

**技术人员需要**:
- 理解系统架构
- 修改和扩展功能
- 部署和维护系统

### 项目的组成部分
```
用户界面层
├── Web 管理后台 (schedule-mgr.html)
├── App 学员端 (learner/schedule.vue)
└── App 管理端 (staff/schedule.vue)
    ↓
API 客户端 (schedule-api.js)
    ↓
REST API (17 个端点)
    ↓
业务逻辑 (ScheduleService)
    ↓
数据库 (MySQL 8 个表)
```

## ✅ 第一步：启动项目

### 1. 启动后端服务（2 分钟）

```
App 管理端: App → 日程管理 → 管理日程
```

## 🎯 基本概念理解

### 日程状态（5 种）
```
0 = 草稿（新创建的日程）
1 = 已发布（用户能看到）
2 = 进行中（自动转换）
3 = 已完成（自动转换）
4 = 已取消（手动操作）
```

### 日程设置（3 个）
- **报到**: 用户到达时需要报到（打卡）
- **签到**: 用户参加时需要签到
- **提醒**: 提前提醒用户参加

### 多租户隔离
每个组织/会议独立存储和查看自己的日程，不会看到其他组织的数据

## 📝 常见操作

### 创建日程（Web 后台）
```
1. 点击"添加日程"按钮
2. 填写：
   - 标题（必填）
   - 开始时间（必填）
   - 结束时间（必填）
   - 地点（必填）
   - 主持人（可选）
   - 主讲人（可选）
   - 备注（可选）
3. 设置选项（可选）：
   - 是否需要报到
   - 是否需要签到
   - 是否需要提醒
4. 点击"添加"保存
```

### 发布日程
```
1. 在日程列表点击日程打开详情
2. 点击"发布"按钮（仅草稿状态可见）
3. 日程状态变为"已发布"
4. 用户现在可以在 App 看到这个日程
```

### 查看统计
```
Web 管理后台顶部显示：
- 总日程数：所有日程的数量
- 总时长：所有日程的总时长（小时）
- 需签到日程：设置了签到的日程数量
- 需提醒日程：设置了提醒的日程数量
```

## 🔍 理解 API 接口

### API 是什么？
- 前端（App/Web）通过 API 向后端请求数据
- API 就像餐厅的菜单，告诉你可以点什么菜

### 17 个 API 接口一览
```
基础操作:
  POST   /api/schedule/create          创建日程
  GET    /api/schedule/{id}            获取日程
  PUT    /api/schedule/{id}            编辑日程
  DELETE /api/schedule/{id}            删除日程

状态操作:
  PUT    /api/schedule/{id}/publish    发布
  PUT    /api/schedule/{id}/cancel     取消
  POST   /api/schedule/{id}/duplicate  复制

查询操作:
  GET    /api/schedule/list            分页列表
  GET    /api/schedule/all             所有日程
  GET    /api/schedule/need-checkin    需签到
  GET    /api/schedule/ongoing         进行中
  GET    /api/schedule/next            下一个
  ...（还有 6 个）
```

### API 调用示例
```javascript
// 创建日程
const response = await ScheduleAPI.createSchedule({
  title: "开幕式",
  startTime: "2024-06-15T09:00",
  endTime: "2024-06-15T10:30",
  location: "主会场"
});

// 发布日程
await ScheduleAPI.publishSchedule(scheduleId);

// 查询所有日程
const schedules = await ScheduleAPI.allSchedules(conferenceId);
```

### 如果功能不正常

**问题 1: 页面显示模拟数据，不是真实数据**
```
原因: API 调用失败
解决:
1. 检查后端是否正在运行
2. 打开浏览器 F12 → Network 标签
3. 查看是否有错误请求
4. 查看是否有 401 or 403 错误（权限问题）
```

**问题 2: 创建日程后没有出现在列表**
```
原因: API 请求出错
解决:
1. 打开浏览器 F12 → Console 标签
2. 查看是否有红色错误信息
3. 检查表单是否都填写了必填项
4. 检查数据库是否正常运行
```

**问题 3: 按钮点击无反应**
```
原因: JavaScript 错误或 schedule-api.js 未加载
解决:
1. 打开浏览器 F12 → Console 标签
2. 输入: ScheduleAPI
3. 如果返回 undefined，说明 API 模块未加载
4. 检查 schedule-mgr.html 是否引入了 schedule-api.js
```
