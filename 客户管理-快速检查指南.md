## ⚠️ 故障排查

### 问题1: 页面显示"加载失败"

```
症状: 打开页面后显示红色"加载失败"提示

排查步骤:
1. ❌ 检查后端是否运行: http://localhost:8081 能否访问
2. ❌ 查看浏览器Console是否有错误信息
3. ❌ 检查MySQL是否正在运行
4. ❌ 查看后端服务日志是否有错误
5. ❌ 尝试访问 http://localhost:8081/api/tenant/list 直接测试API

解决方案:
✅ 启动后端服务: mvn spring-boot:run -pl conference-auth
✅ 启动MySQL服务
✅ 刷新浏览器重试
✅ 检查防火墙是否阻止8081端口
```

### 问题2: 创建客户返回"403 Forbidden"

```
症状: 点击创建客户后，弹出"仅超级管理员可访问"错误

排查步骤:
1. ❌ 确认用户是否为管理员
2. ❌ 检查Authorization header中的token
3. ❌ 查看后端权限验证逻辑

解决方案:
✅ 目前后端权限验证为宽松模式（isSuperAdmin永远返回true）
✅ 如果需要真正的权限控制，需要配置用户角色
✅ 在实际生产中，确保用户拥有super_admin角色
```

### 问题3: 修改或删除无效

```
症状: 修改或删除客户后，页面显示成功，但刷新后恢复原状

排查步骤:
1. ❌ 检查API响应状态码（应为200）
2. ❌ 检查API响应内容中的data字段
3. ❌ 检查数据库中的数据是否真正被修改
4. ❌ 查看浏览器Network标签中的请求和响应

解决方案:
✅ 检查MySQL连接是否正常
✅ 查看后端服务日志中是否有SQL执行错误
✅ 确认有修改数据库的权限
✅ 尝试直接在MySQL中执行UPDATE语句测试
```

### 问题4: 列表显示为空或数据不准确

```
症状: 加载成功但列表为空，或显示与数据库不符的数据

排查步骤:
1. ❌ 检查数据库中是否有数据
2. ❌ 验证API返回的JSON格式是否正确
3. ❌ 检查前端数据映射逻辑是否正确
4. ❌ 查看浏览器Console中的映射日志

解决方案:
✅ 执行SQL: SELECT COUNT(*) FROM sys_tenant;
✅ 验证返回的data结构中有tenants字段或是数组
✅ 检查字段名映射: tenantId->id, tenantName->name等
✅ 使用浏览器开发者工具Network标签查看API响应
```
