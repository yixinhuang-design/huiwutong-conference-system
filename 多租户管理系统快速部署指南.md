# 多租户管理系统 - 快速部署指南

## 🎯 5 分钟快速开始

### 前置条件检查
```bash
# 检查 Java 版本
java -version
# 应输出: java version "21" 或更高

# 检查 Maven
mvn -version
# 应输出: Apache Maven 3.6.0 或更高

# 检查 MySQL
mysql -u root -p -e "SELECT VERSION();"
# 应输出: MySQL 9.6.x
```

---

## 🚀 步骤 1: 启动基础服务

### 1.1 启动 MySQL（如果未启动）
```bash
# Windows
net start MySQL80  # 或对应版本

# Linux
sudo systemctl start mysql

# 验证连接
mysql -h localhost -P 3308 -u root -p
# 密码: Hnhx@123
```

### 1.2 启动 Redis（可选，用于缓存）
```bash
# Windows
redis-server.exe

# Linux
redis-server
```

### 1.3 启动 Nacos（服务注册）
```bash
# 进入 Nacos 安装目录
cd nacos/bin

# Windows
startup.cmd

# Linux/Mac
./startup.sh

# 访问管理界面
http://localhost:8848/nacos
# 用户名: nacos
# 密码: nacos
```

---

## 🔨 步骤 2: 编译后端代码

```bash
# 进入后端目录
cd backend/conference-backend

# 清理并编译（跳过单元测试）
mvn clean package -DskipTests

# 预期输出
# [INFO] BUILD SUCCESS
```

**编译后的 JAR 文件位置：**
```

conference-seating/target/conference-seating-1.0.0.jar
conference-notification/target/conference-notification-1.0.0.jar
[其他微服务...]
```

---

## 🏃 步骤 3: 启动微服务

### 3.1 启动 API 网关（首先启动，端口 8080）
```bash
cd conference-gateway/target
java -jar conference-gateway-1.0.0.jar

# 预期输出:
# Started GatewayApplication in 5.234 seconds
```

### 3.2 启动认证服务（端口 8081，包含租户管理 API）
```bash
cd conference-auth/target
java -jar conference-auth-1.0.0.jar

# 预期输出:
# Started AuthApplication in 4.567 seconds
# Flyway 会自动初始化数据库
```

### 3.3 启动其他微服务（按需启动）
```bash
# 新的终端窗口
cd conference-seating/target
java -jar conference-seating-1.0.0.jar

# 重复以上步骤启动其他服务...
```

**服务端口映射：**
| 服务 | 端口 | JAR 文件 |
|------|------|---------|
| 网关 | 8080 | conference-gateway-1.0.0.jar |
| 认证/租户管理 | 8081 | conference-auth-1.0.0.jar |
| 排座 | 8082 | conference-seating-1.0.0.jar |
| 通知 | 8083 | conference-notification-1.0.0.jar |
| 协作 | 8084 | conference-collaboration-1.0.0.jar |
| AI | 8085 | conference-ai-1.0.0.jar |
| 导航 | 8086 | conference-navigation-1.0.0.jar |
| 数据 | 8087 | conference-data-1.0.0.jar |

## 🌐 步骤 4: 访问管理系统

### 4.1 获取认证令牌
```bash
# 使用 curl 登录
curl -X POST http://localhost:8081/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "admin123"
  }'

# 或使用 Postman：
# POST http://localhost:8081/api/auth/login
# Body (JSON):
# {
#   "username": "admin",
#   "password": "admin123"
# }

# 预期响应:
# {
#   "code": 200,
#   "data": {
#     "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
#   }
# }
```

### 4.2 在浏览器中保存 Token
```javascript
// 在浏览器控制台 (F12) 中执行：
localStorage.setItem('auth_token', 'YOUR_TOKEN_HERE');
localStorage.setItem('current_tenant_id', '1');

// 验证保存成功
localStorage.getItem('auth_token');
```

### 4.3 访问管理后台
```
http://localhost:8080/pages/customer-management.html
```

**页面功能：**
- 📊 租户统计卡片
- 🔍 搜索和筛选
- ➕ 新增客户
- ✏️ 编辑租户
- 🗑️ 删除租户
- 📈 查看统计信息

## 🔍 故障排查

### 问题 1: Port 8081 已被占用
```bash
# 查找占用该端口的进程
netstat -ano | findstr :8081

# Windows: taskkill /PID <PID> /F
# Linux: kill -9 <PID>

# 或指定其他端口
java -jar conference-auth-1.0.0.jar --server.port=8081
```

### 问题 2: Flyway 数据库初始化失败
```bash
# 检查 MySQL 连接
mysql -h localhost -P 3308 -u root -p -e "SHOW DATABASES;"

# 检查 application.yml 中的数据库配置
# spring.datasource.url: jdbc:mysql://localhost:3308/conference_db
# spring.datasource.username: root
# spring.datasource.password: Hnhx@123

# 手动创建数据库
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS conference_db CHARACTER SET utf8mb4;"
```

### 问题 3: CORS 跨域错误
```javascript
// 浏览器控制台会显示：
// Access to XMLHttpRequest at 'http://localhost:8081/api/tenant/list'
// from origin 'http://localhost:3000' has been blocked by CORS policy

// 解决: 在后端 TenantManagementController 中添加：
// @CrossOrigin(origins = "http://localhost:3000")
```

### 问题 4: 401 认证失败
```bash
# 确保 Token 正确传递
# 在请求头中: Authorization: Bearer <TOKEN>

# 检查 Token 是否过期
# Token 默认有效期 24 小时

# 获取新 Token
curl -X POST http://localhost:8081/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "admin123"
  }'
```

### 问题 5: 数据库字符集问题
```sql
-- 检查数据库字符集
SELECT * FROM information_schema.SCHEMATA WHERE SCHEMA_NAME = 'conference_db';

-- 修改数据库字符集
ALTER DATABASE conference_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 检查表的字符集
SHOW CREATE TABLE sys_tenant;
```
