## 1. P2优先级功能清单

根据管理后台界面分析，P2优先级包含以下功能模块：

### 1.1 日程管理（Schedule Management）

**功能描述**: 管理会议的日程安排，包括会议日程、时间表等

**计划的API端点**:
```
1. GET    /api/seating/schedules/{conferenceId}      - 获取会议日程列表
2. GET    /api/seating/schedules/{scheduleId}        - 获取日程详情
3. POST   /api/seating/schedules                      - 创建日程
4. PUT    /api/seating/schedules/{scheduleId}        - 更新日程
5. DELETE /api/seating/schedules/{scheduleId}        - 删除日程
```

**关键字段** (参考数据库表):
- schedule_id: 日程ID
- conference_id: 会议ID
- schedule_date: 日程日期
- start_time: 开始时间
- end_time: 结束时间
- schedule_name: 日程名称
- description: 日程描述
- status: 状态（DRAFT, PUBLISHED, COMPLETED）

**业务规则**:
- ✓ 日程日期不能在会议日期之前
- ✓ 结束时间不能早于开始时间
- ✓ 同一时间段内日程不能重叠
- ✓ 删除日程前检查是否有参会人员已预订

---

### 1.2 住宿安排（Accommodation Management）

**功能描述**: 为参会人员分配住宿房间

**计划的API端点**:
```
1. GET    /api/seating/accommodations/{conferenceId} - 获取住宿列表
2. GET    /api/seating/accommodations/{accomId}     - 获取住宿详情
3. POST   /api/seating/accommodations                - 创建住宿安排
4. PUT    /api/seating/accommodations/{accomId}     - 更新住宿
5. DELETE /api/seating/accommodations/{accomId}     - 删除住宿
```

**关键字段**:
- accommodation_id: 住宿ID
- conference_id: 会议ID
- room_number: 房间号
- room_type: 房间类型（SINGLE, DOUBLE, SUITE）
- capacity: 房间容量
- assigned_count: 已分配人数
- assigned_attendees: 分配的参会人员列表

**业务规则**:
- ✓ 房间已分配人数不能超过房间容量
- ✓ 同一房间不能分配相冲突的参会人员
- ✓ VIP人员优先分配优质房间

## 4. 数据库表参考

### 4.1 conf_seating_schedule 表结构

```sql
CREATE TABLE conf_seating_schedule (
    id BIGINT PRIMARY KEY,
    conference_id BIGINT NOT NULL,
    tenant_id BIGINT NOT NULL,
    schedule_name VARCHAR(100) NOT NULL,
    schedule_date DATE NOT NULL,
    start_time TIME NOT NULL,
    end_time TIME NOT NULL,
    description VARCHAR(500),
    status VARCHAR(20) DEFAULT 'DRAFT',
    created_by BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_by BIGINT,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (conference_id) REFERENCES conference(id),
    FOREIGN KEY (tenant_id) REFERENCES tenant(id)
);
```
