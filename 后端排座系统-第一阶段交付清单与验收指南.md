## 八、常见问题与解决方案

### Q1：编译时提示"找不到com.conference.common"

**解决方案**：
1. 确保conference-common项目已编译并安装到本地Maven仓库
2. 运行命令：`mvn install -DskipTests` 在conference-common目录下
3. 重新编译conference-seating

### Q2：数据库连接失败

**解决方案**：
1. 检查MySQL服务是否启动：`mysql -u root -p`
2. 验证application.yml中的连接信息（url、username、password）
3. 确保conference_seating数据库已创建

### Q3：@MapperScan找不到Mapper

**解决方案**：
1. 确保Mapper接口都添加了@Mapper注解
2. 验证@MapperScan的路径正确：`"com.conference.seating.mapper"`
3. 检查SeatingApplication.java是否正确配置

### Q4：智能分配算法的结果不符合预期

**解决方案**：
1. 检查传入的参数（attendees和seats列表）是否正确
2. 验证算法是否选择正确
3. 编写单元测试验证算法逻辑
4. 可以选择其他算法进行对比
