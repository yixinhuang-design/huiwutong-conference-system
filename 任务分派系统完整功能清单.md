# 📋 任务分派系统完整功能清单

## 系统概述

**页面位置**: `pages/task-management.html`  
**后端服务**: `conference-collaboration` (端口8089)  
**数据库**: `conference_collaboration`  
**主要功能**: 任务创建、分配、追踪、催办、统计

---

## 一、核心功能模块

### 1.1 任务创建功能 ✅

#### 1.1.1 基本信息
- **任务名称**: 文本输入，必填
- **任务描述**: 多行文本输入，选填
- **任务类型**: 下拉选择
  - `preparation` - 筹备任务
  - `venue_check` - 会场检查
  - `reception` - 学员接待
  - `registration` - 报到任务
  - `checkin` - 签到任务
  - `room_check` - 查寝任务
  - `evaluation` - 评价任务
  - `collection` - 征集任务
  - `custom` - 自定义任务

#### 1.1.2 任务分类
- **分类选择**: 
  - `venue` - 场馆相关
  - `student` - 学员相关
  - `custom` - 自定义

#### 1.1.3 优先级设置
- **优先级选项**:
  - `low` - 低优先级 🟢
  - `medium` - 中优先级 🟡 (默认)
  - `high` - 高优先级 🟠
  - `urgent` - 紧急 🔴

#### 1.1.4 截止时间
- **日期选择器**: 日期时间选择
- **时区支持**: 本地时区

---

### 1.2 完成方式配置 ✅

#### 1.2.1 文本+图片验证 (`text_image`)
**配置项**:
- `requireText`: 是否要求文字说明
- `minTextLength`: 最少文字长度
- `requireImage`: 是否要求图片
- `minImageCount`: 最少图片数量

**用途**: 现场检查、场地验收等需要图文报告的任务

#### 1.2.2 位置签到 (`location`)
**配置项**:
- `locationName`: 签到地点名称
- `locationAddress`: 详细地址
- `longitude`: 经度 (高德地图API)
- `latitude`: 纬度 (高德地图API)
- `locationRadius`: 签到半径(米)

**地图集成**:
- 高德地图API集成
- 地点搜索
- 地图标记
- 半径设置

**用途**: 会场签到、指定位置打卡

#### 1.2.3 问卷调查 (`questionnaire`)
**配置项**:
- `questions`: 问题列表数组
  - `question`: 问题内容
  - `type`: 问题类型
    - `rating` - 评分题 (1-5星)
    - `text` - 文本题
    - `choice` - 单选题
    - `multiple` - 多选题
  - `options`: 选项列表 (单选/多选题)
  - `required`: 是否必填

**用途**: 满意度调查、反馈收集

#### 1.2.4 数据征集 (`collection`)
**配置项**:
- `fields`: 字段定义数组
  - `label`: 字段标签
  - `type`: 字段类型
    - `text` - 文本
    - `number` - 数字
    - `date` - 日期
    - `file` - 文件上传
  - `required`: 是否必填
  - `options`: 选项值 (如果有)

**用途**: 信息收集、报名数据补充

---

### 1.3 执行人管理 ✅

#### 1.3.1 执行人分配
- **主执行人**: 
  - 单选，标记为 `isMain=1`
  - 显示特殊标识 👑
  
- **协办人**: 
  - 多选，标记为 `isMain=0`
  - 可添加多个

#### 1.3.2 分配对象
- **工作人员列表**: 从 `conference_registration` 加载
- **学员列表**: 从 `conference_registration` 加载
- **分组分配**: 支持按组分配
  - 选择目标分组 `targetGroups`
  - 自动分配该组所有成员

#### 1.3.3 执行人角色
- `executor` - 执行人 (负责完成任务)
- `assistant` - 协办人 (协助执行)

- 优先级

### 1.7 任务操作功能 ✅

#### 1.7.1 完成任务
- **操作入口**: 任务卡片、详情页
- **操作对象**: 任务创建者、管理员
- **确认机制**: 弹窗确认
- **备注**: 可填写完成备注

#### 1.7.2 取消任务
- **权限**: 仅创建者或管理员
- **原因**: 必须填写取消原因
- **影响**: 
  - 状态变更为 `cancelled`
  - 执行人收到通知
  - 记录操作日志

#### 1.7.3 删除任务
- **权限**: 仅创建者或管理员
- **限制**:
  - 已取消的任务可删除
  - 进行中的任务不可删除
- **操作**: 软删除(`deleted=1`)

#### 1.7.4 催办功能
- **触发条件**: 任务即将逾期或已逾期
- **催办方式**:
  - 发送通知
  - 记录催办日志
- **频率**: 限制重复催办

---

### 1.8 批量操作 ✅

#### 1.8.1 批量提醒
- **选择方式**: 
  - 按状态筛选

- **提醒范围**:
  - 待处理任务

### 1.9 统计功能 ✅

#### 1.9.1 任务统计卡片
- **总任务数**: 所有未取消的任务
- **已完成**: 状态为 `completed` 的任务
- **进行中**: 状态为 `in_progress` 的任务
- **待处理**: 状态为 `pending` 的任务
- **已逾期**: 状态为 `overdue` 的任务
- **完成率**: 完成任务/总任务 × 100%

#### 1.9.2 统计详情
- **任务分类统计**: 各类型任务数量
- **完成方式统计**: 各完成方式任务数量
- **执行情况统计**: 
  - 各执行人任务数
  - 完成率排名

#### 1.9.3 可视化
- **饼图**: 任务类型分布
- **柱状图**: 完成率统计
- **进度条**: 整体进度

## 二、数据关联关系

### 2.1 与会议管理模块的关联 🔗

**关联表**: `conference_registration.conf_meeting`

```
task_info.conference_id → conf_meeting.id
```

**关联说明**:
- 任务属于某个会议
- 切换会议时加载对应会议的任务
- 会议删除时级联删除任务

**数据流向**:
```
会议管理 → 创建会议 → 任务分派 → 为会议创建任务
```

---

### 2.2 与报名管理模块的关联 🔗

**关联表**: 
- `conference_registration.conf_meeting_staff` (工作人员)
- `conference_registration.conf_registration` (报名学员)

**关联字段**:
```
task_assignee.user_id → conf_meeting_staff.user_id
task_assignee.user_id → conf_registration.user_id
task_info.target_groups → conf_group.group_id
```

**关联说明**:
1. **执行人来源**:
   - 工作人员列表从 `conf_meeting_staff` 加载
   - 学员列表从 `conf_registration` 加载

2. **分组分配**:
   - 目标分组关联 `conf_group`
   - 选择分组后自动添加该组成员

**数据流向**:
```
报名管理 → 添加工作人员/学员 → 任务分派 → 分配任务执行人
```

---

### 2.3 与通知模块的关联 🔗

**关联表**: `conference_notification.conf_notification`

**触发通知的操作**:
1. **任务分配**: 
   - 接收对象: 执行人
   - 内容: 您被分配了新任务

2. **任务更新**:
   - 接收对象: 相关执行人
   - 内容: 任务信息已更新

3. **任务催办**:
   - 接收对象: 未完成的执行人
   - 内容: 请尽快完成任务

4. **任务完成**:
   - 接收对象: 任务创建者
   - 内容: 任务已完成

**通知方式**:
- 系统内通知
- 可扩展邮件/短信

---

### 2.4 与用户认证模块的关联 🔗

**关联表**: `conference_auth.sys_user`, `conference_auth.sys_tenant`

**关联字段**:
```
task_info.tenant_id → sys_tenant.id
task_info.creator_id → sys_user.id
task_assignee.user_id → sys_user.id
```

**关联说明**:
1. **多租户隔离**:
   - 每个租户只能看到自己的任务
   - 数据按租户隔离

2. **用户权限**:
   - `creator_id`: 任务创建者
   - 创建者可编辑、删除、完成任务

3. **操作日志**:
   - 所有操作记录用户ID
   - 追溯操作人

**数据权限**:
```
租户隔离 → 只能看到本租户的任务
用户权限 → 只能操作自己创建的任务(管理员除外)
```

---

### 2.5 与排座模块的关联 🔗

**潜在关联**: 场馆相关任务

**关联方式**:
- 场馆检查任务 (`venue_check`) 可关联排座中的场馆信息
- 可通过 `conference_seating.conf_seating_venue` 获取场馆列表

**使用场景**:
- 创建场馆检查任务时选择目标场馆
- 签到地点可设置为场馆入口

---

### 2.6 与日程模块的关联 🔗

**关联表**: `conference_registration.conf_schedule`

**关联字段**:
```
task_info.deadline → conf_schedule.end_time (逻辑关联)
```

**关联说明**:
- 任务截止时间可参考日程时间
- 日程相关的任务(如签到任务)自动关联日程

**自动创建任务**:
```
日程开始前 → 自动创建签到任务
日程结束前 → 自动创建评价任务
```

## 三、数据库表结构

### 3.1 task_info (任务信息表)
```sql
- id: 任务ID (主键)
- tenant_id: 租户ID
- conference_id: 会议ID
- task_name: 任务名称
- task_type: 任务类型
- category: 任务分类
- completion_method: 完成方式
- description: 任务描述
- priority: 优先级 (low/medium/high/urgent)
- status: 状态 (pending/in_progress/completed/overdue/cancelled)
- deadline: 截止时间
- progress: 进度百分比
- creator_id: 创建者ID
- owner_name: 负责人名称
- target_groups: 目标分组 (JSON)
- config: 配置信息 (JSON)
- attachments: 附件信息 (JSON)
- create_time: 创建时间
- update_time: 更新时间
- deleted: 逻辑删除标记
```

### 3.2 task_assignee (任务执行人表)
```sql
- id: 执行人ID (主键)
- task_id: 任务ID (外键)
- user_id: 用户ID (外键)
- user_name: 用户名称
- role: 角色 (executor/assistant)
- status: 状态 (pending/in_progress/completed)
- is_main: 是否主执行人 (0/1)
- remark: 备注
- submit_content: 提交内容 (JSON)
- submit_images: 提交图片 (JSON)
- submit_location: 提交位置信息
- submit_time: 提交时间
- create_time: 创建时间
```

### 3.3 task_log (任务日志表)
```sql
- id: 日志ID (主键)
- task_id: 任务ID (外键)
- operator_id: 操作人ID
- operator_name: 操作人名称
- action: 操作类型 (create/update/assign/submit/complete/cancel/urge)
- content: 操作内容
- remark: 备注
- create_time: 创建时间
```

## 四、API接口清单

### 4.1 任务CRUD接口
| 方法 | 路径 | 功能 | 权限 |
|------|------|------|------|
| POST | `/api/task/create` | 创建任务 | 登录用户 |
| DELETE | `/api/task/{taskId}` | 删除任务 | 创建者/管理员 |
| PUT | `/api/task/{taskId}` | 更新任务 | 创建者/管理员 |
| GET | `/api/task/{taskId}` | 获取任务详情 | 登录用户 |

### 4.2 任务查询接口
| 方法 | 路径 | 功能 | 参数 |
|------|------|------|------|
| GET | `/api/task/list` | 任务列表(分页) | conferenceId, status, category, page, size |
| GET | `/api/task/my-tasks` | 我的任务 | userId, status, page, size |
| GET | `/api/task/stats` | 任务统计 | conferenceId |

### 4.3 执行人管理接口
| 方法 | 路径 | 功能 | 参数 |
|------|------|------|------|
| POST | `/api/task/{taskId}/assign` | 分配执行人 | assignees[] |
| GET | `/api/task/{taskId}/assignees` | 获取执行人列表 | - |

### 4.4 任务执行接口
| 方法 | 路径 | 功能 | 参数 |
|------|------|------|------|
| POST | `/api/task/{taskId}/submit` | 提交任务反馈 | userId, content, images, location |
| POST | `/api/task/{taskId}/complete` | 完成任务 | operatorId, operatorName, remark |
| POST | `/api/task/{taskId}/cancel` | 取消任务 | operatorId, operatorName, remark |
| POST | `/api/task/{taskId}/urge` | 催办任务 | operatorId, operatorName |

### 4.5 日志接口
| 方法 | 路径 | 功能 | 参数 |
|------|------|------|------|
| GET | `/api/task/{taskId}/logs` | 获取任务日志 | - |

## 五、前端交互流程

### 5.1 任务创建流程
```
用户操作流程:
1. 点击"创建任务"按钮
2. 填写任务基本信息
3. 选择完成方式
4. 配置完成方式详情
5. 选择执行人
6. 选择目标分组(可选)
7. 设置截止时间和优先级
8. 保存任务

系统处理:
- 表单验证
- 调用 createTask API
- 分配执行人
- 发送通知
- 刷新列表
```

### 5.2 任务执行流程
```
执行人操作流程:
1. 查看任务列表
2. 打开任务详情
3. 根据完成方式提交:
   a. 文字+图片: 填写说明+上传图片
   b. 位置签到: 地图定位+签到
   c. 问卷调查: 填写问卷
   d. 数据征集: 填写表单
4. 提交反馈

系统处理:
- 验证提交内容
- 更新执行人状态
- 记录提交日志
- 检查是否所有人都完成
- 自动更新任务状态
```

### 5.3 任务催办流程
```
催办触发:
1. 任务即将逾期(24小时内)
2. 任务已逾期
3. 管理员手动催办

催办执行:
- 发送通知给未完成执行人
- 记录催办日志
- 更新催办次数
```
