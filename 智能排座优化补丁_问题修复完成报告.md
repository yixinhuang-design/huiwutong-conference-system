## 📋 问题诊断

### 用户反馈
浏览器控制台日志显示：

**表现**：虽然日程"加载"了，但参会人员数据为空，系统使用默认测试数据。

### 问题链路分析

#### 问题1：会议日程未从API正确加载
**位置**: [seating-schedule-manager.js](seating-schedule-manager.js#L65-L108)  
**原因**: `loadSchedules()`函数使用硬编码的模拟数据（第65-108行），没有调用后端日程API  
**现状**：
- 硬编码3个模拟日程（sched_001, sched_002, sched_003）
- 虽然显示加载成功，但实际是模拟数据
- 没有与真实会议数据同步

#### 问题2：座位和参会人员数据为空
**位置**: [seating-mgr.html](seating-mgr.html#L5215)  
**根本原因**: 后端API `/api/seating/layout` 有**两个定义**，第二个是占位符版本

**后端API重复定义**：
| 位置 | API定义 | 实现 | 问题 |
|------|--------|------|------|
| [SeatingController.java:217](SeatingController.java#L217) | `@GetMapping("/layout")` | 完整实现 `buildVenueLayout()` | ❌ 被覆盖 |
| [SeatingController.java:512](SeatingController.java#L512) | `@GetMapping("/layout")` | 占位符 `TODO: 实现座位平面图查询逻辑` | ✅ 实际使用（后定义） |

**结果**：前端调用 `/api/seating/layout?venueId={venueId}` 得到占位符响应，座位数据为空。

**修改后** - 调用真实API + 降级方案：
```javascript
const loadSchedules = async () => {
    try {
        // 从后端API加载日程数据
        if (context.conference && context.conference.id) {
            const response = await fetch(
                `/api/schedule/all?meetingId=${context.conference.id}`,
                { 
                    method: 'GET',
                    headers: { 'Content-Type': 'application/json' }
                }
            );
            
            if (response.ok) {
                const data = await response.json();
                const schedulesData = data.data || data || [];
                
                // 转换API响应为前端格式
                schedulesList.value = Array.isArray(schedulesData) ? schedulesData.map(schedule => ({
                    id: schedule.id,
                    name: schedule.title || schedule.name,
                    date: schedule.scheduleDate || schedule.date,
                    timeStart: schedule.startTime || schedule.timeStart,
                    timeEnd: schedule.endTime || schedule.timeEnd,
                    venueId: schedule.venueId || 'venue_a',
                    venueName: schedule.venueName || schedule.location || '会场A',
                    capacity: schedule.capacity || 500,
                    attendeeCount: schedule.participantCount || 0,
                    assignedCount: 0,
                    status: schedule.status || 'draft'
                })) : [];
                
                console.log(`✅ 从API加载日程成功，共${schedulesList.value.length}个日程`);
            } else {
                console.warn('从API加载日程失败，使用本地模拟数据');
                loadSchedulesFromLocal();
            }
        } else {
            console.warn('会议信息不完整，使用本地模拟数据');
            loadSchedulesFromLocal();
        }

        // 如果没有选中日程，默认选中第一个
        if (!currentScheduleId.value && schedulesList.value.length > 0) {
            const firstSchedule = schedulesList.value[0];
            currentScheduleId.value = firstSchedule.id;
            Object.assign(currentSchedule, firstSchedule);
            await loadScheduleSeatingData(firstSchedule.id);
            console.log(`✅ 页面加载时，已自动加载日程：${firstSchedule.name}`);
        }
    } catch (error) {
        console.error('❌ 加载日程列表失败:', error);
        loadSchedulesFromLocal();
    }
};
```

#### 新增函数：`loadSchedulesFromLocal()` (第122-159行)

**功能**：降级方案，当API不可用时使用本地模拟数据
```javascript
const loadSchedulesFromLocal = () => {
    schedulesList.value = [
        { id: 'sched_001', name: '开幕式', ... },
        { id: 'sched_002', name: '主题演讲', ... },
        { id: 'sched_003', name: '闭幕式', ... }
    ];
};
```

**API调用链路**：
```

  ↓

  ├─ ✅ 成功 → 加载真实日程数据
  └─ ❌ 失败 → loadSchedulesFromLocal() → 使用模拟数据
```

**影响**：
- ✅ 日程数据与后端同步
- ✅ 当API可用时使用真实数据
- ✅ 当API不可用时自动降级
- ✅ 对现有功能无负面影响
