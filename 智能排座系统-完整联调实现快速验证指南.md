## 一、后端API验证 (2分钟)

### 1.1 启动后端服务
```bash
# 进入项目目录
cd backend/conference-backend

# 启动服务（使用Maven）
mvn spring-boot:run -pl conference-seating

# 或使用编译好的JAR
java -jar conference-seating-1.0.0.jar
```

### 1.2 验证编译
```bash
# 检查是否有编译错误
mvn clean compile -pl conference-seating

# 预期结果: BUILD SUCCESS
```

### 1.3 API接口测试

**使用Postman或curl验证以下6个新接口:**

#### API 1: 获取座位列表
```bash
curl -X GET "http://localhost:8080/api/seating/seats/1" \
  -H "Content-Type: application/json" \
  -H "tenant-id: 1"
```
**预期响应**: 200 OK，包含座位数组

#### API 2: 获取可用座位
```bash
curl -X GET "http://localhost:8080/api/seating/seats/1/available" \
  -H "Content-Type: application/json" \
  -H "tenant-id: 1"
```
**预期响应**: 200 OK，包含可用座位数组

#### API 3: 获取座位平面图
```bash
curl -X GET "http://localhost:8080/api/seating/layout?venueId=1" \
  -H "Content-Type: application/json" \
  -H "tenant-id: 1"
```
**预期响应**: 200 OK，包含seats和stats对象

#### API 4: 分配座位
```bash
curl -X POST "http://localhost:8080/api/seating/seats/assign" \
  -H "Content-Type: application/json" \
  -H "tenant-id: 1" \
  -d '{
    "seatId": 1,
    "attendeeId": 1001,
    "assignType": "MANUAL"
  }'
```
**预期响应**: 200 OK，data为true或false

#### API 5: 取消分配
```bash
curl -X DELETE "http://localhost:8080/api/seating/seats/1/unassign" \
  -H "Content-Type: application/json" \
  -H "tenant-id: 1"
```
**预期响应**: 200 OK，data为true

#### API 6: 交换座位
```bash
curl -X POST "http://localhost:8080/api/seating/seats/swap" \
  -H "Content-Type: application/json" \
  -H "tenant-id: 1" \
  -d '{
    "seatId1": 1,
    "seatId2": 2
  }'
```
**预期响应**: 200 OK，包含swap结果

## 四、问题排查 (遇到问题时)

### 问题1: 后端API返回404
```
解决方案:
1. 检查URL是否正确: /api/seating/seats/{venueId}
2. 确认venueId存在于数据库
3. 检查租户ID是否在请求头中
```

### 问题2: 前端页面显示"undefined"
```
解决方案:
1. F12检查浏览器控制台错误信息
2. 查看Network标签API响应
3. 确认API返回的字段名是否匹配
```

### 问题3: 座位网格未显示
```
解决方案:
1. 检查#seatGridContainer元素是否存在
2. 查看renderSeats()函数是否被调用
3. 在控制台验证: document.getElementById('seatGridContainer')
```

### 问题4: 座位交换不成功
```
解决方案:
1. 确认两个座位都已被分配
2. 检查API返回是否为true
3. 查看SeatSwapResponse对象内容
```

## 八、快速验证脚本

### 自动化验证脚本 (bash)
```bash
#!/bin/bash

echo "=== 智能排座系统快速验证 ==="
echo ""

# 1. 检查后端编译
echo "1. 检查后端编译..."
mvn clean compile -pl conference-seating -q
if [ $? -eq 0 ]; then
    echo "   ✅ 编译成功"
else
    echo "   ❌ 编译失败"
    exit 1
fi

# 2. 检查API服务
echo "2. 检查API服务..."
response=$(curl -s -o /dev/null -w "%{http_code}" "http://localhost:8080/api/seating/venues/1" \
  -H "tenant-id: 1")
if [ "$response" == "200" ]; then
    echo "   ✅ API服务正常 (HTTP $response)"
else
    echo "   ⚠️  API返回 HTTP $response（可能无数据）"
fi

# 3. 检查前端文件
echo "3. 检查前端文件..."
if grep -q "loadSeatsFromAPI\|fetch.*api/seating" admin-pc/conference-admin-pc/pages/seating-mgr.html; then
    echo "   ✅ seating-mgr.html已修改为API调用"
else
    echo "   ❌ seating-mgr.html未修改"
fi

if grep -q "loadSeatsFromAPI\|fetch.*api/seating" app/staff/seat-manage.html; then
    echo "   ✅ seat-manage.html已修改为动态加载"
else
    echo "   ❌ seat-manage.html未修改"
fi

echo ""
echo "=== 验证完成 ==="
```
