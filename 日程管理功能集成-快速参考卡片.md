## 🔧 快速启动

### 1. 启动后端
```bash
cd conference-meeting
mvn spring-boot:run
# 运行在 localhost:9001
```

### 2. 初始化数据库
```bash
mysql -h localhost -P 3308 -u root -pHnhx@123 conference_registration < V7__schedule_schema.sql
```

### 3. 打开前端
```
Web: http://localhost:8080/admin-pc/pages/schedule-mgr.html?conferenceId=VALID_ID
App: App 内置导航菜单
```

## 📋 关键 API 端点

### 创建/查询
| 方法 | 端点 | 功能 |
|------|------|------|
| GET | `/api/schedule/all` | 获取所有日程 |
| POST | `/api/schedule/create` | 创建日程 |
| GET | `/api/schedule/{id}` | 获取详情 |

### 更新/删除
| 方法 | 端点 | 功能 |
|------|------|------|
| PUT | `/api/schedule/{id}` | 更新日程 |
| DELETE | `/api/schedule/{id}` | 删除日程 |

### 状态操作
| 方法 | 端点 | 功能 |
|------|------|------|
| PUT | `/api/schedule/{id}/publish` | 发布日程（0→1） |
| PUT | `/api/schedule/{id}/cancel` | 取消日程（→4） |
| POST | `/api/schedule/{id}/duplicate` | 复制日程 |

### 查询统计
| 方法 | 端点 | 功能 |
|------|------|------|
| GET | `/api/schedule/list` | 分页列表 |
| GET | `/api/schedule/need-checkin` | 需签到 |
| GET | `/api/schedule/need-reminder` | 需提醒 |
| GET | `/api/schedule/ongoing` | 进行中 |
| GET | `/api/schedule/next` | 下一个 |
| GET | `/api/schedule/current` | 当前进行 |
| GET | `/api/schedule/count` | 计数 |

## 📊 日程状态流转

```
┌─────────┐
│ 草稿(0) │ ← 初始状态
└────┬────┘
     │ 发布
     ↓
┌─────────────┐
│ 已发布(1)   │ ← 可以被用户看到
└────┬────────┘
     │ 自动/手动转换
     ↓
┌──────────────┐
│ 进行中(2)    │ ← 可以签到
└────┬─────────┘
     │ 自动完成
     ↓
┌──────────────┐
│ 已完成(3)    │ ← 存档
└──────────────┘

任何状态 ──取消→ 已取消(4) [终态]
```

## 📞 技术支持快速链接

### 错误排查
1. 检查浏览器 Console（F12）
2. 检查 Network 标签中的 API 请求
3. 查看后端日志（spring-boot:run 终端）
4. 参考"常见问题速查"表

### 代码修改
1. 修改后端：重新编译 mvn clean compile
2. 修改前端：刷新浏览器 Ctrl+Shift+R
3. 修改数据库：需要重新迁移

### 性能优化
1. 启用浏览器缓存
2. 添加数据库索引
3. 实现前端分页加载
4. 考虑使用 Redis 缓存
