## 关键说明

### 什么是"Grid占用大小"？

在CSS Grid布局中，每个座位区需要占用一定的"网格单元"。这里的"Grid列数"和"Grid行数"就是指该座位区在整个Grid中占用多少列和多少行。

**例子**：
- 舞台1行13列 → Grid列数=13, Grid行数=1
- 嘉宾区10行15列 → Grid列数=15, Grid行数=10

### 如果我不配置Grid占用大小会怎样？

系统会自动为您计算一个合理的值。但这可能与您的图片效果不完全一致，所以**建议总是配置**。

### Grid列数/行数的范围

- Grid列数：1-30
- Grid行数：1-20

超出范围的值将被自动调整或拒绝。

### 为什么之前会显示"NaN"？

这是系统计算错误导致的。此修复版本已经完全解决了这个问题，现在：
- ✅ 自动计算更加准确
- ✅ 用户可以精确指定Grid占用大小
- ✅ 系统有完善的错误处理机制

## 常见问题

### Q: 我已经有一些座位区，需要重新配置吗？

**A**: 不需要。现有座位区会继续使用自动计算模式。只有新增或编辑时，才能配置Grid占用大小。

### Q: Grid占用大小应该与座位的物理行列一致吗？

**A**: 通常是的。如果您有10行15列的座位布局，就应该设置Grid列数=15, Grid行数=10。这样布局最自然。

但也可以不同，例如：
- 座位区有10行15列，但设置Grid为5列5行（会压缩显示）
- 座位区有10行15列，但设置Grid为20列15行（会拉伸显示）

根据您的视觉需求调整即可。

### Q: 三个区域怎样排成并列（舞台在上，两个嘉宾区在下）？

**A**: 系统使用**流式算法**自动排列：
1. 先放置第一个座位区（舞台）
2. 如果当前行放不下下一个座位区，自动换行
3. 再放置第二、第三个座位区

对于您的三区域布局：
- 舞台：1行13列 → 占用第1行
- 嘉宾区1：10行15列 → 第1行已满，换到第2行，从左边开始
- 嘉宾区2：10行30列 → 与嘉宾区1同行，在右边

这正是自动流式算法的作用！

### Q: Grid容器的总列数是多少？

**A**: 默认是20列。如果您的座位区总宽度超过20列，系统会自动扩展。

对于您的三区域布局，总宽度需要：13 + 15 + 30 = 58列。系统会自动扩展到足够大。

### Q: 响应式会不会改变布局？

**A**: 不会。Grid占用大小由您的gridConfig配置决定，不随屏幕宽度变化。

但**间距(gap)**会根据屏幕宽度自动调整（响应式设计）。

### Q: 如何恢复到流式布局？

**A**: 在顶部选择"流式布局"模式，所有座位区会自动切换到流式显示（左上开始，从上到下、从左到右排列）。

## 调试技巧

### 查看Console日志

打开浏览器开发者工具（F12），在Console标签中您会看到类似的日志：

```
✅ 座位区"舞台" → Grid: C1, R1, 占用13×1
✅ 座位区"嘉宾区1" → Grid: C14, R2, 占用15×10
✅ 座位区"嘉宾区2" → Grid: C29, R2, 占用30×10
```

这表示：
- 舞台：列1开始，行1开始，占用13列1行
- 嘉宾区1：列14开始（13+1间距），行2开始，占用15列10行
- 嘉宾区2：列29开始（14+15），行2开始，占用30列10行

✅ 如果显示的是正常的数字（不是NaN），说明配置成功！

### 检查gridConfig是否保存

在Console执行：
```javascript
seatingAreas.value.forEach(a => 
    console.log(a.name, a.gridConfig)
);
```

输出示例：
```
舞台 {gridCols: 13, gridRows: 1}
嘉宾区1 {gridCols: 15, gridRows: 10}
嘉宾区2 {gridCols: 30, gridRows: 10}
```

✅ 如果显示的是您设置的值，说明保存成功！

## 视频演示（文字版）

### 完整操作流程

1. **打开座位管理页面**
   - 看到左侧座位区列表（可能为空或有现有区域）

2. **点击"添加座位区"**
   - 弹出配置弹窗

3. **在弹窗中填写信息**
   - 基本信息区：名称、行、列

4. **在Grid配置中输入**
   - Grid列数输入框输入数字
   - Grid行数输入框输入数字

5. **点击保存**
   - 座位区添加到左侧列表

6. **重复Step 2-5添加其他座位区**

7. **顶部选择"Grid多块布局"**
   - 中心区域展示Grid布局效果
   - 可看到三个座位区并列排列

✅ 完成！
