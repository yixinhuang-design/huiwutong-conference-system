## 🚀 立即开始使用

### 后端配置（1步骤）
编辑 `application.properties`：
```properties
file.upload.max-file-size=104857600
file.upload.chunk-size=5242880
file.upload.enable-image-compress=true
file.upload.image-quality=85
```

### 编译部署（2步骤）
```bash
# 1. 编译
mvn clean package -DskipTests

# 2. 运行
java -jar target/conference-meeting-*.jar
```

### 前端使用（简单代码）
```javascript
// HTML
<input type="file" id="fileInput" />

// JavaScript
const file = document.getElementById('fileInput').files[0];
const manager = new ChunkUploadManager(file, scheduleId, {
    onProgress: (progress) => {
        console.log(progress.progress + '%');
    },
    onComplete: (attachment) => {
        alert('上传完成：' + attachment.fileName);
    }
});

await manager.start();
```

## 🔒 安全性保证

### 1. 数据隔离
- 每个租户的上传文件存储在独立目录
- 查询和删除前验证tenantId
- 防止跨租户数据访问

### 2. 文件验证
- SHA256哈希校验完整性
- 白名单限制文件类型（13种）
- 文件大小限制（100MB）
- MIME类型检查

### 3. 权限控制
- 验证用户对schedule的访问权限
- 每个操作都带上Authorization header
- 支持多租户环境

### 4. 清理机制
- 临时文件24小时自动清理
- 上传失败时清理已上传分块
- 支持手动取消和清理

## 🐛 故障排查

### 常见问题和解决方案

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 404 Not Found | ChunkUploadController未注册 | 检查编译和部署 |
| "上传任务不存在" | uploadId过期或错误 | 检查24小时限制 |
| 分块上传失败 | 网络中断或文件损坏 | 自动重试或取消重新上传 |
| 合并失败 | 分块哈希不匹配 | 检查网络，查看日志 |
| 临时文件堆积 | 服务异常退出 | 手动删除 uploads/.temp |
| 磁盘满 | 压缩或存储问题 | 清理旧文件或配置压缩 |

### 日志查看
```bash
# 查看上传相关日志
tail -f logs/application.log | grep -i "chunk\|upload\|compress"

# 查看错误
grep ERROR logs/application.log | grep -i upload
```
