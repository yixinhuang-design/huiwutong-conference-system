# 日程管理数据加载失败诊断与解决方案

## 问题分析

### 现象
- 前端加载日程时返回空列表（Total: 0）
- 后端日志显示：`<== Total: 0`
- TenantFilter 工作正常，Token 提取成功，租户 ID 设置正确
- 数据库查询执行正确，但没有返回任何行

### 根本原因
**数据库的 `conf_schedule` 表中没有任何日程数据**。迁移脚本 V7 只创建了表结构，没有插入初始化数据。

### 数据流链路验证
```
✅ 前端发起请求
   GET /api/schedule/all?meetingId=2030309010523144194
   Authorization: Bearer eyJhbGc...

↓

✅ TenantFilter 处理
   - Authorization Header 被识别
   - Token 成功提取
   - Token 验证通过
   - tenantId 设置到 Context: 2027317834622709762
   - userId 设置到 Context: 8

↓

✅ Controller 接收请求
   - meetingId 参数验证通过
   - 调用 scheduleService.listAllSchedules(meetingId)

↓

✅ Service 执行逻辑
   - 获取 tenantId: 2027317834622709762
   - 调用 mapper 查询

↓

✅ MyBatis 生成 SQL
   SELECT * FROM conf_schedule 
   WHERE meeting_id = 2030309010523144194 
   AND tenant_id = 2027317834622709762 
   AND deleted = 0 
   ORDER BY start_time ASC

↓

✅ 数据库执行查询
   <== Total: 0  ← 没有符合条件的记录！

↓

✅ 返回结果给前端
   {
     "code": 0,
     "success": true,
     "data": []  ← 空数组
   }

↓

❌ 前端处理
   this.schedules = []  （空数组）
   没有触发 catch 块，不会 fallback 到模拟数据
```

### 方案 2：通过浏览器 DevTools 创建日程（备选）

如果想通过 API 创建日程而不是直接修改数据库：

1. 打开浏览器的开发者工具（F12）
2. 切换到 Console 标签
3. 执行以下 JavaScript 代码：

```javascript
// 创建日程
const createSchedule = async (title, startTime, endTime) => {
  const response = await fetch('http://localhost:8084/api/schedule/create?meetingId=2030309010523144194', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': localStorage.getItem('authToken') || ''
    },
    body: JSON.stringify({
      title: title,
      startTime: startTime,
      endTime: endTime,
      location: '主会场',
      host: '主持人',
      priority: 1
    })
  });
  return response.json();
};

// 创建示例日程
await createSchedule(
  '开幕式',
  new Date().toISOString().split('T')[0] + 'T09:00:00',
  new Date().toISOString().split('T')[0] + 'T10:00:00'
);
```

## 遇到的问题对照表

| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| API 返回 404 | baseURL 错误（指向 9001 而非 8084） | ✅ 已修复：schedule-api.js baseURL 改为 8084 |
| Token 提取失败 | extractTokenFromHeader 不支持直接 Token | ✅ 已修复：增加了直接 Token 支持 |
| 查询结果为空 | 数据库没有初始化数据 | ⬅️ 现在：执行上述 SQL 插入数据 |
| tenant_id = null | TenantFilter 未正确设置 Context | ✅ 已验证工作正常 |
