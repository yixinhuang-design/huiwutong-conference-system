**部署到服务器**:
```bash

# 备份旧 JAR

# 部署新 JAR
cp target/conference-registration-x.x.x.jar /opt/app/conference-registration.jar

# 启动服务
systemctl start conference-registration

# 查看日志
tail -f /var/log/conference-registration.log
```

**验证服务已启动**:

## 📞 部署后支持

### 常见问题处理

**问题1: 部署后报名仍然找不到**
```
解决方案:
1. 检查 localStorage 中的 tenant_id 值
2. 确认后端日志显示了查询条件
3. 执行诊断 SQL 检查数据库
```

**问题2: 后端日志没有新增日志**
```
解决方案:
1. 检查日志级别配置
2. 确认 logging.level.com.conference.registration=INFO
3. 重启服务后重试
```

**问题3: 性能下降**
```
解决方案:
1. 检查日志输出频率是否过高
2. 可以调整日志级别为 DEBUG（生产环境）
3. 启用日志轮转防止文件过大
```

### 监控和告警

- [ ] 后端进程监控（是否在线）
- [ ] 日志监控（ERROR 数量）
