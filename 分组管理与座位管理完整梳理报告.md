## 三、两个模块的关联关系

### 3.1 业务流程关联

```
报名系统
    ↓
    ├─→ 流程08：自动组群管理
    │   ├─→ [分组编排页面] ← 协管员在此进行分组操作
    │   └─→ 创建群组、分配群任务
    │
    └─→ 流程11：智能排座流程
        ├─→ [座位管理页面] ← 协管员在此进行排座操作
        ├─→ 座位调整、生成导示、座位通知
        └─→ 辅助安排（乘车、住宿、讨论）

两个流程的关联点：
- 分组和座位都是根据参会人员数据
- 分组可作为排座时的参考规则（按分组相邻）
- 座位分配后需要通知参会者，涉及群组沟通
```

### 3.2 数据一致性检查

| 项目 | 分组页面 | 座位页面 | 一致性 |
|------|---------|--------|-------|
| **参会人数** | 8人 (2组×3+2待分) | 180座，已分174人 | ⚠️ 不一致 |
| **人员来源** | 同一次报名 | 同一次报名 | ✓ 一致 |
| **数据更新** | 后端实时同步 | 后端实时同步 | ✓ 一致 |

**问题**：分组页面只显示了8人的分组情况，但座位页面显示有174人已分配座位。这两个数据应该对应同一次会议的参会者。

**原因分析**：分组页面是演示性质，只展示了部分分组示例；座位页面的174人是完整的会议参会者数据。

**建议**：两个页面应该展示同一次会议的完整数据，或在页面上明确说明这是演示数据。
