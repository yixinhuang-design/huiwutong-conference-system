# 排座功能快速测试指南

## 问题回顾

用户反馈："执行排座不生效"

## 问题原因

1. **缺少参会人员数据**
   - 页面首次加载时，没有从localStorage读取已保存的数据
   - 初始化默认数据只创建了座位区，没有创建测试参会人员
   - 排座函数检查到参会人员数为0，直接返回并提示"请先导入参会人员"

2. **缺少座位区验证**
   - 排座算法没有验证座位区是否存在

## 修复方案

### ✅ 已完成的修改

1. **添加初始化参会人员数据** (seating-mgr.html 4469-4487行)
   - 添加15名测试参会人员（2名VIP + 13名普通）
   - 包含完整的个人信息（姓名、部门、职位等）

2. **添加座位区验证** (seating-mgr.html 7038-7046行)
   - 在执行排座前检查座位区是否为空
   - 为空时显示"请先创建座位区"提示

3. **添加详细的排座日志** (多个位置)
   - 用户可以在浏览器Console中看到排座过程
   - 便于诊断排座失败的原因

## 排座失败的诊断

❌ **如果排座不生效，按照以下步骤诊断**：

### 症状1：点击"执行排座"后提示"请先导入参会人员"

**原因**：attendees.value.length === 0

**解决方案**：
```javascript
// 在Console中检查
console.log('参会人员总数:', attendees.value.length);

// 如果为0，则重新初始化
localStorage.removeItem('seating_conf_20240615_001');
location.reload();
```

### 症状2：点击"执行排座"后提示"请先创建座位区"

**原因**：seatingAreas.value.length === 0

**解决方案**：
```javascript
// 在Console中检查
console.log('座位区数量:', seatingAreas.value.length);

// 如果为0，则重新初始化
localStorage.removeItem('seating_conf_20240615_001');
location.reload();
```

### 症状3：排座完成但座位为空

**原因**：座位分配算法可能有问题

**解决方案**：
```javascript
// 查看第一个区域的可用座位数
const area = seatingAreas.value[0];
const availableSeats = area.rows.reduce((sum, row) => {
    return sum + (row.seats.filter(s => s.status === 'available' && !s.isAisle).length || 0);
}, 0);
console.log('可用座位数:', availableSeats);

// 手动执行排座并查看Console日志
await executeAutoArrange();
```

## 浏览器缓存清理方法

如果修改后页面仍显示旧内容，需要清理缓存：

### Windows + Chrome浏览器
1. 按 `Ctrl + Shift + Delete`
2. 选择"所有时间"
3. 勾选"Cookies和其他网站数据"、"缓存的图片和文件"
4. 点击"清除数据"
5. 刷新页面（`Ctrl + R`或`F5`）

### 快速硬刷新
1. 在页面上按 `Ctrl + Shift + R`
2. 等待页面重新加载

### 清理localStorage
```javascript
// 在Console中执行
localStorage.clear();
location.reload();
```
