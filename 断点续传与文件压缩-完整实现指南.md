# 断点续传与文件压缩功能 - 完整实现指南

## 📋 功能概述

本功能模块在原有课件上传基础上，增加了以下高级特性：

### 核心特性
1. **断点续传（Resume Upload）**
   - 将大文件分块上传（默认5MB/块）
   - 支持单个分块失败重试（最多3次）
   - 支持暂停和继续上传
   - 支持上传进度实时查询
   - 自动合并分块到完整文件

2. **文件压缩处理**
   - 图片压缩（JPG/PNG）：降低分辨率、调整质量参数
   - PDF压缩接口（预留集成点）
   - 可配置压缩启用/禁用
   - 自动估算压缩比例

3. **上传管理**
   - 上传任务初始化和元数据管理
   - 分块进度跟踪
   - 过期临时文件自动清理
   - 详细的错误处理和日志记录

## 📦 文件存储结构

### 目录布局
```
项目根目录/
├── uploads/                          # 最终上传目录
│   ├── {tenantId}/                   # 租户隔离
│   │   └── schedule/                 # 日程文件
│   │       └── {scheduleId}/         # 日程ID
│   │           └── {yyyyMMdd}/       # 日期
│   │               ├── 1704067200000_课件.pptx
│   │               ├── 1704067201000_讲义.pdf
│   │               └── ...
│   │
│   └── .temp/                        # 临时目录（分块上传）
│       ├── {uploadId1}/              # 上传任务ID
│       │   ├── chunk_0
│       │   ├── chunk_1
│       │   ├── chunk_2
│       │   └── chunk_3
│       ├── {uploadId2}/
│       │   └── ...
│       └── ...
```

### 文件命名规则
- **最终文件**: `{timestamp}_{originalFileName}`
  - 例：`1704067200000_课件.pptx`
  - 时间戳作为前缀防止重名

- **分块文件**: `chunk_{index}`
  - 例：`chunk_0`, `chunk_1`, `chunk_2`
  - 纯数字索引，便于顺序合并

- **压缩文件**: `{timestamp}_{filename}_compressed.{ext}`
  - 例：`1704067200000_课件_compressed.pptx`

## ⚙️ 工作流程

### 完整的上传流程

```
1. 文件选择
   └─ 用户选择文件
   
2. 文件验证
   ├─ 检查文件大小 (≤100MB)
   ├─ 检查文件类型（白名单）
   └─ 验证通过 ✓
   
3. 哈希计算
   ├─ SHA256(完整文件)
   ├─ 生成uploadId (UUID)
   └─ 分割为5MB分块
   
4. 任务初始化 [POST /api/schedule/chunk/init]
   ├─ 验证日程和权限
   ├─ 创建临时目录
   ├─ 初始化上传任务
   └─ 返回uploadId和分块数
   
5. 分块并发上传 [POST /api/schedule/chunk/upload] ×N
   ├─ 并发上传3个分块（可配置）
   ├─ 计算每块SHA256
   ├─ 写入临时目录
   ├─ 验证分块完整性
   └─ 更新进度(支持暂停/继续)
   
6. 自动合并 [POST /api/schedule/chunk/finish]
   ├─ 检查所有分块已上传
   ├─ 顺序读取分块文件
   ├─ 合并为完整文件
   ├─ 应用压缩处理
   ├─ 保存附件记录
   ├─ 清理临时文件
   └─ 返回附件ID
   
7. 上传完成
   └─ 更新UI，显示下载链接
```

### 暂停和继续流程

```
上传中 ──暂停─→ 已暂停 ──继续─→ 上传中 ──完成─→ 已完成
  │                                              ↑
  └──────────取消──────────→ 已取消──清理临时文件─┘
```

## 📊 性能指标

### 理论性能
- **单分块上传**: ~1-5秒 (5MB @ 10Mbps)
- **100MB文件**: ~50-100秒 (20个分块)
- **1GB文件**: ~8-16分钟 (200个分块，3并发)
- **压缩率**: 图片60-70%, PDF30-50%

### 资源占用
- **内存**: 每个上传任务~1KB元数据 (1000任务 = 1MB)
- **磁盘临时**: 最多100MB × 并发数
- **带宽**: 取决于网络，实时显示

## 🐛 故障处理

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|--------|
| 分块上传失败 | 网络中断 | 自动重试3次，若仍失败可取消重新上传 |
| 临时文件堆积 | 服务重启未清理 | 定时任务自动清理24小时未更新的临时文件 |
| 合并后文件损坏 | 分块传输出错 | SHA256校验，分块失败会拒绝合并 |
| 磁盘空间不足 | 文件过大或未及时清理 | 检查uploads目录，删除过期文件 |

1. **租户隔离**: 所有操作验证tenantId，防止数据泄露

4. **文件类型**: 白名单限制允许的文件类型
5. **大小限制**: 100MB上限防止滥用
6. **临时文件**: 自动清理过期临时文件
7. **日志记录**: 记录所有上传操作便于审计
