## 五、智能分配算法说明

### 5.1 三种分配算法

#### 1. 部门分组算法（DEPARTMENT_GROUPING）
- **算法逻辑**：
  1. 分离VIP和普通参会人员
  2. 将普通参会人员按部门分组
  3. VIP优先安排到前排座位
  4. 普通人员按部门顺序排列
  
- **适用场景**：部门为单位的会议
- **时间复杂度**：O(n log n + m)
- **优点**：便于部门集中管理

#### 2. 职级优先算法（PRIORITY_BASED）
- **算法逻辑**：
  1. 根据职位计算优先级
  2. 按优先级从高到低分配座位
  3. 职位权重：主席(100) > 副市长(70) > 处长(50) > 普通(10)
  
- **适用场景**：政府部门、企业等层级分明的会议
- **时间复杂度**：O(n log n)
- **优点**：尊重职级差异

#### 3. VIP优化算法（VIP_OPTIMIZATION）
- **算法逻辑**：
  1. 分离VIP和非VIP参会人员
  2. VIP分配到专门的VIP座位
  3. 非VIP分配到剩余座位
  
- **适用场景**：VIP人数较多的会议
- **时间复杂度**：O(n + m)
- **优点**：处理速度快

### 5.2 算法对比

| 特性 | 部门分组 | 职级优先 | VIP优化 |
|------|---------|---------|---------|
| **复杂度** | O(n log n) | O(n log n) | O(n + m) |
| **速度** | 中 | 中 | 快 |
| **公平性** | 高 | 中 | 高 |
| **可定制性** | 低 | 高 | 中 |
| **推荐用途** | 部门会议 | 层级会议 | VIP会议 |

```javascript
// 导入参会人员
async function importAttendees() {
  const response = await fetch('/api/seating/attendees/import', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      conferenceId: this.currentConference.id,
      importType: 'EXCEL',
      attendees: this.attendeeList
    })
  });
  const result = await response.json();
  this.$message.success(`导入了${result.data}名参会人员`);
}

// 自动座位分配
async function autoAssign() {
  const response = await fetch('/api/seating/assign/auto', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      conferenceId: this.currentConference.id,
      venueId: this.currentVenue.id,
      algorithmName: this.selectedAlgorithm,
      assignType: 'AUTO'
    })
  });
  const result = await response.json();
  if (result.code === 0) {
    this.$message.success('分配成功');
    this.displayAssignmentResult(result.data);
  }
}
```
