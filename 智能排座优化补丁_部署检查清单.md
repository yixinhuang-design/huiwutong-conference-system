## 🔧 部署步骤

### 步骤1：编译后端代码

```bash
# 进入后端项目目录
cd backend/conference-backend

# 清理和编译
mvn clean compile

# 期望输出
# [INFO] Building conference-backend 1.0.0-SNAPSHOT
# [INFO] Building conference-meeting 1.0.0-SNAPSHOT
# [INFO] Building conference-seating 1.0.0-SNAPSHOT
# [INFO] BUILD SUCCESS
```

**检查清单**：
- [ ] 编译无错误
- [ ] 编译无警告（或只有无关警告）
- [ ] 依赖解析完整

### 步骤2：生成可执行包

```bash
# 打包（跳过单元测试）
mvn clean package -DskipTests

# 期望生成文件
# target/conference-backend-1.0.0.jar
# target/conference-seating-1.0.0.jar

# 文件大小应该正常（>10MB）
ls -lh target/*.jar
```

**检查清单**：
- [ ] JAR 文件生成成功
- [ ] JAR 文件大小正常（>10MB）
- [ ] 打包过程无错误

### 步骤3：停止旧服务

```bash
# 查找运行的 Java 进程
ps aux | grep java | grep conference

# 获取 PID（进程ID）
# 示例输出：root 12345 ...

# 杀死旧进程（替换 12345 为实际 PID）
kill -9 12345

# 等待进程完全退出
sleep 2

# 验证进程已停止
ps aux | grep java | grep conference
# 应该没有输出（除了 grep 命令本身）
```

**检查清单**：
- [ ] 旧服务进程已停止
- [ ] 端口 8080 已释放

### 步骤4：启动新服务

#### 方式A：直接运行 JAR

```bash

cd backend/conference-backend/target

# 启动应用（使用 nohup 在后台运行）
nohup java -jar conference-backend-1.0.0.jar > app.log 2>&1 &

# 或者使用 screen（保持会话）
screen -S conference-backend
java -jar conference-backend-1.0.0.jar

# 使用 Ctrl+A 然后 D 分离 screen 会话
```

#### 方式B：使用启动脚本

```bash

./startup.sh

# 查看日志
tail -f logs/application.log
```

**检查清单**：
- [ ] 应用启动无错误
- [ ] 看到 "Started Application in X.XXX seconds" 日志
- [ ] 应用监听在 8080 端口

### 步骤5：验证后端服务

## 🚨 故障排查

### 问题1：API 仍返回 404

**症状**：
```
GET /api/seating/layout 404 (Not Found)
```

**排查步骤**：
1. [ ] 确认后端服务已重启（`curl localhost:8080/actuator/health`）
2. [ ] 确认 SeatingController.java 已重新编译
3. [ ] 检查是否有两个 `@GetMapping("/layout")` 定义
   ```bash
   grep -n '@GetMapping("/layout")' SeatingController.java
   # 应该只有 1 个结果
   ```
4. [ ] 检查 JAR 文件是否是最新版本
   ```bash
   ls -l target/conference-backend-*.jar
   # 查看修改时间
   ```

**解决方案**：
```bash
# 重新编译
mvn clean compile

# 重新打包
mvn package -DskipTests

# 停止旧服务
pkill -f conference-backend

# 启动新服务
java -jar target/conference-backend-1.0.0.jar
```

### 问题2：浏览器仍显示"参会人员为空"

**症状**：
```
[loadSeatingData] 参会人员为空，补充默认测试数据
```

**排查步骤**：
1. [ ] 确认浏览器缓存已清除
   ```javascript
   // 在控制台检查缓存
   localStorage.clear();
   location.reload();
   ```
2. [ ] 确认 Network 中 `/api/seating/layout` 返回完整数据
3. [ ] 确认响应包含 `seats` 字段（不是占位符）

**解决方案**：
```javascript
// 在浏览器控制台
localStorage.clear();
sessionStorage.clear();
location.reload();  // 强制刷新
```
