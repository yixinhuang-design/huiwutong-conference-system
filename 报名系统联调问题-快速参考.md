## 问题简明说明

### 问题1：数据库字段映射混乱
**现象**：Java代码用realName，但数据库存name
```java
@TableField("name") private String realName;        // 混乱！
@TableField("organization") private String department;  // 混乱！
```
**影响**：代码混淆，容易出错  
**修复难度**：高（涉及全局改动）

---

### 问题2：前后端字段定义不同步
**现象**：前端hardcode过滤字段，后端hardcode验证字段，两端列表不同
```javascript
// 前端
const fixed = new Set(['name', 'realname', 'phone', 'unit', 'department', 'position', 'remark', 'email']);

// 后端
defaults.add(systemField("name", "姓名", true));     // 不包括'realname'和'email'
defaults.add(systemField("phone", "手机号", true));
defaults.add(systemField("unit", "单位", true));
defaults.add(systemField("position", "职位", true));
```
**影响**：字段验证行为不一致  
**修复难度**：中等（需要同步两端定义）

---

### 问题3：字段配置加载失败看不出来
**现象**：getRegistrationFormFields查询数据库失败时，悄悄回退到默认配置，无日志
```java
if (!StringUtils.hasText(configText)) {
    return defaultFormFields();  // ← 无法知道是否真的加载了数据库配置
}
```
**影响**：无法诊断为什么字段配置不生效  
**修复状态**：✅ 已添加日志

---

### 问题4：idCard验证要求与前端发送值不匹配
**现象**：
- 前端可能发'未识别'（OCR失败）
- 后端要求@NotBlank，不允许特殊值
```javascript
// 前端
idCard: this.upload?.ocr?.idCard || '未识别'

// 后端
@NotBlank(message = "身份证号不能为空")
private String idCard;
```
**影响**：提交失败，错误信息与实际不符  
**修复状态**：✅ 已修复（移除@NotBlank，添加容错）

---

### 问题5：OCR被识别两次
**现象**：
- 前端上传后，立即调用OCR接口识别
- 后端submitRegistration中，再次调用OCR识别同一张图

```javascript
// 前端已识别
const ocr = await request({ url: REGISTRATION_OCR?photoUrl=... });

// POST到后端时
{ registrationData: JSON.stringify({ ocr: {...} }) }

// 后端再识别！
if (StringUtils.hasText(request.getIdCardPhotoUrl())) {
    Map<String, String> ocrResult = ocrIdCard(request.getIdCardPhotoUrl());  // ←重复！
}
```
**影响**：
- 浪费OCR服务调用（可能产生成本）
- 两个识别结果可能不同，导致数据不一致
- 增加响应时间

**修复状态**：✅ 已修复（添加hasOcrResult检查）

---

### 问题6：JSON解析失败被悄悄忽略
**现象**：
```java
private JSONObject safeObject(String json) {
    try {
        return JSON.parseObject(json);
    } catch (Exception e) {
        return new JSONObject();  // ← 直接返回空对象，无日志！
    }
}
```
**影响**：数据被丢失，无法诊断解析失败的原因  
**修复状态**：⚠️ 未修复（需评估是否启用严格模式）

## 快速诊断

### 如果报名提交失败

**检查清单**：

1. **查看前端console**
   ```javascript
   // 应该看到：
   // ✓ 报名提交成功
   // ✗ 提交失败: xxx
   ```

2. **查看后端日志**
   ```
   # 新增日志会显示：
   [DEBUG] idCard为特殊值[未识别]，已转换为null
   [DEBUG] 会议[xxx]的动态字段配置: [...]
   [ERROR] 必填字段[phone]手机号为空
   ```

3. **查看Network请求**
   - 请求URL: `POST http://localhost:8082/api/registration/submit`
   - 响应码: `200`（成功）或 `4xx/5xx`（失败）
   - 响应体: `{"code": 200, "message": "操作成功", "data": {...}}`或`{"code": 1001, "message": "...错误信息"}`

4. **检查数据库**
   ```sql
   SELECT * FROM conf_registration ORDER BY create_time DESC LIMIT 1;
   ```
   - 如果有记录 → 提交成功
   - 如果无记录 → 后端save失败（查看日志和code）
