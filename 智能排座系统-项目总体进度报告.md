- [x] 导入依赖和异常处理完善

### 2.2 核心功能说明

**功能**: 交换两个座位的分配人员

**API端点**: 
```
POST /api/seating/seats/swap/{seatId1}/{seatId2}
```

**业务规则**:
1. 验证两个座位都存在
2. 验证两个座位属于同一会场
3. 交换座位的assigned_user_id和assigned_user_name
4. 返回交换前后的对比结果

**响应示例**:
```json
{
  "success": true,
  "seatId1": 1,
  "seatId2": 2,
  "beforeSeat1Assignee": "张三",
  "beforeSeat2Assignee": "李四",
  "afterSeat1Assignee": "李四",
  "afterSeat2Assignee": "张三",
  "swapTime": "2026-03-12T11:00:00"
}
```
