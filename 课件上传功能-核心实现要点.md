#### 3. 关键业务逻辑

**文件哈希防重复**:
```java
// 步骤1: 计算文件SHA256哈希
String fileHash = calculateFileHash(file);

// 步骤2: 查询是否存在相同哈希
ScheduleAttachment existing = scheduleAttachmentMapper.selectByFileHash(scheduleId, fileHash);

// 步骤3: 如果存在则拒绝上传
if (existing != null && existing.getDeleted() == 0) {
    throw new BusinessException("此文件已上传过，无需重复上传");
}
```

**文件保存路径**:
```
uploads/
├── {tenantId}/              // 租户隔离
│   └── schedule/
│       └── {scheduleId}/    // 按日程分组
│           └── {date}/      // 按日期分组 (yyyyMMdd)
│               └── {timestamp}_{originalFileName}
```

**权限和租户隔离**:
```java
// 每个操作都验证租户
Long tenantId = TenantContextHolder.getTenantId();

// 每个操作都验证权限
if (!attachment.getTenantId().equals(tenantId)) {
    throw new BusinessException("无权限访问");
}
```

### 数据库设计

#### 表结构：conf_schedule_attachment

```sql
CREATE TABLE conf_schedule_attachment (
    id BIGINT PRIMARY KEY,                    -- 附件ID
    schedule_id BIGINT NOT NULL,              -- 所属日程
    meeting_id BIGINT NOT NULL,               -- 所属会议（冗余）
    tenant_id BIGINT NOT NULL,                -- 租户ID（隔离）
    file_name VARCHAR(255) NOT NULL,          -- 原始文件名
    file_size BIGINT,                         -- 文件大小（字节）
    file_type VARCHAR(50),                    -- 文件扩展名
    file_url VARCHAR(500) NOT NULL,           -- 访问路径
    file_hash VARCHAR(255),                   -- SHA256哈希（防重复）
    description VARCHAR(255),                 -- 文件描述
    upload_by BIGINT,                         -- 上传人ID
    upload_time DATETIME DEFAULT CURRENT_TIMESTAMP, -- 上传时间
    download_count INT DEFAULT 0,             -- 下载次数（统计）
    deleted TINYINT DEFAULT 0,                -- 删除标记（逻辑删除）
    
    KEY idx_schedule_id (schedule_id),
    KEY idx_meeting_id (meeting_id),
    KEY idx_tenant_id (tenant_id),
    UNIQUE KEY uk_schedule_file (schedule_id, file_hash, deleted)
);
```

#### 关键字段说明

| 字段 | 用途 | 示例值 |
|------|------|--------|
| id | 主键 | 1234567890 |
| schedule_id | 关联日程 | 1001 |
| tenant_id | 租户隔离 | 2027317834622709762 |
| file_hash | 防重复 | a1b2c3d4e5f6...（SHA256） |
| file_url | 访问路径 | /uploads/2027.../schedule/1001/.../xxx.pptx |
| download_count | 热度统计 | 5（下载5次） |
| deleted | 逻辑删除 | 0=正常, 1=已删除 |

## 🎯 技术特点

### 1. 文件验证（三层）

**前端验证**:
- 文件选择器中的accept属性限制类型
- JavaScript中的文件大小检查

**后端验证**:
- 文件大小检查（≤100MB）
- 文件类型白名单（ppt, pptx, pdf, doc, docx, xls, xlsx, txt, jpg, png, gif）
- MIME类型验证（可选）

**数据库约束**:
- file_name唯一性（同scheduleId下）
- file_hash唯一性（防重复）

### 2. 安全特性

**多租户隔离**:
- 每个请求从TenantContextHolder获取tenantId
- 所有查询、更新、删除都验证tenantId
- 存储路径包含tenantId: `uploads/{tenantId}/...`

**权限检查**:
- 每个操作都验证Authorization header
- 每个操作都验证用户是否有权访问该资源
- 逻辑删除（deleted标记），不进行物理删除

**文件安全**:
- 不直接暴露文件系统路径，而是通过API访问
- 文件名使用时间戳前缀防止名字冲突
- SHA256哈希防止恶意重复上传

### 3. 性能优化

**防重复**:
- SHA256哈希计算（同一文件多次上传只需上传一次）
- 数据库唯一约束

**统计分析**:
- download_count字段（无需额外查询）
- 可直接获取热门文件排行

**并发处理**:
- 批量上传时使用Promise.all()并发处理
- 单个失败不影响其他文件（容错机制）

## 📊 文件流转全过程

```
用户操作
  ↓
前端JavaScript
  ├─ 选择文件（accept白名单）
  ├─ 前端验证（大小、类型）
  ├─ 构建FormData
  └─ 调用ScheduleAPI.uploadAttachment()
  ↓
后端HTTP请求
  ├─ 验证Authorization header
  ├─ 提取参数（scheduleId, file, description）
  └─ ScheduleAttachmentController.uploadAttachment()
  ↓
业务逻辑处理
  ├─ 验证tenantId（从TenantContextHolder）
  ├─ 验证scheduleId存在性
  ├─ 验证文件大小（≤100MB）
  ├─ 验证文件类型（白名单）
  ├─ 计算SHA256哈希
  ├─ 检查是否重复（selectByFileHash）
  ├─ 保存到服务器（uploads/{tenantId}/schedule/{scheduleId}/{date}/）
  ├─ 插入数据库（conf_schedule_attachment）
  └─ 返回ScheduleAttachmentResponse
  ↓
前端收到响应
  ├─ 提取data.id（附件ID）
  ├─ 添加到scheduleForm.attachments
  ├─ 刷新UI显示新文件
  └─ 显示成功提示
  ↓
用户下载
  ├─ 点击下载按钮
  ├─ 前端调用ScheduleAPI.recordDownload()（增加计数）
  ├─ 前端调用ScheduleAPI.getDownloadUrl()（获取URL）
  ├─ 获取response.data.downloadUrl
  ├─ 创建<a href="{url}">并click()
  └─ 浏览器下载文件
  ↓
后端处理下载
  ├─ FileDownloadController收到请求
  ├─ 验证tenantId和权限
  ├─ 查询数据库（conf_schedule_attachment）
  ├─ 验证文件存在（Files.exists(path)）
  ├─ 增加下载计数（downloadCount++）
  ├─ 设置HTTP响应头（Content-Disposition: attachment）
  ├─ 返回文件内容（FileSystemResource）
  └─ 下载完成
```

## ❓ 常见问题速查

| 问题 | 原因 | 解决方案 |
|------|------|--------|
| "请先保存日程" | 没有scheduleId | 先保存日程获得ID |
| "文件已上传过" | SHA256哈希重复 | 这是防重复功能，不需要重新上传 |
| "不支持的文件类型" | 文件在黑名单中 | 检查ALLOWED_EXTENSIONS白名单 |
| "文件大小不能超过100MB" | 文件太大 | 修改MAX_FILE_SIZE或分割文件 |
| "权限不足" | 跨租户访问 | 检查tenantId是否匹配 |
| 下载404错误 | 文件物理不存在 | 检查uploads目录和file_url |
| 跨域报错 | CORS未配置 | @CrossOrigin(origins="*")已添加 |
