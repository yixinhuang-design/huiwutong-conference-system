## ⚙️ 核心概念

### 1. 容量管理
```
总容量 (capacity) = 100
已分配 (assignedCount) = 60
可用 (availableSeats) = 40

✅ 分配前检查：必须 assignedCount < capacity
✅ 响应时自动计算：availableSeats = capacity - assignedCount
```

### 2. 多租户隔离
```
✅ 所有查询都按 tenant_id 和 conference_id 隔离
✅ 跨租户数据完全隔离
✅ 无需手动处理租户上下文
```

### 3. 唯一性约束
```
住宿:   房间号在同一会议内唯一
用餐:   餐厅名在同一会议内唯一
车辆:   车牌号在同一会议内唯一
```

### 4. 事务一致性
```
✅ 创建、更新、删除操作都有事务保护
✅ 异常时自动回滚
✅ 确保数据一致性
```

## ⚠️ 常见问题速答

### Q: 删除住宿时提示"有人员分配"？
```
A: 必须先取消所有人的住宿分配，才能删除
   检查: assignedCount 是否为 0
```

### Q: 分配人员时提示"容量已满"？
```
A: 该住宿/用餐/车辆已满
   解决: 选择其他有空余的资源
   或: 查询 /available 端点获取可用资源
```

### Q: 创建住宿时提示"房间号已存在"？
```
A: 同一会议内房间号不能重复
   解决: 使用其他房间号
```

### Q: 数据查询返回为空？
```
A: 1. 检查 conference_id 是否正确
   2. 检查租户ID隔离是否生效
   3. 检查数据是否真的存在
```
