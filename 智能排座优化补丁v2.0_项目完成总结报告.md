- 位置：SeatingController.java:69
- 定义：`@GetMapping("/venues/{conferenceId}")`
- 状态：✅ API存在

**诊断点3**：日程加载函数
- 位置：seating-schedule-manager.js:70
- 代码：`if (context.conference && context.conference.id)`
- 问题：❌ `context.conference.id` 返回 undefined

**诊断点4**：Vue Ref对象结构
- `context.conference` 是 Ref对象，结构：`Ref{value: {...}}`
- `context.conference.id` 试图访问Ref对象的.id属性（不存在）
- 正确方式：`context.conference.value.id`

#### 根本原因

```
┌─ 问题根源 ─────────────────────────────┐
│                                        │
│  前端 (seating-mgr.html)              │
│    ↓                                  │
│  const conference = ref(null)         │
│    ↓                                  │
│  useScheduleManager({conference})     │
│    ↑                                  │
│  传入的是Ref对象                      │
│    ↓                                  │
│  后端 (seating-schedule-manager.js)  │
│    ↓                                  │
│  context.conference.id                │
│    ↓                                  │
│  尝试访问Ref的.id属性 (不存在)       │
│    ↓ ❌                               │
│  undefined → 条件失败 → 使用本地数据  │
│                                        │
└────────────────────────────────────────┘
```

**根本原因**：Vue Ref对象的属性访问层级错误，导致会议ID识别失败

## 🔧 修复详情

### 修复原理

使用**可选链操作符**和**逻辑OR运算符**来兼容两种数据结构：

```javascript
const conferenceId = context.conference?.value?.id || context.conference?.id;
```

**工作流程**：
1. 尝试 `context.conference?.value?.id`：如果context.conference是Ref对象，通过.value访问实际数据
2. 如果上一步失败，尝试 `context.conference?.id`：如果context.conference是普通对象，直接访问
3. 两者都失败则返回undefined（安全降级）

**兼容性**：
```
输入1：{ conference: Ref{value: {id: 123}} }  → 返回 123 ✅
输入2：{ conference: {id: 456} }              → 返回 456 ✅
输入3：{ conference: null }                   → 返回 undefined (安全)✅
```
