### 测试2：在左侧插入过道 ✅

**前置条件：**
- 至少有一个座位区域（例如：A区 5行10列）
- 座位数据已正确加载

**操作步骤：**
1. 找到一个标准座位区域
2. 右键点击第5列的任意座位
3. 从菜单中选择"在左边插入过道"
4. 观察座位区的变化

**预期结果：**
- [ ] 新的过道列插入在第5列的左侧（即原来的第4-5列之间）
- [ ] 过道列显示为虚线框，背景为浅橙色
- [ ] 过道列中心显示"--- 过道 ---"文字
- [ ] 原来的第5-10列现在变成第6-11列
- [ ] 原来的第1-4列保持不变
- [ ] 页面右下方的统计信息更新（总座位数增加）
- [ ] 浏览器Console输出：
  ```
  [insertColumnAisle] 已在第 5 列的左边插入过道列
  ```

**验证方法：**
1. 鼠标悬停在插入后的座位上，查看座位编号提示
2. 检查座位的 col 属性值是否正确

## 故障排除

### 如果菜单没有显示

**诊断步骤：**

1. **检查浏览器控制台**
   ```
   打开 F12 → Console 标签
   右键点击座位，查看是否有错误信息
   ```

2. **检查右键事件绑定**
   ```javascript
   // 在Console中运行以下代码验证：
   console.log('contextMenu状态:', contextMenu);
   console.log('showSeatContextMenu函数:', showSeatContextMenu);
   ```

3. **清除浏览器缓存**
   ```
   Ctrl+Shift+Delete 打开清除缓存对话框
   清除所有缓存后刷新页面
   ```

4. **检查座位元素**
   ```
   F12 → Elements 标签
   找到 .seat 元素，检查是否有 @contextmenu 事件绑定
   ```

### 如果过道没有插入

**诊断步骤：**

1. **检查列号**
   - 确保 `seat.col` 值正确
   - 在Console中输出座位对象：`console.log(seat)`

2. **检查insertColumnAisle函数**
   - 查看Console是否有调用日志
   - 检查传入的参数是否正确

3. **查看localStorage**
   ```javascript
   // 在Console中运行以下代码查看保存的数据：
   const key = 'seating_' + conference.value.id;
   console.log(localStorage.getItem(key));
   ```
