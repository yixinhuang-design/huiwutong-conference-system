## 🔍 关键问题解答

### Q: 这个功能的核心是什么？
**A:** 用户可以直接在座位布局图上点击按钮来添加过道，而**不会改变座位的编号**。这是最重要的需求。

### Q: 座位编号是如何保持连贯的？
**A:** 过道行的编号设为 null，不参与"排"的编号。其他行仍保持原有编号（1, 2, 3...）。这样添加过道也不会改变原有座位号。

### Q: 这个功能有哪 4 个操作按钮？
**A:** 
- `⬆` 在该行上方插入过道
- `⬇` 在该行下方插入过道
- `⬅` 在该列左边插入过道
- `➡` 在该列右边插入过道

### Q: 性能如何？
**A:** 对于 12×20 的座位区，完整操作耗时 < 5ms，无感知延迟。

### Q: 哪些浏览器支持？
**A:** Chrome 51+、Firefox 54+、Safari 13+、Edge 79+ 都完全支持。IE 11 不支持（需要 Vue 3）。

## 🆘 遇到问题？

### 问题1：按钮不显示
```
解决步骤：
1. 刷新浏览器 (Ctrl+F5)
2. 打开开发者工具 (F12)
3. 查看 Console 是否有错误
4. 确认创建了座位区域
```

### 问题2：点击按钮无反应
```
解决步骤：
1. 检查浏览器开发者工具 Console
2. 查看是否有 JavaScript 错误
3. 确认您的浏览器版本支持（Vue 3.x 需要）
4. 尝试在控制台输入：insertRowAisle
   - 如果返回函数定义，说明函数已加载
```

### 问题3：过道行编号不对
```
这不是问题！
过道行的编号是 null（不显示行号）
这是正确的行为，确保座位编号连贯
如果您看到过道行显示"过道"文本，这就是正确的
```

### 问题4：座位编号改变了
```
这不应该发生！
插入过道不会改变座位编号
如果发现编号改变了，请：
1. 刷新页面重新加载
2. 报告此问题（这是 Bug）
3. 查看 seating-mgr.html 中的 insertRowAisle 函数
```
