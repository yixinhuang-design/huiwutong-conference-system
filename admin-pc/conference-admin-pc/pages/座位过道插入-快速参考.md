# 座位过道插入 - 快速参考卡片

## 📌 一页纸总结

### 什么是这个功能？
在座位管理系统的主座位布局显示区中，用户可以直接点击按钮来添加过道（走道），无需打开配置弹窗。

### 怎么用？

```
座位布局：
    ⬅ 左侧插入过道列
     ↓
 ⬆ 1排 [□][□][□][□] ➡ 右侧插入过道列
 ⬇
 ⬆ 2排 [□][□][□][□] ➡
 ⬇
```

**4个操作按钮：**
- `⬆` 在该行上方插入过道
- `⬇` 在该行下方插入过道  
- `⬅` 在该列左边插入过道
- `➡` 在该列右边插入过道

### 重要特性
✅ 座位编号保持连贯（插入过道不改变座位号）
✅ 过道行不占用编号（过道行的number为null）
✅ 操作即时生效（无需确认）
✅ 自动保存（刷新后仍存在）

## 🆘 常见问题 FAQ

**Q: 能否撤销已插入的过道？**
A: 当前版本不支持。可以编辑区域重新配置，或删除区域重新创建。

**Q: 过道座位能否分配参会人员？**
A: 不能。过道座位显示为灰色虚线，无法点击分配。

**Q: 插入过道会改变座位编号吗？**
A: 不会。这是本功能最重要的特性——座位编号保持连贯不变。

**Q: 能否调整过道宽度（2列或更多）？**
A: 当前版本不支持，只能插入单列/单行。未来版本可能支持。

**Q: 已分配的座位会受影响吗？**
A: 不会。座位号不变，分配信息完全保留。

## 🚀 性能指标

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 插入行过道 | O(n) | n = 行数，需要重新编号 |
| 插入列过道 | O(n×m) | n×m = 座位总数 |
| 数据保存 | O(n×m) | 序列化整个座位区 |

**典型场景（12排×20列）：**
- 插入行过道：< 1ms
- 插入列过道：< 2ms
- 总计：< 5ms（包括保存）
