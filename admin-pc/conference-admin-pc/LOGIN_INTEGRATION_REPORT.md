## 🚀 快速开始

### 1. 启动后端服务
```bash
cd "G:\huiwutong新版合集\backend\conference-backend"
mvn -s maven-settings.xml -f conference-auth\pom.xml spring-boot:run
```
服务将在 `http://localhost:8081` 启动

### 2. 打开登录页面
```
file:///G:/huiwutong新版合集/admin-pc/conference-admin-pc/pages/login.html
```
或访问: `http://localhost:8081/pages/login.html` (如果配置web服务器)

### 3. 使用默认凭证登录
**系统管理员模式**:
- 用户名: `admin`
- 密码: `123456`

**租户管理员模式**:
- 租户: `默认租户 (default)`
- 用户名: `admin`
- 密码: `123456`

### 4. 登录成功后
- 自动跳转到首页 (`index.html`)
- 用户信息保存在localStorage
- 可以继续使用系统各项功能

## 📊 API调用流程

```
┌─────────────────────────────────────┐
│   用户在登录页输入凭证              │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│   验证表单（客户端验证）            │
│   - 检查必填字段                    │
│   - 检查格式                        │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│   调用AuthService.login()           │
│   发送POST请求到/auth/login         │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│   后端验证凭证                      │
│   - 查询用户                        │
│   - 验证密码                        │
│   - 生成JWT Token                   │
└──────────────┬──────────────────────┘
               │
               ▼
         ┌──┴──┐
         │     │
    成功 │     │ 失败
         │     │
         ▼     ▼
    ┌─────┐ ┌──────────┐
    │保存 │ │显示错误  │
    │Token│ │提示      │
    └──┬──┘ └──────────┘
       │
       ▼
    跳转首页
```

## 💾 localStorage 数据结构

### 认证信息
```javascript
{
    authToken: "eyJhbGciOiJIUzM4NCJ9...",      // 访问令牌
    refreshToken: "eyJhbGciOiJIUzM4NCJ9...",   // 刷新令牌
    tokenType: "Bearer",                        // Token类型
    tokenExpireTime: 1772187168000,             // 过期时间戳
    expiresIn: 86400                            // 过期秒数
}
```

### 用户信息
```javascript
{
    userId: "1",                        // 用户ID
    username: "admin",                  // 用户名
    realName: "管理员",                 // 真实姓名
    userType: "admin",                  // 用户类型(admin/staff/learner)
    tenantId: "1",                      // 租户ID
    tenantName: "默认租户",             // 租户名称
    tenantCode: "default"               // 租户编码
}
```

### 偏好设置
```javascript
{
    rememberedUsername: "admin",        // 记住的用户名
    rememberedTenantCode: "default",    // 记住的租户
    lastVisitTime: "2026-02-26T..."     // 最后访问时间
}
```

## 📝 使用说明

1. 修改API地址: `auth-service.js` 第5行的 `baseURL`

1. 在服务器上部署登录页面
2. 配置后端认证服务地址
3. 确保数据库中有用户数据
4. 定期审查登录日志

### 用户
1. 在浏览器中打开登录页
2. 选择登录方式（系统管理员/租户管理员）
3. 输入凭证后登录
4. 系统自动跳转到首页

## 🐛 已知限制

1. Token存储在localStorage中，存在XSS风险
   - **解决方案**: 后续升级到使用httpOnly Cookie

2. 不支持自动重连
   - **解决方案**: 用户需要刷新页面或重新登录

3. 没有实现登出日志
   - **解决方案**: 后续可在后端记录登出事件

4. 租户列表硬编码为默认租户
   - **解决方案**: 需要从后端API动态加载

## 📞 技术支持

### 常见问题

**Q: 登录后页面空白？**
A: 检查浏览器控制台是否有JavaScript错误，确保所有依赖脚本都加载完成。

**Q: 无法连接到后端服务？**
A: 确保认证服务在 http://localhost:8081 正常运行。

**Q: Token过期后怎么办？**
A: 系统会自动尝试刷新Token，如果失败则重定向到登录页。

**Q: 忘记密码怎么办？**
A: 目前需要由系统管理员手动重置，后续会实现自助重置功能。
