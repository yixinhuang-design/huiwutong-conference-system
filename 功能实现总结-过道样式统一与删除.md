# 过道样式统一与删除功能实现总结

## 问题描述
用户报告的两个主要问题：
1. 左右插入的过道样式与上下插入的过道样式不一致
2. 插入的过道无法被删除

## 解决方案

### 1. 过道样式统一

#### 问题分析
- **行级过道（上下插入）**：使用 `.seat-row.aisle-row` 样式，具有橙色虚线边框和浅橙色背景
- **列级过道（左右插入）**：只显示小灰色点标记（`.aisle-marker`），视觉表现不一致

#### 实现方案
统一使用 `.seat.aisle` CSS 类应用于所有过道座位：

```css
/* 过道座位样式 - 统一的视觉表现 */
.seat.aisle {
    border: 2px dashed var(--warning-color);      /* 橙色虚线边框 */
    background: rgba(245, 158, 11, 0.08);         /* 浅橙色背景 */
    box-shadow: none;
    cursor: context-menu;                          /* 右键菜单光标 */
}

.seat.aisle:hover {
    background: rgba(245, 158, 11, 0.15);         /* 悬停时加深背景 */
}
```

#### 技术实现细节
1. **CSS 样式位置**：[seating-mgr.html](seating-mgr.html#L1130-L1140)
2. **class 绑定**：通过 `getSeatClass()` 方法自动绑定（line 4805）
3. **应用方式**：
   - 当 `seat.isAisle === true` 时自动添加 `aisle` 类
   - 与行级过道样式视觉上保持一致

### 2. 过道删除功能

#### 实现流程

##### 步骤 1：右键菜单集成
- **菜单位置**：[seating-mgr.html](seating-mgr.html#L3329-L3336)
- **显示条件**：`v-if="contextMenu.isAisle"`
- **删除按钮**：
  ```html
  <div class="context-menu-item" @click="deleteAisle(); hideContextMenu()">
      <i class="fas fa-trash-alt" style="color: #ef4444;"></i>
      <span style="color: #ef4444;">删除过道</span>
  </div>
  ```

##### 步骤 2：菜单上下文更新
修改 `showSeatContextMenu()` 方法检测过道座位：
```javascript
contextMenu.seat = seat;                    // 保存座位对象
contextMenu.isAisle = seat.isAisle || false; // 检测是否过道
```
- **位置**：[seating-mgr.html](seating-mgr.html#L5735-L5750)

##### 步骤 3：ReactiveObject 扩展
更新 `contextMenu` 对象结构：
```javascript
const contextMenu = reactive({
    visible: false,
    x: 0,
    y: 0,
    type: null,
    area: null,
    rowIndex: null,
    colIndex: null,
    seat: null,      // NEW：座位对象引用
    isAisle: false   // NEW：过道标志
});
```
- **位置**：[seating-mgr.html](seating-mgr.html#L4209-L4220)

##### 步骤 4：deleteAisle() 方法实现
位置：[seating-mgr.html](seating-mgr.html#L5866-L5922)

**功能说明**：
1. **行级过道删除**：
   - 检测当前行是否为 `isAisle === true`
   - 从 `area.rows` 数组中删除整行
   - 更新后续行的行号

2. **列级过道删除**：
   - 查找指定列中的过道座位
   - 从每一行中删除对应列的座位
   - 更新后续座位的列号

3. **数据持久化**：
   - 调用 `saveData()` 保存更改
   - 调用 `updateStatistics()` 更新统计信息
   - 自动隐藏右键菜单

##### 步骤 5：方法导出
在组件返回对象中添加导出：
```javascript
deleteAisle,  // 添加到返回列表
```
- **位置**：[seating-mgr.html](seating-mgr.html#L8985)

#### 菜单隐藏处理
更新 `hideContextMenu()` 清除新增字段：
```javascript
contextMenu.seat = null;
contextMenu.isAisle = false;
```
- **位置**：[seating-mgr.html](seating-mgr.html#L5755-L5762)

## 测试场景

### 场景 1：列级过道样式
1. 创建座位区域
2. 右键点击某个座位，选择"← 插入过道左"或"→ 插入过道右"
3. **预期结果**：新插入的列级过道显示与行级过道相同的橙色虚线边框和浅橙色背景

1. 右键点击已插入的列级过道座位

3. 点击删除
4. **预期结果**：整列过道被删除，后续列号自动更新

1. 右键点击行级过道座位
2. 菜单中应显示红色的"删除过道"选项
3. 点击删除
4. **预期结果**：整行过道被删除，后续行号自动更新

## 使用指南

### 统一过道样式
- 系统现在自动为所有过道座位应用统一的橙色虚线边框样式
- 无需额外配置

### 删除过道
1. 右键点击要删除的过道座位
2. 在右键菜单中点击红色"删除过道"选项
3. 过道被删除，座位编号自动更新

## 注意事项

1. **删除不可逆**：当前版本删除过道后无法撤销，请谨慎操作
2. **数据自动保存**：删除过道后会自动保存数据
3. **行号/列号自动更新**：删除过道后，系统自动更新后续的行号和列号
4. **统计更新**：删除过道后统计信息自动更新
