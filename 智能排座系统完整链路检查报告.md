### 2️⃣ 数据保存链路

#### 链路图
```

    ├─→ conferenceId: 会议ID

    │   ├─→ numberingMode, startNumber
    │   ├─→ layoutJson: 完整区域JSON
    │   └─→ seats: 座位数组
    │       ├─→ seatId, row, col, seatNumber
    │       ├─→ seatType, status
    │       └─→ occupantId, occupantName

后端处理

    ├─→ 保存/更新 SeatingSeat（座位明细）
    └─→ 保存参会人员关联
    ↓
返回结果
    ├─→ 成功：console.log确认
    └─→ 失败：console.warn，数据仍在localStorage
```

## ⚠️ 发现的问题

### 问题1：批量分配前端算法未对接后端
**严重程度**：🟡 P2 - 功能限制

**问题描述**：
- 前端有完整的批量分配算法
- 但仅在前端计算，未调用后端API
- 后端没有对应的批量分配接口

**影响**：
- 前端批量分配后，需要手动保存
- 后端无法直接执行批量分配

**建议**：
- 后端添加批量分配接口
- 支持多种分配策略

### 问题2：Excel导入导出仅前端实现
**严重程度**：🟢 P3 - 便利性功能

**问题描述**：
- Excel导入导出仅使用前端库
- 数据仅在前端处理
- 未同步到后端数据库

**影响**：
- 导入的数据需要手动保存
- 导出的数据可能与数据库不一致

**建议**：
- 后端添加导入导出接口
- 支持服务端生成Excel
