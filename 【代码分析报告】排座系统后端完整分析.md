### 6.2 表结构详解

#### 表1: conf_seating_venue (会场配置表)
```
主要字段: id, conference_id, tenant_id, venue_name, venue_type(normal/ushape/round/theater)
         total_rows, total_columns, capacity, layout_data(JSON), status
索引优化: uk_conf_venue, idx_tenant_conference, idx_status
特点: 支持多种会场布局类型
```

#### 表2: conf_seating_seat (座位表)
```
主要字段: id, conference_id, venue_id, tenant_id, seat_number, row_num, column_num
         seat_type(normal/vip/reserved/aisle), status(available/assigned/reserved/aisle)
         assigned_user_id, assigned_user_name
索引优化: uk_seat_number, idx_conference_venue, idx_tenant_conference, idx_status
特点: 座位号唯一性保证，易于查找
```

#### 表3: conf_seating_attendee (参会人员表)
```
主要字段: id, conference_id, tenant_id, user_id, name, department, position
         phone, email, is_vip, is_reserved, assigned_seat_id, assigned_at
         confirmed, attendance_status, extra_data
索引优化: idx_conference_attendee, idx_tenant_conference, idx_user_id, idx_is_vip
特点: 包含VIP标记、预留座位标记、出席状态、扩展数据
```

#### 表4: conf_seating_assignment (分配记录表)
```
主要字段: id, conference_id, seat_id, attendee_id, tenant_id, schedule_id
         assign_type(auto/manual/swap), algorithm_name
         assign_time, assigned_by, status, confirm_time, cancel_reason
索引优化: idx_conference_assignment, idx_tenant_conference, idx_seat_attendee, idx_status
特点: 记录分配历史和取消原因，支持追溯
```

#### 表5: conf_seating_schedule (日程表)
```
主要字段: id, conference_id, tenant_id, schedule_name, schedule_date
         start_time, end_time, description, is_default
索引优化: idx_conference_schedule, idx_tenant_conference, idx_is_default
特点: 支持多日程管理，可设置默认日程
```

#### 表6-8: 辅助表 (车辆、住宿、用餐)
```
特点: 都包含conference_id, tenant_id用于多租户隔离
     都包含基本的时间戳和计数字段
     支持批量分配和占用统计
```

#### 表9: conf_seating_assignment_history (分配历史表)
```
主要字段: id, assignment_id, conference_id, tenant_id, seat_id, attendee_id
         old_seat_id, operation(assign/cancel/swap), operator_id
功能: 记录每一次座位分配变更，支持完整的操作审计
```

### 6.3 多租户支持

所有表都包含 `tenant_id` 字段：
- 复合索引 `idx_tenant_conference` 加速多租户查询
- 数据完全隔离，无数据跨租户泄露风险
- 支持单一数据库多租户部署

### 6.4 索引策略

**主键索引**: 所有表的 id 字段
**唯一索引**: 
- uk_seat_number: 座位号的唯一性
- uk_tenant_seat: 租户内座位号唯一
- uk_room: 酒店房间唯一

**普通索引**:
- idx_tenant_conference: 多租户+会议查询
- idx_status: 状态查询
- idx_assigned_user: 人员查询
- idx_is_vip: VIP过滤
- 等共30+个
