## 📊 常见问题排查

### Q1: 编译错误 "cannot resolve symbol"

**原因**: 导入路径不对或文件位置错误

**解决**:
```bash
# 检查文件位置
find src -name "AccommodationCreateRequest.java"

# 应该在：
# src/main/java/com/conference/seating/dto/AccommodationCreateRequest.java

# 检查包名
head -5 src/main/java/com/conference/seating/dto/AccommodationCreateRequest.java
# 应该是：package com.conference.seating.dto;
```

### Q2: SQL错误 "Table doesn't exist"

**原因**: 忘记执行初始化脚本

**解决**:
```sql
-- 检查表是否存在
SHOW TABLES LIKE 'conf_seating_%';

-- 如果没有，执行脚本
SOURCE p2-seating-functions-init.sql;

-- 再次检查
SHOW TABLES LIKE 'conf_seating_%';
```

### Q3: API返回 403 Forbidden

**原因**: 多租户隔离导致权限检查失败

**解决**:
```java
// 检查当前租户ID
Long tenantId = TenantContextHolder.getTenantId();
System.out.println("Current tenant: " + tenantId);

// 检查数据库中的租户ID
SELECT DISTINCT tenant_id FROM conf_seating_accommodation;

// 确保查询的资源属于当前租户
```

### Q4: 容量检查失败

**原因**: assignedCount >= capacity

**解决**:
```sql
-- 检查容量
SELECT id, capacity, assigned_count FROM conf_seating_accommodation;

-- 重置容量（仅测试环境）
UPDATE conf_seating_accommodation SET assigned_count = 0 WHERE id = 1;
```

### Q5: 性能下降

**原因**: 索引未创建或查询优化问题

**解决**:
```sql
-- 检查索引是否存在
SHOW INDEX FROM conf_seating_accommodation;
SHOW INDEX FROM conf_seating_dining;
SHOW INDEX FROM conf_seating_transport;

-- 如果索引缺失，重新创建
ALTER TABLE conf_seating_accommodation 
ADD INDEX idx_conference (conference_id, tenant_id);

-- 分析查询性能
EXPLAIN SELECT * FROM conf_seating_accommodation 
WHERE conference_id = 1 AND tenant_id = 1;
```
