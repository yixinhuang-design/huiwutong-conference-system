## 三、核心功能实现

**支持三种导入方式**：

1. **Excel导入**
   - 接收AttendeeData列表
   - 支持name, department, position, phone, email, isVip等字段
   - 自动验证必填字段

2. **报名系统导入**
   - 指定报名系统API地址
   - 自动调用API获取参会人员数据
   - TODO: 需要实现具体的API调用逻辑

3. **模拟数据导入**
   - 指定参会人员数量（1-1000）
   - 自动生成模拟数据
   - 包含5个部门、6个职位的随机组合
   - 前3个参会人员自动标记为VIP

### 3.2 座位分配功能

**两种分配模式**：

1. **自动分配（Auto）**
   - 支持3种算法：DEPARTMENT_GROUPING、PRIORITY_BASED、VIP_OPTIMIZATION
   - 完整的分配结果统计
   - 自动保存分配结果到数据库

2. **预览模式（Preview）**
   - 生成分配结果但不保存到数据库
   - 用于用户预览分配效果
   - 支持调整参数后重新预览

### 3.3 座位管理功能

**座位生命周期管理**：

```
AVAILABLE → OCCUPIED → AVAILABLE
    ↓
  RESERVED
    ↓
  OCCUPIED
```

**关键操作**：
- 批量导入座位（根据行列数自动生成）
- 查询座位列表
- 查询可用座位
- 分配座位（自动生成座位号：A01, A02...）
- 取消分配
- 座位统计

### 3.4 参会人员管理功能

**参会人员生命周期管理**：

```
导入 → 注册 → 签到 → 参会
         ↓
      缺席
```

**关键操作**：

## 六、完整的工作流程

### 6.1 参会人员分配的完整流程

```
1. 导入参会人员 (POST /api/seating/attendees/import)
   ↓
2. 导入座位 (POST /api/seating/seats/import)
   ↓
3. 预览分配结果 (POST /api/seating/assign/preview)
   ↓
4. 确认无误后自动分配 (POST /api/seating/assign/auto)
   ↓
5. 如需调整，手动分配特定座位 (POST /api/seating/seats/assign)
   ↓
6. 查看分配统计 (GET /api/seating/statistics/...)
```
