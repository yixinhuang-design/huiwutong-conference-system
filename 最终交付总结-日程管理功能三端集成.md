## 🚀 技术架构亮点

### 1. 完整的微服务架构
```
客户端层 (Web + 两个 App)
     ↓
API 客户端 (schedule-api.js)
     ↓
REST API (17 个端点)
     ↓
服务层 (IScheduleService)
     ↓
数据访问层 (Mapper)
     ↓
MySQL 数据库 (8 个表)
```

### 2. 多租户设计
- 所有数据按 tenant_id 隔离
- 自动过滤租户数据
- 用户只能访问自己的数据

### 3. 状态机管理
```
0(草稿) → 1(已发布) → 2(进行中) → 3(已完成)
        ↘ 4(已取消) ←―――――――――――――――
```

### 4. 完整的错误处理
```
Controller(请求验证) → Service(业务逻辑) → Mapper(数据库)
         ↓                  ↓                  ↓
      400/401             业务异常          DB异常
         ↓                  ↓                  ↓
         └──────────→ 统一异常处理 ←─────────┘
                         ↓
                     用户友好提示
```

### 5. 前端降级策略
```
API 成功 → 显示真实数据
    ↓
API 失败 → 降级到模拟数据（只读）
    ↓
继续为用户提供服务
```
