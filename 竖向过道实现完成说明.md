# 竖向过道实现完成说明

## 实现内容

### 1. 竖向过道数据结构
- 在每个座位区域（area）中添加了 `columnAisles` 数组
- 该数组存储需要显示竖向过道的列号（colIndex）
- 支持多个竖向过道同时存在

### 2. 竖向过道插入功能
- 用户右键点击座位后，可选择"在左边插入竖向过道"或"在右边插入竖向过道"
- 过道列号会被添加到 `area.columnAisles` 数组中
- 支持在任意列位置插入过道

### 3. 竖向过道删除功能
- 用户右键点击竖向过道本身，显示"删除竖向过道"选项
- 点击后过道列号会从 `area.columnAisles` 数组中移除
- 删除后过道立即从界面消失

### 4. 竖向过道显示样式
- 使用 2px 橙色虚线边框（`border: 2px dashed #f59e0b`）
- 浅橙色背景（`rgba(245, 158, 11, 0.08)`）
- 与横向过道样式统一
- 鼠标悬停时背景颜色加深

### 5. 竖向过道位置计算
- 使用动态DOM测量方式获取实际座位位置
- 在组件挂载后自动测量所有座位的水平位置
- 将座位位置存储在 `seatColumnPositions` 响应式对象中
- 过道位置通过 `getSeatColumnPosition()` 方法动态获取
- 插入或删除过道后自动重新测量位置，确保准确性

### 6. 右键菜单集成
- 座位右键菜单：显示"在左边插入竖向过道"和"在右边插入竖向过道"选项
- 竖向过道右键菜单：显示"删除竖向过道"选项
- 行级过道右键菜单：显示"删除过道"选项

## 技术要点

### DOM 测量方式
- 在Vue的 `onMounted` 生命周期中自动调用
- 使用 `nextTick` 确保DOM已渲染
- 使用 `setTimeout` 额外延迟100ms，确保复杂布局的测量准确
- 只需测量第一个座位行，其他行假设列对齐

### 绝对定位原理
- 竖向过道使用绝对定位（`position: absolute`）
- 相对于 `.seating-area-container`（`position: relative`）定位
- `left` 值通过测量座位的实际位置动态设置
- `height: 100%` 使其跨越容器的整个高度

### 响应式更新
- `seatColumnPositions` 是 `reactive` 对象，座位位置改变时会重新渲染
- 插入/删除过道时自动重新测量，确保位置正确

## 已知限制

1. 如果座位行的宽度不同（由于 `justify-content: center`），某些行的对齐可能略有偏差
   - 解决方案：实际测量时取所有行的平均位置或最常见位置

2. 竖向过道不能自动调整座位列号
   - 目前的实现是在既有列号处插入过道，不改变座位的col值

3. 暂不支持过道列号的自动编号（如插入后自动更新后续列号）

- DOM 测量只在组件挂载和过道插入/删除时执行
- 使用缓存机制避免重复测量同一列
- 绝对定位不影响文档流，不需要重排其他元素
