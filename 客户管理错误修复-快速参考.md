## 🔍 400 错误深度诊断

### 原因分析

`400 Bad Request` 通常意味着：

1. **请求格式错误** → 参数不正确
2. **缺少认证令牌** → Authorization header不存在或无效
3. **JSON格式错误** → 请求体格式不对

### 逐步排查

#### 检查1: 后端是否在运行

```bash
# 在PowerShell中测试:
$response = Invoke-WebRequest -Uri "http://localhost:8081/api/tenant/list" -Method GET -ErrorAction SilentlyContinue
$response.StatusCode

# 预期结果:
# 400 (意味着后端在运行，只是参数问题)
# 500 (后端有错误，查看日志)
# 连接失败 (后端未启动)
```

#### 检查2: 修改后端代码禁用权限检查（临时）

如果权限是问题，可以临时禁用权限检查来诊断：

**文件**: `backend/.../TenantManagementController.java`

```java
// 在第47-53行，修改:
private Result<Void> checkSuperAdminPermission(Long userId) {
    // ❌ 注释掉权限检查（仅用于诊断）
    // if (!isSuperAdmin(userId)) {
    //     return Result.fail(ResultCode.FORBIDDEN, "...权限..."); 
    // }
    
    // ✅ 临时允许所有请求
    return null;
}
```

然后重新编译和运行：
```bash
mvn clean package -DskipTests -pl conference-auth
```

#### 检查3: 使用API测试工具的"获取列表"

使用工具中的功能直接测试API，观察响应：

```
状态码 200 → ✅ API正常
状态码 400 → 参数问题，检查后端日志
状态码 403 → 权限问题，禁用权限检查
```

## FAQ - 常见问题

### Q1: 我看到的错误信息与上面描述不同怎么办？

**A**: 可能有其他问题。收集错误信息和调试数据（见上文），对照 `客户管理API错误修复指南.md` 中的问题诊断部分。

### Q2: 修复后仍然显示演示数据而不是真实数据

**A**: 说明无法连接后端API。检查：
1. 后端服务是否运行在 http://localhost:8081
2. 防火墙是否阻止了8081端口  
3. MySQL数据库是否有4个租户

使用API测试工具验证后端连接。

### Q3: 页面显示"加载中..."后卡住

**A**: 后端响应超时或无响应。检查：
1. 后端是否仍在运行（查看Maven日志）
2. 数据库连接是否正常
3. 网络连接是否正常

尝试使用API测试工具的"检查后端"按钮。

### Q4: 创建/编辑/删除不工作

**A**: 如果是权限错误（403），需要：
1. 在数据库中验证当前用户是否是super_admin
2. 或临时禁用权限检查进行测试（见上文）

### Q5: Clipboard.min.js 错误持续出现

**A**: 这是一个独立问题，不影响主功能。可以：
1. 尝试在无痕模式下打开页面
2. 换个浏览器试试
3. 禁用浏览器扩展

可以安全地忽略这个错误。
