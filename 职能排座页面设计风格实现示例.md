## 表单实现示例

### 示例1: 完整的配置表单

```html
<!-- HTML 结构 -->
<div class="form-group">
    <label>
        参会人数
        <span class="required">*</span>
        <span class="hint">必填项</span>
    </label>
    <div class="number-input">
        <button @click="attendeeCount--" :disabled="attendeeCount <= 1">−</button>
        <input 
            type="number" 
            v-model.number="attendeeCount"
            min="1"
            max="1000"
            @change="validateAttendeeCount"
        />
        <button @click="attendeeCount++">+</button>
    </div>
    <span class="input-hint">输入范围: 1 - 1000 人</span>
</div>

<!-- 验证反馈 -->
<span v-if="errors.attendeeCount" class="error-message">
    <i class="fas fa-exclamation-circle"></i>
    {{ errors.attendeeCount }}
</span>

<span v-if="validations.attendeeCount" class="success-message">
    <i class="fas fa-check-circle"></i>
    {{ validations.attendeeCount }}
</span>
```

## 常见问题排查

### 问题1: 弹窗样式不显示

**检查清单**：
- [ ] CSS 文件已正确引入
- [ ] 浏览器控制台没有 404 错误
- [ ] CSS 文件路径正确
- [ ] 没有被其他样式覆盖

**解决方案**：
```bash
# 在浏览器控制台执行
console.log(getComputedStyle(document.querySelector('.modal-content')).backgroundColor)
# 应该显示半透明白色
```

### 问题2: 弹窗超出屏幕

**检查清单**：
- [ ] 弹窗宽度设置正确
- [ ] `max-height: 90vh` 已设置
- [ ] `modal-body` 的 `overflow-y: auto` 已设置

**解决方案**：
```css
/* 确保 modal-body 可滚动 */
.modal-body {
    flex: 1;
    overflow-y: auto;
    max-height: calc(90vh - 200px);  /* 减去头部和页脚的高度 */
}
```

### 问题3: 响应式不工作

**检查清单**：
- [ ] HTML 有正确的 viewport meta 标签
- [ ] CSS 媒体查询正确
- [ ] 没有硬编码宽度

**解决方案**：
```html
<!-- 在 <head> 中确保有 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

## 优化建议

### 性能优化

1. **使用 CSS 变量**：所有颜色、间距都应使用 CSS 变量，便于主题切换
2. **精简选择器**：避免过长的选择器链
3. **考虑使用 CSS Grid**：替代 Flexbox 用于复杂布局

### 可维护性优化

1. **添加注释**：为复杂的 CSS 规则添加注释
2. **使用 BEM 命名法**：保持类名语义清晰
3. **创建 SCSS 变量**：如果后续转为 SCSS

### 可访问性优化

1. **增加焦点指示器**：确保键盘用户能清晰看到焦点
2. **使用 ARIA 属性**：为复杂组件添加角色和状态信息
3. **测试屏幕阅读器**：确保内容可读性
