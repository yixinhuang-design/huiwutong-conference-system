# 报名弹窗模拟数据问题诊断与修复方案

## 问题描述

用户点击报名统计数字（42人已报名、2人待审核、40人已通过）后，弹窗中显示的是**硬编码的模拟数据**，而不是实际数据库中的真实报名数据。

## 根本原因分析

### 1. 数据显示流程

```
点击统计数字 (line 155, 160, 165)
  ↓
openRegistrationModal(type)  [line 1275]
  ↓ 设置 this.showRegistrationModal = true, this.modalType = type
  ↓
弹窗显示
  ↓
getFilteredRegistrationList()  [line 1300]
  ↓
使用 this.registrationLists[this.modalType]  [line 1301]
  ↓
显示数据
```

```javascript
getFilteredRegistrationList() {
    const list = this.registrationLists[this.modalType] || [];  // ❌ 直接使用硬编码数据
    const filtered = list.filter(item => {
        return item.name.includes(this.searchKeyword) || item.phone.includes(this.searchKeyword);
    });
    const start = (this.currentPage - 1) * this.pageSize;
    return filtered.slice(start, start + this.pageSize);
}
```

### 3. 为什么显示统计数字是正确的

**文件**: [admin-pc/conference-admin-pc/pages/registration-mgr.html](registration-mgr.html)

后端API `/api/registration/stats` 调用 (Line 1064)
```javascript
async loadRegistrationStats() {
    // ... 从实际数据库获取统计 ...
    const resp = await fetch(
        `http://localhost:8082/api/registration/stats?conferenceId=${encodeURIComponent(String(conferenceId))}`,
        { method: 'GET', headers: this.getApiHeaders() }
    );
    const result = await resp.json();
    if (resp.ok && result.code === 200 && result.data) {
        const stats = result.data;
        approved = Number(stats.approved || 0);  // ✅ 真实数据
        pending = Number(stats.pending || 0);    // ✅ 真实数据
        total = approved + pending + ...;
    }
}
```

而conference-detail.html中虽然也加载了统计，但**没有同时加载详细列表数据**。

## 问题的完整链路

### Backend API 已提供

- ✅ `GET /api/registration/stats?conferenceId=X` → 返回统计数字 (status, count)
- ✅ `GET /api/registration/list?conferenceId=X&page=1&pageSize=1000` → 返回报名列表

## 修复方案

### 方案：修改conference-detail.html

**修改3个位置**:

#### 修改1: 替换硬编码的registrationLists数据源

**位置**: Line 1024-1200左右
**动作**: 将hardcoded数据删除，改为空数组初始化

#### 修改2: 在openRegistrationModal中加载真实数据

**位置**: Line 1275-1282
**动作**: 调用API加载实际数据

#### 修改3: 添加辅助方法loadRegistrationListData

**动作**: 新增方法调用 /api/registration/list API

### 数据加载流程 (修复后)

```
点击统计数字
  ↓
openRegistrationModal(type)
  ↓ 调用 loadRegistrationListData(conferenceId, type)
  ↓
fetch('/api/registration/list?conferenceId=X&status=type')
  ↓
获取实际数据，填充到 registrationLists[type]
  ↓
弹窗显示真实数据
```
