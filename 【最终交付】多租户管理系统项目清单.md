## 🎯 核心功能清单

### ✅ 后端 API (7 个端点)

```
1. GET /api/tenant/list
   - 获取租户列表（分页）
   - 权限: 超级管理员
   - 参数: page, pageSize, searchName, status

2. GET /api/tenant/{tenantId}
   - 获取租户详情
   - 权限: 超级管理员
   - 返回: 租户完整信息

3. POST /api/tenant/create
   - 创建新租户
   - 权限: 超级管理员
   - 验证: 编码唯一性、必填字段

4. PUT /api/tenant/{tenantId}
   - 更新租户信息
   - 权限: 超级管理员
   - 允许字段: 名称、描述、状态

5. DELETE /api/tenant/{tenantId}
   - 删除租户
   - 权限: 超级管理员
   - 警告: 级联删除，无法撤销

6. PUT /api/tenant/{tenantId}/status
   - 切换租户状态
   - 权限: 超级管理员
   - 支持: active / inactive

7. GET /api/tenant/{tenantId}/stats
   - 获取租户统计
   - 权限: 超级管理员
   - 包含: 用户数、会议数、注册数等
```

## 🔐 权限控制

### 权限架构
```
所有 API 端点均要求: 超级管理员权限

验证流程:
1. 请求包含 Authorization: Bearer {token}
2. TenantFilter 从 Token 中提取 tenantId/userId/username
3. TenantContextHolder 存储到 ThreadLocal
4. checkSuperAdminPermission() 检查用户是否为超级管理员
5. 不满足权限条件返回 403 FORBIDDEN

错误响应:
{
  "code": 403,
  "message": "仅超级管理员可访问租户管理功能"
}
```

### 权限实现位置
```
TenantManagementController:
- isSuperAdmin(Long userId)              // 权限判断
- checkSuperAdminPermission(Long userId) // 权限检查

所有 7 个端点都调用 checkSuperAdminPermission()
```

## 🚀 快速启动流程

### Step 1: 编译后端
```bash
cd backend/conference-backend
mvn clean package -DskipTests
# 预期: BUILD SUCCESS (13-15 秒)
```

### Step 2: 启动服务
```bash
# 终端 1 - 启动网关
java -jar conference-gateway/target/conference-gateway-1.0.0.jar

# 终端 2 - 启动认证服务（含租户管理 API）
java -jar conference-auth/target/conference-auth-1.0.0.jar
```

### Step 3: 登录获取 Token
```bash
curl -X POST http://localhost:8081/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'
```

### Step 4: 保存 Token
```javascript
// 浏览器控制台
localStorage.setItem('auth_token', 'YOUR_TOKEN_HERE');
localStorage.setItem('current_tenant_id', '1');
```

### Step 5: 访问管理后台
```
http://localhost:8080/pages/customer-management.html
```

### Step 6: 验证功能
- 查看租户列表
- 创建新租户
- 编辑租户信息
- 删除租户
