## 2️⃣ LocalStorage依赖优化 ✅

### 实施内容

#### 2.1 创建排座API模块
**文件**：`admin-pc/conference-admin-pc/js/api/seating-api.js`

**功能**：
- `saveLayout()` - 保存座位布局到后端
- `loadLayout()` - 从后端加载座位布局
- `assignSeat()` - 分配座位
- `swapSeats()` - 交换座位

#### 2.2 修改加载逻辑

**修改位置**：`loadSeatingDataFromLocalStorage()` 方法（约5650行）

**优化前**：
```javascript
// 仅从LocalStorage加载
const savedData = localStorage.getItem(seatingKey);
if (savedData) {
    const data = JSON.parse(savedData);
    seatingAreas.value = data.seatingAreas || [];
}
```

**优化后**：
```javascript
// ===== 优化：优先从后端加载数据 =====
try {
    console.log(`尝试从后端加载布局数据...`);
    const data = await SeatingAPI.loadLayout(confId, schedId);
    
    if (data && data.areas) {
        seatingAreas.value = data.areas || [];
        // 同时缓存到LocalStorage
        localStorage.setItem(cacheKey, JSON.stringify(data));
        console.log(`✅ 从后端成功加载数据`);
        return;
    }
} catch (err) {
    console.warn(`从后端加载失败，尝试从LocalStorage加载`);
}

// 后端加载失败，从LocalStorage加载（fallback）
const savedData = localStorage.getItem(seatingKey);
if (savedData) {
    const data = JSON.parse(savedData);
    seatingAreas.value = data.seatingAreas || [];
}
```

|-----|-------|-------|------|
| 后端存储 | ⚠️ 仅保存 | ✅ 保存+加载 | ⬆️ 100% |

### 数据流转策略

```
优先级1: 后端存储（DB） ← 主要存储
    ↓ 失败时
优先级2: LocalStorage ← 缓存fallback
    ↓ 失败时
优先级3: 初始化空数据 ← 最后兜底
```
