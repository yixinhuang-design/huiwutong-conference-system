### 2. ID 不匹配问题

#### 错误的ID（旧值）
- **uni-app入口页面硬编码**: `conferenceId=2030309010523144194` 
  - 位置: [app-uniapp/pages/common/index/index.vue#L11](app-uniapp/pages/common/index/index.vue#L11)

#### 正确的ID（新值）
- **数据库实际存储**: `meeting_id=2030309010523144200`
- **ID差值**: `2030309010523144200 - 2030309010523144194 = 6`

#### 影响链路
```
前端入口(错误ID) → 报名页面(继承错误ID) → 后端保存(使用错误ID) 
                                                    ↓
管理后台(获取不到ID或使用错误ID) ← ← ← ← 数据库(meeting_id=正确ID)
```

### 3. 管理后台ID获取逻辑

**getConferenceId()方法** ([registration-mgr.html#L945](admin-pc/conference-admin-pc/pages/registration-mgr.html#L945)):
```javascript
getConferenceId() {
    // 优先级1: URL参数
    const conferenceId = new URLSearchParams(window.location.search).get('conferenceId');
    if (conferenceId) return conferenceId;
    
    // 优先级2: 全局上下文
    if (window.conferenceContext && window.conferenceContext.currentConference) {
        return window.conferenceContext.currentConference.id;
    }
    
    // 优先级3: localStorage
    return localStorage.getItem('currentConferenceId');
}
```

**getPageUrl()方法** ([index.html#L705](admin-pc/conference-admin-pc/index.html#L705)):
```javascript
getPageUrl(pagePath) {
    const conferenceId = this.currentConference ? this.currentConference.id : 
                       (this.conferences.length > 0 ? this.conferences[0].id : 'conf_20240615_001');
    return `${pagePath}?conferenceId=${conferenceId}`;
}
```

**问题**: 
- 如果后端会议列表加载失败，会使用mock数据的ID `conf_20240615_001`（字符串格式）
- 即使成功加载，也可能没有设置正确的默认会议

### 修复2: 管理后台自动设置默认会议ID ✅

**文件**: [admin-pc/conference-admin-pc/index.html](admin-pc/conference-admin-pc/index.html)

**修改内容**:
```javascript
async initializeApp() {
    try {
        // 初始化会议上下文
        await this.loadConferences();
        
        // 设置默认会议ID（确保报名管理页面能正确查询数据）
        const defaultConferenceId = '2030309010523144200';
        if (!localStorage.getItem('currentConferenceId')) {
            localStorage.setItem('currentConferenceId', defaultConferenceId);
            console.log('已设置默认会议ID:', defaultConferenceId);
        }
        
        // 如果没有加载到真实会议，设置当前会议为默认值
        if (this.conferences.length === 0 || this.conferences[0].id.startsWith('conf_')) {
            this.currentConference = {
                id: defaultConferenceId,
                name: '当前会议',
                status: 'preparing'
            };
            // 同时更新window全局上下文
            window.conferenceContext = { currentConference: this.currentConference };
        }
        
        this.setupRealtimeUpdates();
    } catch (error) {
        console.error('应用初始化失败:', error);
        this.showNotification('应用初始化失败', 'error');
    }
}
```

**修改位置**: 第418-440行（initializeApp方法内）

## 技术要点说明

### 1. 数据库字段映射
```java
// Registration.java
@TableName("conf_registration")
public class Registration {
    @TableField("meeting_id")  // 数据库列名
    private Long conferenceId;  // Java字段名
}
```
**注意**: 后端代码使用 `conferenceId`，但数据库列名是 `meeting_id`

### 2. MyBatis-Plus查询条件
```java
// RegistrationServiceImpl.java
LambdaQueryWrapper<Registration> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(Registration::getConferenceId, conferenceId);  // 自动映射到 meeting_id
```

### 3. 前端ID传递链路
```
uni-app入口页面 (index.vue)
  ↓ URL参数: ?conferenceId=2030309010523144200
报名页面 (scan-register.vue)
  ↓ onLoad: this.conferenceId = options.conferenceId
提交方法 (submit)
  ↓ POST请求: { conferenceId: Number(this.conferenceId) }
后端Controller (RegistrationController)
  ↓ @RequestBody RegistrationRequest request
后端Service (RegistrationServiceImpl)
  ↓ registration.setConferenceId(request.getConferenceId())
数据库 (conf_registration)
  ↓ INSERT: meeting_id = 2030309010523144200
```

### 4. 管理后台ID获取优先级
```
1. URL参数 (?conferenceId=xxx)
   ↓ 如果没有
2. window.conferenceContext.currentConference.id
   ↓ 如果没有
3. localStorage.getItem('currentConferenceId')
   ↓ 如果还是没有
4. 返回 null 或 undefined（查询失败）
```

## 潜在问题与预防

### 问题1: 雪花ID生成不一致
**现象**: 不同时间点生成的会议ID不同，导致数据关联失败

**解决方案**:
- 使用统一的会议ID配置文件或环境变量
- 建议创建 `app-uniapp/common/config.js`:
  ```javascript
  export const DEFAULT_CONFERENCE_ID = '2030309010523144200';
  ```

### 问题2: Mock数据覆盖真实数据
**现象**: 后端接口失败时，前端使用mock数据的ID（字符串格式）

**解决方案**:
- 确保mock数据的ID与真实ID格式一致（长整型）
- 添加类型检查，拒绝字符串格式的会议ID

### 问题3: 多会议场景下的ID管理
**现象**: 系统有多个会议时，ID管理混乱

**建议方案**:
- 在管理后台增加 **"会议选择"** 下拉菜单
- 动态切换 `currentConferenceId`
- URL参数优先级应高于localStorage
