## 📋 集成概述

智能排座系统现在已经与报名系统完全集成，参会人员数据直接来源于报名系统中已审核通过的人员，无需手动导入或Excel操作。


## 🎯 核心功能

### 1. 自动导入参会人员

**触发方式**:
- ✅ 页面初始化时自动导入
- ✅ 点击"从报名系统导入"按钮手动导入

**导入条件**:
- 仅导入 `status === 'approved'` (已审核通过) 的人员
- 自动过滤待审核 (pending) 和已拒绝 (rejected) 的人员

### 2. 数据字段映射

| 报名系统字段 | 参会人员字段 | 说明 |
|------------|------------|------|
| `registration.id` | `attendee.registrationId` | 保留原始报名ID |
| `registration.name` | `attendee.name` | 姓名 |
| `registration.unit` | `attendee.department` | 单位作为部门 |
| `registration.department` | `attendee.originalDepartment` | 保留原始部门 |
| `registration.position` | `attendee.position` | 职位 |
| `registration.phone` | `attendee.phone` | 手机号 |
| `registration.email` | `attendee.email` | 邮箱 |
| - | `attendee.isVip` | 根据职位自动判断 |
| `registration.submitTime` | `attendee.registrationTime` | 报名时间 |
| `registration.auditTime` | `attendee.auditTime` | 审核时间 |
| - | `attendee.source` | 标记为 'registration' |

### 3. VIP自动判断

**VIP职位列表**:
- 书记、市长、局长、主任、部长
- 副主席、副主任、副部长
- 秘书长、副秘书长

**判断逻辑**:
```javascript
isVipByPosition(position) {
    const vipPositions = ['书记', '市长', '局长', '主任', '部长', ...];
    return vipPositions.some(keyword => position.includes(keyword));
}
```

## 🚀 使用流程

### 方式一：自动导入（推荐）

1. 用户在报名系统提交报名信息
2. 管理员在报名管理页面审核通过
3. 打开智能排座系统，**系统自动导入已审核通过的人员**
4. 执行智能排座

### 方式二：手动导入

1. 在报名管理页面点击"智能排座"按钮
2. 系统自动保存报名数据到localStorage
3. 跳转到智能排座系统
4. 在排座系统点击"从报名系统导入"按钮
5. 确认导入结果

### 方式三：Excel导入（备用）

1. 如果报名系统不可用
2. 可以使用"导入Excel名单"功能
3. 或使用"手动添加"功能

## 🔧 技术实现

### 核心方法

#### 1. importFromRegistration()
从报名系统导入的主方法

```javascript
async importFromRegistration() {
    // 1. 显示加载提示
    this.$showLoading('正在从报名系统导入...');

    // 2. 获取已审核通过的人员
    const registrations = await this.fetchApprovedRegistrations();

    // 3. 转换数据格式
    this.attendees = this.convertRegistrationsToAttendees(registrations);

    // 4. 更新统计
    this.updateStatistics();

    // 5. 显示结果
    this.$showToast('成功导入 N 名参会人员', 'success');
}
```

#### 2. fetchApprovedRegistrations()
获取已审核通过的报名人员

```javascript
async fetchApprovedRegistrations() {
    // 1. 从localStorage获取
    const storedData = localStorage.getItem('registrationData');
    if (storedData) {
        const data = JSON.parse(storedData);
        return data.registrations.filter(r => r.status === 'approved');
    }

    // 2. 返回模拟数据（生产环境应调用API）
    return this.getMockApprovedRegistrations();
}
```

#### 3. convertRegistrationsToAttendees()
转换报名数据为参会人员数据

```javascript
convertRegistrationsToAttendees(registrations) {
    return registrations.map(reg => ({
        id: `att_${reg.id}`,
        registrationId: reg.id,
        name: reg.name,
        department: reg.unit,  // 单位作为部门
        position: reg.position,
        isVip: this.isVipByPosition(reg.position),
        hasRegistered: true,
        source: 'registration'
    }));
}
```

## 📝 注意事项

### 1. 数据一致性

- ✅ 报名系统中的 `status === 'approved'` 才会被导入
- ✅ 排座系统使用 `source: 'registration'` 标记数据来源
- ⚠️ 报名信息更新后需要重新导入

### 2. VIP判断

- ✅ 根据职位关键词自动判断
- ⚠️ 可能在排座系统中手动调整VIP状态
- ⚠️ VIP判断规则可以根据实际情况修改

### 3. 数据存储

- ✅ 使用localStorage存储报名数据
- ⚠️ localStorage有大小限制（通常5-10MB）
- 💡 大量数据建议使用后端API

### 4. 数据更新

- ✅ 报名数据更新后，点击"从报名系统导入"刷新
- ⚠️ 重新导入会覆盖现有参会人员列表
- ⚠️ 已排座的结果会被清空

1. **报名阶段**

2. **数据同步**
   - 报名数据更新后及时重新导入
   - 保持数据一致性

   - 排座结果需要二次确认

## 📞 技术支持

### 常见问题

**Q: 导入后找不到参会人员？**
A: 检查报名系统中是否有已审核通过的人员。只有 `status === 'approved'` 的人员才会被导入。

**Q: VIP判断不准确？**
A: 可以在 `isVipByPosition()` 方法中修改VIP职位关键词列表。

**Q: 部门显示不正确？**
A: 默认使用 `unit` (单位) 作为部门。如需使用 `department`，可以修改转换逻辑。

**Q: 如何批量导入？**
A: 报名系统支持批量导入，审核后会自动同步到排座系统。
