## 技术实现细节

### 核心方法

#### 1. `showSeatContextMenu()`
```javascript
触发条件：用户右键点击座位元素
功能：显示列级右键菜单（左/右插入过道）
传入参数：event, seat, area, row, seatIndex
响应内容：{
    visible: true,
    type: 'column',
    area: 座位区域对象,
    colIndex: seat.col (1-based列号),
    x, y: 鼠标坐标
}
```

#### 2. `insertColumnAisle()`
```javascript
触发条件：用户点击"在左边插入过道"或"在右边插入过道"
功能：在指定列的左侧或右侧插入过道座位
核心逻辑：
  1. 转换列索引（1-based → 0-based 数组索引）
  2. 在每一行的指定位置插入过道座位
  3. 更新受影响的座位列号
  4. 保存数据并更新统计
时间复杂度：O(行数 × 列数)
```

#### 3. `hideContextMenu()`
```javascript
功能：隐藏右键菜单并清空相关状态
```

## 用户指南

### 快速开始

1. **打开智能排座系统**
   - 访问 seating-mgr.html

2. **找到座位**
   - 在座位图中找到需要添加过道的座位

3. **右键点击**
   - 右键点击座位（会看到提示："右键：在左右方插入过道"）

4. **选择方向**
   - 从菜单中选择"在左边插入过道"或"在右边插入过道"

5. **完成！**
   - 过道立即创建，座位编号自动调整

### 常见场景

**场景1：创建左右两侧通道**
```
操作：右键座位第1列 → 选择"在左边插入过道"
     右键座位最后一列 → 选择"在右边插入过道"
结果：形成左右两侧的通道，中间是完整的座位区
```

**场景2：分隔VIP区和普通区**
```
操作：右键座位第5列 → 选择"在右边插入过道"
结果：第1-5列为一个区域，第6-10列为另一个区域
```

**场景3：撤销错误操作**
```
操作：点击"变更历史"→ "撤销"
结果：刚才的过道插入被撤销，座位回到原状
```

## 支持与反馈

### 常见问题解决

**Q：菜单没有显示？**
A：
1. 打开浏览器 F12 → Console
2. 右键点击座位，查看是否有错误
3. 检查 contextMenu.visible 是否为 true
4. 清除浏览器缓存重试

**Q：过道插入位置不对？**
A：
1. 检查座位的列号（col 属性）
2. 查看 Console 中的日志信息
3. 刷新页面后重试

**Q：数据没有保存？**
A：
1. 检查浏览器 localStorage 是否启用
2. 查看浏览器开发者工具的 Application → Storage
3. 检查浏览器是否禁用了本地存储
