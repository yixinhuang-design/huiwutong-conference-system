# 过道配置交互优化说明

## 概述

为了让用户更直观地配置座位布局中的过道，我们重新设计了座位区配置界面，提供了两种过道配置方式：

1. **快速模式**：一键选择预设的过道布局
2. **自定义模式**：在座位布局预览上直接点击座位来标记过道

## 使用方法

### 方式1：快速模式（推荐）

在创建或编辑座位区时，在"过道配置"部分有快速模式按钮：

| 按钮 | 说明 | 效果 |
|------|------|------|
| **无过道** | 不创建过道 | 所有座位都可用 |
| **左** | 左侧过道 | 左侧1列为过道 |
| **右** | 右侧过道 | 右侧1列为过道 |
| **中** | 中间过道 | 中间1列为过道 |
| **左右** | 双过道 | 左右各1列为过道 |
| **左中右** | 三过道 | 左、中、右各1列为过道 |
| **十字** | 十字过道 | 中间列+中间行都是过道 |

**操作步骤：**
1. 点击对应的快速模式按钮
2. 预览区会立即显示对应的过道效果
3. 点击"保存"按钮保存配置

### 方式2：自定义模式（灵活）

对于需要复杂过道布局的情况：

**操作步骤：**
1. 首先选择"快速模式"中的任何模式（如"中"）
2. 然后在"布局预览"区域 **点击具体的座位** 来标记或取消标记过道
3. 每次点击都会在预览中显示变化
4. 预览下方会实时显示统计：总座位数、实际可用座位数、过道座位数
5. 点击"保存"按钮保存自定义配置

**点击预览座位的效果：**
- 点击白色座位 → 标记为过道（显示为虚线框）
- 点击虚线框座位 → 取消过道标记（恢复为普通座位）
- 点击时会有放大动画反馈

## 新功能亮点

### 1. 直观的按钮式选择
- 快速模式按钮替代了原来的下拉菜单
- 一目了然，快速选择常见的过道布局

### 2. 实时预览反馈
- 选择模式后立即显示预览效果
- 支持旋转显示（根据角度配置自动旋转）
- 座位数统计实时更新

### 3. 可交互的座位预览
- 在自定义模式下，可以直接点击座位
- 每个座位可以独立地标记为过道或取消标记
- 支持任意复杂的过道配置

### 4. 视觉反馈
- 可点击座位在悬停时会放大并显示阴影
- 过道座位显示为虚线框，易于识别
- 按钮在选中时高亮显示

## 技术改进

### 代码结构优化
1. **新函数**：`togglePreviewSeatAisle()` - 处理预览座位的点击事件
2. **样式增强**：
   - `.preview-seat.clickable` - 使座位可点击
   - `.btn-mode` - 快速模式按钮样式
   - 新增hover和active状态

### 配置流程改进
1. 简化了UI复杂度
2. 提高了用户交互的直观性
3. 保留了原有的自定义过道配置功能

## 兼容性

- 所有快速模式与之前的预设模式功能完全相同
- 自定义模式提供了更灵活的座位标记方式
- 旧版本的过道配置可以直接升级使用

## 示例场景

### 场景1：简单的会议室（中间有一条走道）
```
选择 "中" 快速模式
→ 自动创建中间1列的过道
→ 左右各有7-8列座位供参会人员
```

### 场景2：大型会场（需要复杂过道）
```
选择 "左中右" 快速模式
→ 自动创建左、中、右三条过道
→ 如需微调，可在预览中点击具体座位
→ 最终保存自定义配置
```

### 场景3：U型会议（特殊布局）
```
1. 先创建基本座位区（行数×列数）
2. 选择 "十字" 快速模式
3. 在预览中根据实际需要点击调整
4. 保存配置
```

## 故障排除

### Q: 选择了快速模式但预览没有显示过道？
A: 
- 确认已点击了对应的快速模式按钮（按钮应该高亮）
- 检查预览中间是否有虚线框的座位
- 尝试刷新页面重新操作

### Q: 点击座位没有反应？
A:
- 自定义模式下座位才可点击
- 确认已点击过某个快速模式按钮
- 在预览区域内点击座位（不是预览区域外）

### Q: 如何恢复之前的下拉菜单选择？
A:
- 快速模式按钮功能与之前的下拉菜单完全相同
- 如需更复杂的配置，可以在预览中进行点击标记
