## 🔌 后端 API 接口

### 1. 获取租户列表
```http
GET /api/tenant/list?page=1&pageSize=10&searchName=&status=
```
**权限**: 超级管理员  
**响应示例**:
```json
{
  "code": 200,
  "message": "获取租户列表成功",
  "data": {
    "total": 24,
    "page": 1,
    "pageSize": 10,
    "items": [
      {
        "id": 1,
        "tenantCode": "tenant_001",
        "tenantName": "示例租户 1",
        "description": "这是一个示例租户",
        "status": "active",
        "createdAt": "2024-01-15T10:30:00",
        "updatedAt": "2024-01-20T14:45:00"
      }
    ]
  }
}
```

### 2. 获取租户详情
```http
GET /api/tenant/{tenantId}
```
**权限**: 超级管理员  
**路径参数**: 
- `tenantId` (Long): 租户 ID

### 3. 创建租户
```http
POST /api/tenant/create
Content-Type: application/json

{
  "tenantCode": "tenant_new",
  "tenantName": "新租户名称",
  "description": "租户描述"
}
```
**权限**: 超级管理员  
**验证规则**:
- `tenantCode`: 必填，不能重复
- `tenantName`: 必填
- `description`: 可选

**错误码**:
- `1003`: 租户编码已存在（DUPLICATE_ERROR）
- `1001`: 参数错误（PARAM_ERROR）

### 4. 更新租户
```http
PUT /api/tenant/{tenantId}
Content-Type: application/json

{
  "tenantName": "更新的租户名称",
  "description": "更新的描述",
  "status": "active"
}
```
**权限**: 超级管理员  
**路径参数**: 
- `tenantId` (Long): 租户 ID

### 5. 删除租户
```http
DELETE /api/tenant/{tenantId}
```
**权限**: 超级管理员  
**警告**: ⚠️ 此操作会删除租户及其所有相关数据，无法撤销

### 6. 切换租户状态
```http
PUT /api/tenant/{tenantId}/status?status=active
```
**权限**: 超级管理员  
**查询参数**:
- `status` (String): `active` 或 `inactive`

### 7. 获取租户统计
```http
GET /api/tenant/{tenantId}/stats
```
**权限**: 超级管理员  
**响应示例**:
```json
{
  "code": 200,
  "message": "获取数据成功",
  "data": {
    "tenantId": 1,
    "tenantName": "示例租户 1",
    "userCount": 42,
    "conferenceCount": 5,
    "registrationCount": 128,
    "lastActivity": "2024-01-25 15:30:00"
  }
}
```

## 🗄️ 数据库设计

### 多租户隔离

所有表都包含 `tenant_id` 列用于数据隔离：

```sql
CREATE TABLE sys_tenant (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    tenant_code VARCHAR(64) UNIQUE NOT NULL,
    tenant_name VARCHAR(255) NOT NULL,
    description TEXT,
    status VARCHAR(20) DEFAULT 'active',
    created_by BIGINT,
    updated_by BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 其他表示例
CREATE TABLE sys_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    tenant_id BIGINT NOT NULL,  -- 租户 ID
    username VARCHAR(64) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    -- ...
    FOREIGN KEY (tenant_id) REFERENCES sys_tenant(id),
    INDEX idx_tenant_id (tenant_id)
);
```

### SQL 拦截器

```java
// TenantInterceptor 在执行查询时自动添加 tenant_id 条件
// 支持 WHERE、ORDER BY、LIMIT、GROUP BY 等复杂 SQL 语句

String sql = "SELECT * FROM sys_user WHERE username = ?";
// 自动转换为：
String interceptedSql = "SELECT * FROM sys_user WHERE tenant_id = ? AND username = ?";
```

## 🔍 常见问题排查

### Q1: 403 权限错误
**原因**: 用户不是超级管理员  
**解决**: 检查用户权限配置，临时将 `isSuperAdmin()` 返回 true

### Q2: CORS 跨域错误
**原因**: 前后端端口不同  
**解决**: 在后端配置 CORS，允许来自前端的请求

```java
@CrossOrigin(origins = "http://localhost:3000")
@RestController
@RequestMapping("/api/tenant")
public class TenantManagementController { ... }
```

### Q3: 数据库连接错误
**原因**: MySQL 服务未启动或凭证错误  
**解决**: 检查 MySQL 服务状态和 application.yml 配置

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3308/conference_db
    username: root
    password: Hnhx@123
```

### Q4: Flyway 迁移失败
**原因**: 数据库已存在旧表结构  
**解决**: 清空数据库或指定 Flyway 表名

```yaml
spring:
  flyway:
    enabled: true
    out-of-order: true
    table: flyway_schema_history
```
