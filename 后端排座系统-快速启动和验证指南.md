## 一、快速启动（5分钟）

### 步骤1：数据库准备（2分钟）

```bash
# 1. 连接MySQL数据库
mysql -h127.0.0.1 -P3308 -uroot -p

# 2. 执行建表脚本
source seating_schema.sql;

# 3. 验证表创建成功
SHOW TABLES LIKE 'conf_seating_%';
```

**预期结果**：显示9张表
```
conf_seating_accommodation
conf_seating_assignment
conf_seating_assignment_history
conf_seating_attendee
conf_seating_dining
conf_seating_schedule
conf_seating_seat
conf_seating_transport
conf_seating_venue
```

### 步骤2：配置修改（1分钟）

编辑 `application.yml`，修改数据库连接信息：

```yaml
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3308/conference_seating
    username: root
    password: your-password  # 改为你的MySQL密码
    driver-class-name: com.mysql.cj.jdbc.Driver
```

### 步骤3：编译和启动（2分钟）

```bash
# 编译项目
mvn clean package -DskipTests

# 启动应用
java -jar target/conference-seating-1.0.0.jar

# 或使用Spring Boot插件启动
mvn spring-boot:run
```

**启动成功的日志**：
```

2026-03-12 10:30:05 INFO DeferredRepositoryInitializationListener: Spring Data repositories initialized
```

### 5.2 验证座位导入

```sql
SELECT COUNT(*) as '总座位数',
       SUM(CASE WHEN status = 'AVAILABLE' THEN 1 ELSE 0 END) as '可用座位',
       SUM(CASE WHEN status = 'OCCUPIED' THEN 1 ELSE 0 END) as '已占用座位'

WHERE venue_id = 2001;
```

**预期结果**（自动分配后）：
```
总座位数  可用座位  已占用座位

```

### 5.3 验证分配结果

```sql
SELECT COUNT(*) as '分配记录数',
       SUM(CASE WHEN status = 'SUCCESS' THEN 1 ELSE 0 END) as '成功分配',
       COUNT(DISTINCT algorithm_name) as '使用的算法数'
FROM conf_seating_assignment 
WHERE conference_id = 1001;
```

**预期结果**：
```
分配记录数  成功分配  使用的算法数
98        98       1
```

### 5.4 验证参会人员座位分配

```sql
SELECT d.department, COUNT(*) as '人数',
       SUM(CASE WHEN a.assigned_seat_id IS NOT NULL THEN 1 ELSE 0 END) as '已分配'
FROM conf_seating_attendee a 
LEFT JOIN conf_seating_attendee d ON a.id = d.id
WHERE a.conference_id = 1001
GROUP BY a.department;
```

## 六、常见问题排查

### 问题1：启动时数据库连接失败

**错误信息**：
```
SQLException: Access denied for user 'root'@'localhost'
```

**解决方案**：
1. 检查MySQL是否启动：`mysql -h127.0.0.1 -P3308 -uroot -p`
2. 修改application.yml中的password
3. 确保MySQL监听的是127.0.0.1:3308

### 问题2：表不存在

**错误信息**：
```
SQLException: Table 'conference_seating.conf_seating_seat' doesn't exist
```

**解决方案**：
1. 确认数据库conference_seating已创建
2. 重新执行seating_schema.sql建表脚本
3. 验证：`SHOW TABLES LIKE 'conf_seating_%';`

### 问题3：API返回400参数验证错误

**错误信息**：
```json
{
  "code": 400,
  "message": "conferenceId: 会议ID不能为空"
}
```

**解决方案**：
1. 检查请求体中是否包含所有必填字段
2. 检查字段类型是否正确
3. 参考API文档中的请求示例

### 问题4：自动分配返回成功率低

**可能原因**：
- 座位数量少于参会人员数量
- 导入座位时指定的行列数太小

**解决方案**：
- 增加座位数量：`totalRows=20&totalColumns=20`
- 检查座位导入结果

### 问题5：Swagger UI无法打开

**错误信息**：
```
404 Not Found: /swagger-ui.html
```

**可能原因**：
- 未配置Swagger/SpringDoc
- 应用未启动

**解决方案**：
- 检查pom.xml是否包含springdoc-openapi依赖
- 检查应用是否启动
- 访问其他API测试应用是否正常
