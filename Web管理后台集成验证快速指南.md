# Web 管理后台集成验证快速指南

**目的**: 快速验证 schedule-mgr.html 的 API 集成是否正确工作  
**估计时间**: 15-20 分钟  
**前置条件**: 后端服务运行中，schedule-api.js 已创建

---

## 🔧 前置设置

### 1. 启动后端服务
```bash
cd conference-meeting
mvn spring-boot:run
# 或者使用 IDE 运行 ScheduleController
```

### 2. 验证后端 API
```bash
# 测试基础连接
curl http://localhost:9001/api/schedule/list?pageNo=1&pageSize=10 \
  -H "Authorization: Bearer YOUR_TOKEN"

# 预期返回: { "code": 200, "data": [...], "message": "success" }
```

### 3. 获取测试用会议ID
从数据库或后台系统获取一个有效的 `conferenceId`，例如：
```sql
SELECT id FROM conf_conference LIMIT 1;
-- 返回类似: conf_meeting_2024_001
```

## 📖 逐步测试流程

### **测试 1: 页面加载和数据初始化** ✅
**目标**: 验证页面能否加载并从 API 获取日程列表

**步骤**:
1. 打开浏览器，访问:
   ```
   http://localhost:8080/admin-pc/pages/schedule-mgr.html?conferenceId=YOUR_CONFERENCE_ID
   ```

2. 打开开发者工具（F12 → Network 标签）

3. 观察:
   - [ ] 页面加载完成（无红色错误）
   - [ ] Network 中有 `/api/schedule/allSchedules` 请求
   - [ ] 响应状态为 200
   - [ ] 快速统计卡片显示数字（总日程数、总时长等）
   - [ ] 表格或日程列表中有数据显示

**排查**:
- 如果列表为空，检查是否有权限访问该会议的日程
- 如果网络请求失败，检查后端服务是否运行
- 如果看到模拟数据，说明 API 调用失败，已自动降级

---

### **测试 2: 创建日程** ➕
**目标**: 验证新建日程功能

**步骤**:
1. 点击页面顶部"添加日程"按钮
2. 填写表单:
   - 标题: `测试日程 ${Date.now()}`
   - 开始时间: 今天 10:00
   - 结束时间: 今天 11:00
   - 地点: 主会场
   - 主持人: 测试员
3. 点击"添加"按钮

4. 观察:
   - [ ] 出现"正在添加..."加载指示
   - [ ] Network 中有 POST `/api/schedule/create` 请求
   - [ ] 响应成功（200）
   - [ ] 表单关闭
   - [ ] 新日程出现在列表中
   - [ ] 出现"日程已添加"提示

**预期结果**: 新日程立即出现在列表顶部或按时间排序

**排查**:
```javascript
// 在浏览器控制台检查
console.log(app.$data.schedules); // 应该包含新增的日程
```

## 🐛 常见问题排查

### 问题 1: "缺少会议ID参数"
**原因**: URL 中没有 `?conferenceId=xxx` 参数  
**解决**: 正确的 URL 格式:
```
http://localhost:8080/admin-pc/pages/schedule-mgr.html?conferenceId=VALID_MEETING_ID
```

### 问题 2: 加载时显示模拟数据
**原因**: API 调用失败，已自动降级  
**检查**:
1. 打开开发者工具 → Console
2. 查看是否有错误信息
3. 检查 `/api/schedule/allSchedules` 请求是否有 401/403 错误
4. 验证 Bearer Token 是否有效

### 问题 3: 创建日程后没有出现在列表
**原因**: 可能是响应格式问题或状态码不对  
**检查**:
1. Network 中检查 POST 请求的响应体
2. 确保响应格式为: `{ "code": 200, "data": {...} }`
3. 检查浏览器 Console 中是否有错误

### 问题 4: 按钮点击无反应
**原因**: API 模块未正确导入或加载错误  
**检查**:
```javascript
// 在 Console 中检查
console.log(ScheduleAPI); // 应该是对象，不是 undefined
console.log(ScheduleAPI.createSchedule); // 应该是函数
```

### 问题 5: 修改后数据不同步
**原因**: API 返回的数据格式与前端期望不符  
**检查**:
1. API 响应的日程对象是否包含所有必要字段
2. 字段命名是否与代码中的期望一致
3. 时间字段的格式是否为 ISO 8601 格式
