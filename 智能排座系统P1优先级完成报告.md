- ✅ Service实现：`SeatingSeatServiceImpl.swapSeats()`

- ✅ API端点：`POST /api/seating/seats/swap/{seatId1}/{seatId2}`

**核心业务逻辑：**
```
1. 验证两个座位ID存在且有效
2. 验证两个座位属于同一会场（防止跨会场交换）
3. 保存交换前的分配信息（座位所有者）
4. 交换两个座位的分配人员信息
5. 更新数据库
6. 返回交换前后的对比结果
```

**关键实现细节：**
- **事务处理**: `@Transactional(rollbackFor = Exception.class)` 确保数据一致性
- **异常处理**: 使用`SeatingBusinessException`处理业务异常
- **数据验证**: 座位存在性、会场一致性检查
- **日志记录**: 完整的操作日志用于审计

### 3.1 座位交换API

**端点**: `POST /api/seating/seats/swap/{seatId1}/{seatId2}`

**请求参数**:
```
URL路径参数:

- seatId2: Long - 第二个座位ID
```

**请求示例**:

```bash
Authorization: Bearer {token}
```

**响应成功示例** (HTTP 200):

```json
{
  "code": 200,
  "message": "座位交换成功",
  "data": {
    "success": true,
    "seatId1": 1,
    "seatId2": 2,
    "beforeSeat1Assignee": "张三",
    "beforeSeat2Assignee": "李四",
    "afterSeat1Assignee": "李四",
    "afterSeat2Assignee": "张三",
    "swapTime": "2026-03-12T10:30:00",
    "errorMessage": null
  }
}
```

**响应异常示例** (HTTP 400):
```json
{
  "code": 400,
  "message": "座位交换失败: 座位不在同一会场，无法交换",
  "data": null
}
```

**响应异常示例** (HTTP 500):
```json
{
  "code": 500,
  "message": "交换失败: 座位不存在",
  "data": null
}
```

**异常场景处理**:
1. 座位ID为空 → 400 IllegalArgumentException
2. 座位1不存在 → 400 SeatingBusinessException.SeatNotFound
3. 座位2不存在 → 400 SeatingBusinessException.SeatNotFound
4. 两座位不在同一会场 → 400 SeatingBusinessException("座位不在同一会场，无法交换")
5. 数据库异常 → 500 Database Exception

## 4. 技术实现细节

### 4.1 swapSeats方法实现流程

**文件**: `SeatingSeatServiceImpl.java` (254-318行)

```java
@Override
@Transactional(rollbackFor = Exception.class)
public SeatSwapResponse swapSeats(Long seatId1, Long seatId2) {
    // Step 1: 参数验证
    if (seatId1 == null || seatId2 == null) {
        throw new IllegalArgumentException("座位ID不能为空");
    }

    // Step 2: 查询座位信息
    SeatingSeat seat1 = this.getById(seatId1);
    SeatingSeat seat2 = this.getById(seatId2);

    // Step 3: 座位存在性检查
    if (seat1 == null) throw new SeatingBusinessException.SeatNotFound(seatId1);
    if (seat2 == null) throw new SeatingBusinessException.SeatNotFound(seatId2);

    // Step 4: 会场一致性检查
    if (!seat1.getVenueId().equals(seat2.getVenueId())) {
        throw new SeatingBusinessException("座位不在同一会场，无法交换");
    }

    // Step 5: 保存交换前信息（审计用）
    Long seat1UserId = seat1.getAssignedUserId();
    String seat1UserName = seat1.getAssignedUserName();

    // Step 6: 执行交换
    seat1.setAssignedUserId(seat2.getAssignedUserId());
    seat1.setAssignedUserName(seat2.getAssignedUserName());

    seat2.setAssignedUserId(seat1UserId);
    seat2.setAssignedUserName(seat1UserName);

    // Step 7: 数据库持久化
    this.updateById(seat1);
    this.updateById(seat2);

    // Step 8: 构建响应
    return SeatSwapResponse.builder()
            .success(true)
            .seatId1(seatId1)
            .seatId2(seatId2)
            .beforeSeat1Assignee(seat1UserName)
            .beforeSeat2Assignee(seat2UserName)
            .afterSeat1Assignee(seat2UserName)
            .afterSeat2Assignee(seat1UserName)
            .swapTime(LocalDateTime.now())
            .build();
}
```

### 4.2 Controller端点实现

**文件**: `SeatingController.java` (360-382行)

```java
@PostMapping("/seats/swap/{seatId1}/{seatId2}")
@Operation(summary = "交换座位分配", description = "交换两个座位的分配人员，两个座位必须属于同一会场")
public ApiResponse<?> swapSeats(@PathVariable Long seatId1, @PathVariable Long seatId2) {
    try {
        log.info("开始交换座位: seatId1={}, seatId2={}", seatId1, seatId2);
        
        SeatSwapResponse result = seatingSeatService.swapSeats(seatId1, seatId2);
        
        if (result.getSuccess() != null && result.getSuccess()) {
            return ApiResponse.success(result, "座位交换成功");
        } else {
            return ApiResponse.error(400, "座位交换失败: " + 
                    (result.getErrorMessage() != null ? result.getErrorMessage() : "未知错误"));
        }
    } catch (Exception e) {
        log.error("座位交换异常", e);
        return ApiResponse.error(500, "交换失败: " + e.getMessage());
    }
}
```
