## 一、快速启动（5分钟）

### 1.1 环境要求

```
✓ JDK 17+
✓ Maven 3.8+
✓ MySQL 9.6
✓ Git
```

### 1.2 快速部署步骤

**第1步**：建立数据库
```bash
mysql -h localhost -P 3308 -u root -proot
> CREATE DATABASE conference_seating CHARACTER SET utf8mb4;
> USE conference_seating;
> source seating_schema.sql;
```

**第2步**：编译项目
```bash
cd conference-seating
mvn clean package -DskipTests
```

**第3步**：启动应用
```bash
mvn spring-boot:run
# 或
java -jar target/conference-seating-1.0.0.jar
```

**第4步**：验证
```bash
curl http://localhost:8085/actuator/health
```

## 六、问题排查指南

### Q1：启动时报错"No qualifying bean of type 'TenantFilter'"

**排查步骤**：
1. 检查SeatingApplication.java中是否有@ComponentScan注解
2. 确保@ComponentScan路径包含"com.conference.common"
3. 重新编译并启动

**验证命令**：
```java
// 在startup log中查找
"Scanning for mapper: com.conference.seating.mapper"
"Scanning packages: [com.conference.seating, com.conference.common]"
```

### Q2：启动时报错"Cannot find class for bean with name 'xxMapper'"

**排查步骤**：
1. 检查Mapper接口是否添加了@Mapper注解
2. 检查SeatingApplication.java中是否有@MapperScan注解
3. 确保@MapperScan路径正确：`"com.conference.seating.mapper"`

### Q3：数据库查询返回null

**排查步骤**：
1. 确保表中有数据：`SELECT COUNT(*) FROM table_name;`
2. 检查SQL条件是否正确
3. 检查Entity字段名是否与数据库列名一致（使用@TableField如有不同）

### Q4：分配结果不符合预期

**排查步骤**：
1. 检查传入的参数（attendees、seats列表）
2. 验证选择的算法是否正确
3. 可以直接调用算法测试：
```java
List<SeatingAttendee> testAttendees = ...;
List<SeatingSeat> testSeats = ...;
SeatingAlgorithm algo = new DepartmentGroupingAlgorithm();
Map<Long, Long> result = algo.execute(testAttendees, testSeats);
System.out.println(result);  // 打印分配结果
```
