## 问题 → 原因 → 修复

### 问题（用户反馈）
```
❌ 点击"待审核(2)"  → 显示42条
❌ 点击"已通过(40)" → 显示42条
✅ 点击"已报名(42)" → 显示42条
❌ 待审核人员未优先显示
```

### 原因（代码分析）
- 后端API `listRegistrations()` **无status参数**，永远返回全部42条
- 前端虽然传递status参数，但**后端忽略**
- 前端**未进行本地过滤**

### 修复方案（前端客户端过滤）
```javascript
// 1️⃣ 获取所有42条数据
const records = API返回的42条;

// 2️⃣ 根据modalType过滤
if (modalType === 'pending')   records = records.filter(x => x.status === 0)  // 2条
if (modalType === 'approved')  records = records.filter(x => x.status === 1)  // 40条
if (modalType === 'registered') records = 排序(待审核优先)                    // 42条

// 3️⃣ 显示过滤后的数据
```

### 核心代码
```javascript
// 过滤逻辑（第1495-1515行）
if (typeKey === 'pending') {
    records = records.filter(item => item.status === 0);  // ✅ 待审核
} else if (typeKey === 'approved') {
    records = records.filter(item => item.status === 1);  // ✅ 已通过
} else if (typeKey === 'registered') {
    records.sort((a, b) => {                              // ✅ 排序
        if (a.status === 0 && b.status !== 0) return -1;  // 待审核优先
        if (a.status !== 0 && b.status === 0) return 1;
        return new Date(b.createdTime) - new Date(a.createdTime);
    });
}
```
