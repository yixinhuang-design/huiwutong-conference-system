## 🗄️ 数据库设计

### 创建的表结构（8个核心表）

1. **conf_schedule** - 日程主表
   - 核心字段：id, meeting_id, tenant_id, title, start_time, end_time, location, host, speaker, status, priority
   - 索引：meeting_id, tenant_id, start_time, status, created_time

2. **conf_schedule_settings** - 日程设置表
   - 功能：管理报到、签到、提醒等配置
   - 字段：need_report, need_checkin, need_reminder, reminder_target, reminder_time, reminder_methods

3. **conf_schedule_attachment** - 日程附件表
   - 功能：存储课件、PPT等附件信息
   - 字段：file_name, file_type, file_url, file_hash, download_count

4. **conf_schedule_participant** - 日程参与人员表
   - 功能：管理日程的参与人员及签到记录
   - 字段：participant_id, participant_role, status, checkin_time

5. **conf_schedule_reminder** - 日程提醒记录表
   - 功能：记录发送给用户的提醒信息
   - 字段：target_user_id, reminder_type, reminder_channel, send_time, status

6. **conf_schedule_checkin** - 日程签到记录表
   - 功能：记录日程的签到数据
   - 字段：participant_id, checkin_time, checkin_method, qrcode_id

7. **conf_schedule_log** - 日程操作日志表
   - 功能：审计日志，记录所有操作
   - 字段：operator_id, action_type, action_detail, old_value, new_value

8. **conf_schedule_template** - 日程模板表
   - 功能：快速创建日程的模板库
   - 字段：template_name, duration_minutes, location, settings

### 数据库迁移脚本
- 位置：`backend/conference-backend/sql/V7__schedule_schema.sql`
- 包含：表创建、索引优化、注释定义

### 关键类说明

#### Service接口 (IScheduleService)
提供以下核心方法：
- `createSchedule(Long meetingId, ScheduleCreateRequest request)` - 创建日程
- `updateSchedule(Long id, ScheduleUpdateRequest request)` - 更新日程
- `deleteSchedule(Long id)` - 删除日程
- `getScheduleDetail(Long id)` - 获取详情
- `listSchedules(Long meetingId, Page page)` - 分页查询
- `listAllSchedules(Long meetingId)` - 全部查询
- `listNeedCheckinSchedules(Long meetingId)` - 查询需要签到的
- `listNeedReminderSchedules(Long meetingId)` - 查询需要提醒的
- `listOngoingSchedules(Long meetingId)` - 查询进行中的
- `listUpcomingSchedules(Long meetingId)` - 查询即将开始的
- `publishSchedule(Long id)` - 发布日程
- `cancelSchedule(Long id)` - 取消日程
- `duplicateSchedule(Long id)` - 复制日程

## 🔌 前端集成

### 前端API模块
- 位置：`admin-pc/conference-admin-pc/js/api/schedule-api.js`
- 提供JavaScript模块，封装所有API调用
- 自动处理请求头、错误处理、Token管理

### 使用示例

```javascript
// 引入API模块
<script src="js/api/schedule-api.js"></script>

// 创建日程
ScheduleAPI.createSchedule(meetingId, {
    title: "日程主题",
    startTime: "2024-06-15 09:00:00",
    endTime: "2024-06-15 10:30:00",
    location: "主会场A厅",
    host: "主持人",
    speaker: "主讲人",
    settings: {
        needCheckin: true,
        checkinMethod: "qrcode",
        needReminder: true,
        reminderTime: 15,
        reminderMethods: ["app", "sms"]
    }
}).then(response => {
    console.log("创建成功:", response);
}).catch(error => {
    console.error("创建失败:", error);
});

// 获取全部日程
ScheduleAPI.allSchedules(meetingId).then(schedules => {
    console.log("日程列表:", schedules);
});

// 发布日程
ScheduleAPI.publishSchedule(scheduleId).then(() => {
    console.log("发布成功");
});

// 复制日程
ScheduleAPI.duplicateSchedule(scheduleId).then(newSchedule => {
    console.log("复制成功，新日程ID:", newSchedule.id);
});
```

### HTML中的集成
在schedule-mgr.html中修改saveSchedule、deleteSchedule等方法，调用API而不是使用本地数据：

```javascript
// 原代码（使用本地数据）
async saveSchedule() {
    // ...
    this.schedules.push(schedule);
    // ...
}

// 修改为（调用后端API）
async saveSchedule() {
    try {
        if (this.isEditMode) {
            await ScheduleAPI.updateSchedule(this.scheduleForm.id, this.scheduleForm);
        } else {
            await ScheduleAPI.createSchedule(this.conference.id, this.scheduleForm);
        }
        // 重新加载列表
        await this.loadSchedules();
        this.closeModal();
        this.$showToast('保存成功', 'success');
    } catch (error) {
        this.$showToast('保存失败: ' + error.message, 'error');
    }
}
```
