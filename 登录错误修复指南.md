# 登录错误修复 - 租户不存在问题

**错误信息**: `Error: 租户不存在`  
**原因**: 数据库中 `sys_tenant` 表没有创建租户记录  
**解决方案**: 运行SQL初始化脚本  

---

## 🚀 快速解决（3步）

### Step 1: 打开MySQL客户端

使用任何MySQL工具（推荐Navicat、MySQL Workbench或命令行）：

```bash
# 命令行方式
mysql -h localhost -P 3308 -u root -p
# 输入密码：Hnhx@123
```

### Step 2: 执行初始化脚本

**选项A：使用提供的脚本文件**

```bash
# 进入数据库
USE conference_auth;

# 运行脚本
source "G:\huiwutong新版合集\初始化默认租户.sql";
```

**选项B：直接执行SQL命令**

复制以下内容粘贴到MySQL客户端执行：

```sql
-- 插入默认租户
INSERT INTO sys_tenant (
    id,
    tenant_code,
    tenant_name,
    description,
    status,
    created_at,
    updated_at,
    deleted
) 
SELECT 
    1,
    'default',
    '默认租户',
    '系统默认租户，用于测试',
    'active',
    NOW(),
    NOW(),
    0
WHERE NOT EXISTS (
    SELECT 1 FROM sys_tenant WHERE tenant_code = 'default' AND deleted = 0
);

-- 插入默认管理员用户
INSERT INTO sys_user (
    id,
    tenant_id,
    username,
    password,
    email,
    real_name,
    status,
    created_at,
    updated_at,
    deleted
)
SELECT 
    1,
    1,
    'admin',
    MD5('123456'),
    'admin@default.com',
    '系统管理员',
    'active',
    NOW(),
    NOW(),
    0
WHERE NOT EXISTS (
    SELECT 1 FROM sys_user WHERE username = 'admin' AND tenant_id = 1 AND deleted = 0
);
```

### Step 3: 验证数据

执行以下SQL确认初始化成功：

```sql
-- 检查租户
SELECT * FROM sys_tenant WHERE tenant_code = 'default';

-- 检查用户
SELECT id, tenant_id, username, email FROM sys_user WHERE username = 'admin';
```

**应该看到**：
- 租户表返回 1 条记录（tenant_code='default'）
- 用户表返回 1 条记录（username='admin'）

如果收到错误 `Access denied for user 'root'`


**检查步骤**：
1. 打开F12开发者工具 → Network标签
2. 点击登录，查看POST请求到 `/auth/login`
3. 检查响应是否为JSON格式
4. 如果有错误信息，记录下来
