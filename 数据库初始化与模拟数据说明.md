## 🗂️ 数据库表结构一览

### 核心表（12 张）

#### 1️⃣ 租户管理（1 张）
```
sys_tenant 租户表
├─ id (BIGINT) - 租户ID
├─ tenant_code (VARCHAR) - 租户编码 [UNIQUE]
├─ tenant_name (VARCHAR) - 租户名称
├─ description (VARCHAR) - 租户描述
├─ status (VARCHAR) - 状态：active/inactive
├─ created_at (TIMESTAMP) - 创建时间
└─ updated_at (TIMESTAMP) - 更新时间

索引: idx_tenant_code, idx_status
```

#### 2️⃣ 权限管理（5 张）
```
sys_user 用户表
├─ id (BIGINT) - 用户ID
├─ tenant_id (BIGINT) - 租户ID [FK]
├─ username (VARCHAR) - 用户名 [UNIQUE per tenant]
├─ password (VARCHAR) - 密码（加密）
├─ email (VARCHAR) - 邮箱
├─ phone (VARCHAR) - 电话
├─ real_name (VARCHAR) - 真实姓名
├─ status (VARCHAR) - 状态
├─ last_login_at (TIMESTAMP) - 最后登录时间
└─ created_at (TIMESTAMP) - 创建时间

索引: uk_tenant_username, idx_tenant_id, idx_email

---

sys_role 角色表
├─ id (BIGINT) - 角色ID
├─ tenant_id (BIGINT) - 租户ID [FK]
├─ role_name (VARCHAR) - 角色名称
├─ role_code (VARCHAR) - 角色编码 [UNIQUE per tenant]
├─ description (VARCHAR) - 角色描述
├─ status (VARCHAR) - 状态
└─ created_at (TIMESTAMP) - 创建时间

索引: uk_tenant_code, idx_tenant_id

---

sys_permission 权限表
├─ id (BIGINT) - 权限ID
├─ tenant_id (BIGINT) - 租户ID [FK]
├─ permission_name (VARCHAR) - 权限名称
├─ permission_code (VARCHAR) - 权限编码
├─ resource_type (VARCHAR) - 资源类型
├─ resource_id (VARCHAR) - 资源ID
├─ action (VARCHAR) - 操作：create/read/update/delete
└─ created_at (TIMESTAMP) - 创建时间

索引: idx_tenant_id, idx_resource_type

---

sys_user_role 用户角色关联表
├─ id (BIGINT) - 关联ID
├─ tenant_id (BIGINT) - 租户ID [FK]
├─ user_id (BIGINT) - 用户ID [FK]
├─ role_id (BIGINT) - 角色ID [FK]
└─ created_at (TIMESTAMP) - 创建时间

索引: uk_user_role, idx_user_id

---

sys_role_permission 角色权限关联表
├─ id (BIGINT) - 关联ID
├─ tenant_id (BIGINT) - 租户ID [FK]
├─ role_id (BIGINT) - 角色ID [FK]
├─ permission_id (BIGINT) - 权限ID [FK]
└─ created_at (TIMESTAMP) - 创建时间

索引: uk_role_permission, idx_role_id
```

#### 3️⃣ 会议管理（3 张）
```
conference 会议表
├─ id (BIGINT) - 会议ID
├─ tenant_id (BIGINT) - 租户ID [FK]
├─ conference_name (VARCHAR) - 会议名称
├─ conference_date (DATE) - 会议日期
├─ location (VARCHAR) - 会议地点
├─ status (VARCHAR) - 状态：planning/active/completed
├─ created_at (TIMESTAMP) - 创建时间
└─ updated_at (TIMESTAMP) - 更新时间

---

conference_registration 注册表
├─ id (BIGINT) - 注册ID
├─ tenant_id (BIGINT) - 租户ID [FK]
├─ conference_id (BIGINT) - 会议ID [FK]
├─ user_id (BIGINT) - 用户ID [FK]
├─ registration_date (TIMESTAMP) - 注册时间
└─ status (VARCHAR) - 状态：registered/cancelled

---

seating_arrangement 座位表
├─ id (BIGINT) - 座位ID
├─ tenant_id (BIGINT) - 租户ID [FK]
├─ conference_id (BIGINT) - 会议ID [FK]
├─ seat_number (VARCHAR) - 座位号
├─ assigned_user_id (BIGINT) - 分配给用户ID [FK]
├─ created_at (TIMESTAMP) - 创建时间
└─ updated_at (TIMESTAMP) - 更新时间
```

#### 4️⃣ 通知与协作（3 张）
```
notification 通知表
├─ id (BIGINT) - 通知ID
├─ tenant_id (BIGINT) - 租户ID [FK]
├─ recipient_id (BIGINT) - 接收人ID [FK]
├─ title (VARCHAR) - 标题
├─ content (VARCHAR) - 内容
├─ type (VARCHAR) - 类型：system/email/sms
├─ status (VARCHAR) - 状态：sent/read/failed
├─ created_at (TIMESTAMP) - 创建时间
└─ read_at (TIMESTAMP) - 阅读时间

---

collaboration 协作表
├─ id (BIGINT) - 协作ID
├─ tenant_id (BIGINT) - 租户ID [FK]
├─ conference_id (BIGINT) - 会议ID [FK]
├─ user_id_1 (BIGINT) - 用户1 [FK]
├─ user_id_2 (BIGINT) - 用户2 [FK]
├─ collaboration_type (VARCHAR) - 协作类型
├─ created_at (TIMESTAMP) - 创建时间
└─ updated_at (TIMESTAMP) - 更新时间

---

navigation 导航表
├─ id (BIGINT) - 导航ID
├─ tenant_id (BIGINT) - 租户ID [FK]
├─ conference_id (BIGINT) - 会议ID [FK]
├─ from_seat_id (BIGINT) - 起点座位ID [FK]
├─ to_seat_id (BIGINT) - 终点座位ID [FK]
├─ route_distance (DECIMAL) - 路线距离
├─ estimated_time (INTEGER) - 预计时间（秒）
├─ created_at (TIMESTAMP) - 创建时间
└─ updated_at (TIMESTAMP) - 更新时间
```

## 🚀 数据库初始化流程

### 自动初始化（推荐）

#### Step 1: Flyway 自动执行
```
系统启动时，Flyway 会自动执行：
1. conference-common/src/main/resources/db/migration/V1__init_database.sql
2. conference-auth/src/main/resources/db/migration/V1__init_database.sql
3. conference-notification/src/main/resources/db/migration/V1__init_database.sql
4. 其他模块的初始化脚本...

执行顺序: 按文件名的版本号
V1__xxx.sql → V2__xxx.sql → V3__xxx.sql ...
```

#### Step 2: 验证初始化成功
```bash
# MySQL 命令行查看
mysql -h 127.0.0.1 -P 3308 -u root -p123456

# 查看表
use conference_db;
show tables;

# 查看租户
select * from sys_tenant;

# 查看用户
select * from sys_user;
```

### 手动初始化

#### 方式1: MySQL 命令行
```bash
# 连接数据库
mysql -h 127.0.0.1 -P 3308 -u root -p123456 conference_db

# 执行脚本
source conference-backend/conference-common/src/main/resources/db/migration/V1__init_database.sql;
source conference-backend/conference-auth/src/main/resources/db/migration/V1__init_database.sql;

# 验证
select * from sys_tenant;
```

#### 方式2: Navicat
```
1. 在 Navicat 中右键数据库
2. 选择 "运行 SQL 文件"
3. 选择 V1__init_database.sql
4. 点击运行
```

#### 方式3: Java 程序
```java
// 通过 @PostConstruct 自动执行
@Component
public class DatabaseInitializer {
    
    @PostConstruct
    public void init() {
        // Flyway 会自动检测并执行 SQL 迁移脚本
        // 无需手动执行
    }
}
```

## ✨ 多租户数据隔离

### 隔离机制

#### 1. SQL 层面隔离
```java
// TenantInterceptor 自动添加 tenant_id 条件
// 所有查询都会自动过滤：WHERE tenant_id = ?

SELECT * FROM sys_user
// 实际执行：
SELECT * FROM sys_user WHERE tenant_id = 1
```

#### 2. ThreadLocal 存储
```java
// TenantContextHolder 存储当前租户信息
TenantContextHolder.setTenantId(1);  // 设置
TenantContextHolder.getTenantId();   // 获取
TenantContextHolder.clear();         // 清除
```

#### 3. 数据库约束
```sql
-- 联合唯一约束确保租户内用户唯一
UNIQUE KEY uk_tenant_username (tenant_id, username)

-- 联合主键和外键保证数据关联
UNIQUE KEY uk_user_role (tenant_id, user_id, role_id)
```

### 隔离验证
```sql
-- 租户1的用户无法看到租户2的数据
SELECT * FROM sys_user WHERE tenant_id = 1;  -- 3 条记录
SELECT * FROM sys_user WHERE tenant_id = 2;  -- 1 条记录

-- 联合查询时需要指定租户
SELECT u.*, r.role_name 
FROM sys_user u
JOIN sys_user_role ur ON u.id = ur.user_id
JOIN sys_role r ON ur.role_id = r.id
WHERE u.tenant_id = 1 AND r.tenant_id = 1;  -- 必须指定租户ID
```

## 🛠️ 常见操作

### 重置所有数据
```bash
# 方式1: 删除并重新初始化
mysql -h 127.0.0.1 -P 3308 -u root -p123456 conference_db -e "DROP DATABASE conference_db; CREATE DATABASE conference_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 然后重启应用，Flyway 会自动重建
```

### 清空特定表
```sql
-- 清空用户角色关联（保留其他数据）
DELETE FROM sys_user_role;

-- 清空用户（会级联清空关联）
DELETE FROM sys_user;

-- 清空租户（谨慎，会影响所有关联数据）
DELETE FROM sys_tenant;
```

### 导出数据备份
```bash
# 导出整个数据库
mysqldump -h 127.0.0.1 -P 3308 -u root -p123456 conference_db > backup.sql

# 导出特定表
mysqldump -h 127.0.0.1 -P 3308 -u root -p123456 conference_db sys_user > sys_user.sql

# 导出仅结构不要数据
mysqldump -h 127.0.0.1 -P 3308 -u root -p123456 --no-data conference_db > schema.sql
```

### 恢复数据
```bash
# 恢复全部数据
mysql -h 127.0.0.1 -P 3308 -u root -p123456 conference_db < backup.sql

# 恢复单个表
mysql -h 127.0.0.1 -P 3308 -u root -p123456 conference_db < sys_user.sql
```
