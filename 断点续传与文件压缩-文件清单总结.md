## 🎯 使用场景

### 场景1: 上传小文件（<5MB）
```
用户选择文件 → 验证 → 初始化 → 上传(1块) → 合并 → 完成
耗时: 3-10秒
```

### 场景2: 上传中等文件（5-50MB）
```
用户选择文件 → 验证 → 初始化 → 并发上传(3块×N) → 合并 → 完成
耗时: 30-60秒
分块数: 1-10个
```

### 场景3: 上传大文件（50-100MB）
```
用户选择文件 → 验证 → 初始化 → 并发上传(3块×N) → 暂停 → 继续 → 合并 → 完成
耗时: 60-120秒
分块数: 10-20个
特点: 支持暂停/继续
```

### 场景4: 网络不稳定
```
上传中 → 网络断开 → 自动重试(1-3次) → 恢复成功或显示失败 → 用户重新上传或继续
```

## 📞 故障排查快速指南

| 症状 | 可能原因 | 检查项 |
|------|--------|--------|
| POST /api/schedule/chunk/init 404 | 类未编译或路由未注册 | 检查ChunkUploadController.java是否存在 |
| "上传任务不存在" | uploadId过期或错误 | 检查是否超过24小时，查看服务器日志 |
| 临时文件堆积 | cleanExpiredTempFiles()未运行 | 手动删除uploads/.temp或重启服务 |
| 文件合并失败 | 分块不完整或损坏 | 查看服务器日志，检查哈希值是否匹配 |
| 图片压缩失败 | FileCompressionUtil错误或权限不足 | 检查enable-image-compress配置，查看日志 |
| 无法下载文件 | 文件URL错误或文件不存在 | 检查fileUrl字段，验证文件系统路径 |
