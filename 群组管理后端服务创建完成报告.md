## 2️⃣ Web Key持久化设计 ✅

### 设计原理

**什么是Web Key？**
- 类似于UUID的唯一标识符
- 由后端自动生成
- 前端使用Web Key访问资源
- 避免暴露数据库ID

**优势**：
- 🔒 **安全性**：不暴露数据库ID
- 🔗 **无状态**：前后端解耦
- 🌐 **跨域友好**：API更容易调用
- 📱 **离线支持**：前端可缓存Web Key

### 实现方式

```java
// 实体类中的Web Key字段
private String webKey; // 每个实体都有

// 自动生成Web Key
String webKey = "group_" + IdUtil.simpleUUID().substring(0, 16);
group.setWebKey(webKey);

// 唯一索引
ALTER TABLE conference_group
ADD UNIQUE KEY uk_web_key (web_key);
```

### API接口示例

**创建群组**：
```
POST /api/group/create
Response: {
    "code": 200,
    "message": "群组创建成功",
    "data": {
        "id": 1,
        "groupName": "测试群组",
        "webKey": "group_abc123..."
    }
}
```

**使用Web Key操作**：
```
GET /api/group/group_abc123...
PUT /api/group/group_abc123...

```

## 🔧 Web Key生成策略

### 前缀规则
- 群组：`group_xxxxxxxxxxxxxxxx`
- 成员：`mem_xxxxxxxxxxxxxxxx`
- 任务：`task_xxxxxxxxxxxxxxxx`
- 消息：`msg_xxxxxxxxxxxxxxxx`

### 生成算法
```java
// 使用Hutool工具类
String webKey = "group_" + IdUtil.simpleUUID().substring(0, 16);
// 示例：group_3a5b7c9d1f2e4b6

// 确保唯一性
ALTER TABLE conference_group
ADD UNIQUE KEY uk_web_key (web_key);
```
