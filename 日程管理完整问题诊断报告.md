## 🔍 问题链路分析（完整链条）

### Phase 1: Token 提取与租户隔离 ✅ **已修复**

**问题（已解决）**：
- TenantFilter 的 `extractTokenFromHeader` 方法只支持 "Bearer " 前缀格式
- 如果前端直接发送 Token（无前缀），则提取失败，返回 null
- 导致租户 ID 为 null，数据库查询被污染

**修复内容**：
```java
// 修改前：只支持 "Bearer " 前缀
if (authHeader != null && authHeader.startsWith("Bearer ")) {
    return authHeader.substring(7);
}
return null;  // 其他格式直接返回 null ❌

// 修改后：同时支持两种格式
if (authHeader.startsWith("Bearer ")) {
    return authHeader.substring(7).trim();
}
// 支持直接 token 格式（不需要 Bearer 前缀）
return token.isEmpty() ? null : token;  ✅
```

**修复文件**: `backend/conference-backend/conference-common/src/main/java/com/conference/common/util/JwtUtils.java`
