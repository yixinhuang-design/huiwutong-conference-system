## 🎯 核心功能导航地图

### 协管员的日常工作流

```
1️⃣ 打开仪表盘
   app/staff/dashboard.html
   └─ 查看核心数据：报名统计、签到统计、异常预警
   └─ 快速导航到各个管理功能

2️⃣ 处理报名审核
   app/staff/registration-manage.html
   └─ 查看报名列表
   └─ 批量选择并进行：批量通过、批量退回、导出
   └─ 支持智能审核和审核日志

3️⃣ 分配座位
   app/staff/seat-manage.html
   └─ 查看座位布局
   └─ 拖拽分配待分配成员
   └─ 查看分配统计

4️⃣ 编排分组
   app/staff/grouping-manage.html
   └─ 创建讨论分组
   └─ 分配成员、组长、工作人员
   └─ 查看分组列表

5️⃣ 派发任务
   app/staff/task-list.html
   └─ 创建任务卡片
   └─ 分配负责人
   └─ 跟踪执行进度（待执行、进行中、已完成、已逾期）

6️⃣ 维护日程
   app/staff/schedule.html
   └─ 录入日程信息（时间、讲师、地点、人数限制）
   └─ 配置日程状态（已发布、待确认、草稿）
   └─ 支持批量导入

7️⃣ 发送通知
   app/staff/notice-manage.html
   └─ 选择通知模板
   └─ 精准推送（全体/分组/按状态筛选）
   └─ 配置通知渠道（短信、APP、推送）
   └─ 追踪发送效果

8️⃣ 工作沟通
   app/staff/chat-room.html
   └─ 协管员群聊
   └─ 支持发送任务卡片
   └─ 公告、提醒展示

9️⃣ 查看数据
   app/staff/data-analysis.html
   └─ 多维度数据分析（报名、签到、任务、通知）
   └─ 用户行为分析
   └─ 系统性能监控
   └─ 导出数据报表

🔟 配置预警
   app/staff/alert-config.html
   └─ 新建预警规则（报名率、签到率、任务完成率等）
   └─ 设置预警阈值和通知方式
   └─ 启用/禁用规则

1️⃣1️⃣ 处理预警
   app/staff/alert-handle.html
   └─ 查看预警事件列表
   └─ 标记处理、完成事件
   └─ 评估预警效果
```

## 💡 常见场景快速定位

### Q: 如何查看数据统计？
A: `app/staff/dashboard.html` 核心数据4宫格或 `app/staff/data-analysis.html` 详细数据分析

### Q: 如何发送通知？
A: `app/staff/notice-manage.html` → 选择快捷操作或自定义通知 → 配置对象和渠道

### Q: 如何管理座位？
A: 
- 协管员: `app/staff/seat-manage.html` 进行分配
- 学员: `app/learner/seat.html` 查看自己的座位和会场导航

### Q: 如何查看日程？
A:
- 协管员: `app/staff/schedule.html` 录入和维护
- 学员: `app/learner/schedule.html` 查看和设置提醒

### Q: 如何派发和完成任务？
A:
- 协管员: `app/staff/task-list.html` 派发和跟踪
- 学员: `app/learner/task-list.html` 查看和完成

### Q: 如何配置预警？
A:
- 配置: `app/staff/alert-config.html` 新建规则
- 处理: `app/staff/alert-handle.html` 处理预警事件

### Q: 如何进行群聊交流？
A:
- 协管员: `app/staff/chat-room.html` 工作人员群聊
- 学员: `app/learner/chat-room.html` 全体学员群

### Q: 如何批量审核报名？
A: `app/staff/registration-manage.html` → 全选或多选报名 → 点击批量操作按钮

## 🚀 下一步开发

### 对接后端API
```javascript
// 示例：如何将静态页面改为动态数据

// 修改前（静态）
<div>报名数: 160</div>

// 修改后（动态）
<div id="registration-count">加载中...</div>

<script>
  // 调用后端接口获取数据
  fetch('/api/registration/count')
    .then(res => res.json())
    .then(data => {
      document.getElementById('registration-count').textContent = data.count;
    });
</script>
```

### 实时功能实现
```javascript
// WebSocket 实时通知示例
const ws = new WebSocket('ws://server:8080/notify');

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  if (data.type === 'chat') {
    addChatMessage(data.message);
  } else if (data.type === 'alert') {
    showAlert(data.alert);
  }
};
```

## 📞 常见问题

**Q: 如何修改会议名称？**
A: 所有页面的会议名"2025党务干部培训班"都是静态HTML，开发时需要改为动态数据。

**Q: 如何添加新的用户头像？**
A: 修改 `.chat-avatar` 的背景色和图标即可，或替换为 `<img>` 标签。

**Q: 如何实现实时消息同步？**
A: 使用 WebSocket 或 Server-Sent Events，后端推送消息到客户端。

**Q: 如何设置权限控制？**
A: 后端验证用户身份后，返回对应角色的菜单和功能权限。

**Q: 如何部署到生产环境？**
A: 打包静态文件，部署到 Web 服务器（Nginx、Apache等）或 CDN。
