# 座位编号系统集成指南

## 概览

本文档说明如何将新的座位编号系统集成到现有的 `seating-mgr.html` 中。

---

## 一、修正要点总结

### 编号方式的准确定义

根据您提供的实际图片数据，修正了原分析中对编号方式的理解：

#### 原错误 → 修正后

1. **中间左右**
   - ❌ 错: 从中间向外的简单扩散
   - ✅ 对: 中心位置编号最小，向两侧递增（6-4-2-1-3-5-7）

2. **由左至右** 
   - ✅ 正确（无需修正）
   - 编号: 1-2-3-4-5-6-7 (逐行递增)

3. **左右右左**
   - ❌ 错: "左至右左"
   - ✅ 对: 奇数行LTR，偶数行RTL的蛇形编号
   - 编号: 行1(1-7)，行2(14-8)

4. **由右至左**
   - ✅ 正确（无需修正）
   - 编号: 7-6-5-4-3-2-1 (逐行递减)

5. **右左左右** （新增）
   - 📌 新补充的模式
   - ✅ 对: 奇数行RTL，偶数行LTR的反向蛇形
   - 编号: 行1(7-1)，行2(8-14)

```html
// ============ Vue 应用代码 ============
const { createApp, ref, reactive, computed, onMounted } = Vue;
const app = createApp({
  // ...existing code...
});
</script>
```

### Step 2: 修改 `createSeatingArea` 函数

在现有的 seating-mgr.html 中找到 `createSeatingArea` 函数，修改如下：

```javascript
// 原版本
const createSeatingArea = (areaId, areaName, venueType, rows, cols, startRow) => {
  return {
    id: areaId,
    name: areaName,
    type: venueType,
    rows,
    cols,
    startRow,
    seats: Array.from({ length: rows * cols }, (_, idx) => ({
      // 原来的座位生成逻辑...
    }))
  };
};

// 修改后版本（支持编号方式）
const createSeatingArea = (
  areaId, 
  areaName, 
  venueType, 
  rows, 
  cols, 
  startRow,
  numberingMode = 'left_to_right'  // ← 新增参数
) => {
  // 使用新的座位编号生成器
  const generator = new SeatNumberGenerator(rows, cols, numberingMode);
  const seats = generator.generateAll().map(seat => ({
    ...seat,
    id: `${areaId}_${seat.row}_${seat.col}`,
    areaId,
    status: 'empty',
    occupant: null,
    assignedAt: null,
    // ... 其他原有的座位属性
  }));
  
  return {
    id: areaId,
    name: areaName,
    type: venueType,
    rows,
    cols,
    startRow,
    numberingMode,              // ← 保存编号方式
    numberingModeName: generator.getModeName(),
    seats
  };
};
```

### Step 3: 更新座位区域初始化代码

在初始化座位区域时，为各区域指定不同的编号方式：

```javascript
// 初始化座位区域 - 保持原有的四个区域 + 舞台
seatingAreas.value = [
  // 舞台：中间左右模式（突出主位）
  createSeatingArea('area_stage', '主席台', 'stage', 1, 8, 0, 'center_outward'),
  
  // A区（VIP）：中间左右模式（对称、尊重）
  createSeatingArea('area_a', 'A区 (VIP区)', 'vip', 5, 10, 0, 'center_outward'),
  
  // B区（普通）：由左至右模式（标准、易于理解）
  createSeatingArea('area_b', 'B区', 'regular', 10, 15, 0, 'left_to_right'),
  
  // C区（普通）：左右右左模式（蛇形、便于通道）
  createSeatingArea('area_c', 'C区', 'regular', 10, 15, 0, 'alternating_ltr_start')
];
```

### Step 4: 添加编号方式配置界面

在座位区域配置对话框中添加编号方式选择器：

```html
<!-- 在座位分配对话框中添加 -->
<div class="numbering-mode-selector">
  <h4>选择座位编号方式</h4>
  <div class="mode-options">
    <label v-for="mode in numberingModes" :key="mode.key" class="mode-option">
      <input 
        type="radio" 
        :value="mode.key" 
        v-model="selectedNumberingMode"
        @change="updateAreaNumbering"
      />
      <span class="mode-name">{{ mode.name }}</span>
      <span class="mode-desc">{{ mode.desc }}</span>
    </label>
  </div>
</div>

<script>
// 在 Vue setup 中
const numberingModes = SeatNumberGenerator.getSupportedModes();
const selectedNumberingMode = ref('left_to_right');

const updateAreaNumbering = () => {
  // 更新选中区域的编号方式
  if (currentEditingArea.value) {
    const newArea = createSeatingArea(
      currentEditingArea.value.id,
      currentEditingArea.value.name,
      currentEditingArea.value.type,
      currentEditingArea.value.rows,
      currentEditingArea.value.cols,
      currentEditingArea.value.startRow,
      selectedNumberingMode.value
    );
    
    const idx = seatingAreas.value.findIndex(a => a.id === currentEditingArea.value.id);
    if (idx !== -1) {
      seatingAreas.value.splice(idx, 1, newArea);
    }
  }
};
</script>
```

## 五、数据迁移

### 如果已有历史数据

```javascript
// 如果旧数据中只记录了行列位置，需要转换为新的座位号
const migrateOldData = (oldSeats, areaConfig) => {
  const generator = new SeatNumberGenerator(
    areaConfig.rows,
    areaConfig.cols,
    areaConfig.numberingMode
  );
  
  return oldSeats.map(oldSeat => ({
    ...oldSeat,
    number: generator.generate(oldSeat.row, oldSeat.col),
    // 保留原有的行列信息，用于兼容性
    row: oldSeat.row,
    col: oldSeat.col
  }));
};

// 在加载历史数据时调用
onMounted(() => {
  // ... 现有的初始化代码 ...
  
  // 迁移历史座位数据
  seatingAreas.value.forEach(area => {
    if (area.seats && area.numberingMode) {
      area.seats = migrateOldData(area.seats, area);
    }
  });
});
```

## 七、常见问题

### Q1: 编号方式改变后，现有座位分配怎么办？

**A:** 有两种方案：
- 方案A（推荐）：迁移数据，使用 `SeatNumberConverter` 将旧编号转换为新编号
- 方案B：清空重新分配，这样更干净但会丢失历史数据

### Q2: 能否动态改变某个区域的编号方式？

**A:** 可以，使用 `createSeatingArea()` 重新创建该区域：
```javascript
const updateAreaNumbering = (areaId, newMode) => {
  const area = seatingAreas.value.find(a => a.id === areaId);
  if (area) {
    const newArea = createSeatingArea(
      area.id,
      area.name,
      area.type,
      area.rows,
      area.cols,
      area.startRow,
      newMode
    );
    const idx = seatingAreas.value.indexOf(area);
    seatingAreas.value.splice(idx, 1, newArea);
  }
};
```

### Q3: 舞台（主席台）应该用哪种编号方式？

**A:** 推荐用 `center_outward`（中间左右），这样中央位置编号最小，体现尊重和对称的美感。

### Q4: 如何在导出Excel时正确显示座位号？

**A:** 确保导出使用 `seat.number` 字段：
```javascript
const exportData = seatingAreas.value.flatMap(area =>
  area.seats
    .filter(s => s.occupant)
    .map(s => ({
      areaName: area.name,
      seatNumber: s.number,          // ← 用number，不用row/col
      seatPosition: s.displayText,   // 可选：同时显示行列位置
      personName: s.occupant.name,
      personTitle: s.occupant.title,
      // ...
    }))
);
```

## 八、性能考虑

### 编号方式切换性能

对于大规模座位区（比如10行×50列），重新计算所有座位号是 O(rows×cols) 的操作，通常在 100ms 内完成，用户基本感受不到延迟。

### 建议

- ✅ 初始化时一次性生成所有座位号（使用 `generateAll()`）
- ✅ 编号方式切换后需要重新渲染，可加入简单的加载动画
- ✅ 不要在循环内反复调用 `generate()`，改用 `generateAll()` 后再使用
