## 🐛 发现的问题

### 1. API调用方式不一致
**问题**：前端API调用时未传递 `meetingId` 参数
```javascript
// 当前代码
const response = await ScheduleAPI.createSchedule(this.scheduleForm);

// 应该是
const response = await ScheduleAPI.createSchedule(this.conference.id, this.scheduleForm);
```

**影响**：
- 创建日程可能失败（后端需要 meetingId 参数）
- 编辑和删除操作不受影响（直接使用 scheduleId）

**建议修复**：
```javascript
// 在 saveSchedule() 方法中
if (this.isEditMode) {
    await ScheduleAPI.updateSchedule(this.scheduleForm.id, this.scheduleForm);
} else {
    const response = await ScheduleAPI.createSchedule(this.conference.id, this.scheduleForm);
}
```

### 2. 文件上传时序问题
**问题**：创建日程后才能上传附件，但用户可能期望一次性完成
```javascript
// 当前逻辑
if (!this.scheduleForm.id) {
    this.$showToast('请先保存日程，再上传课件', 'warning');
    return;
}
```

**影响**：用户体验不佳，需要两次操作

**建议优化**：
- 方案A：创建日程时自动保存草稿，获得ID后再上传附件
- 方案B：支持附件临时暂存，创建时一并提交
- 方案C：调整UI流程，先填写基本信息并保存，再跳转上传

### 3. 时间范围查询API未使用
**问题**：后端已实现 `/by-time-range` 接口，但前端未调用

**建议补充**：
- 添加日期范围筛选器
- 支持快速筛选"今天"、"明天"、"本周"等
