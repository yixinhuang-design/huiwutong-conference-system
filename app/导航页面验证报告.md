### 问题2：共有界面链接路径错误 ❌ → ✅

**问题描述**：
- nav-all.html 中共有界面的链接使用相对路径（如 `href="home.html"`）
- 但实际文件位置在 `common/` 目录下（如 `common/home.html`）
- 导致从 nav-all.html 点击共有界面卡片时无法找到页面

**原因分析**：
nav-all.html 文件位于 app/ 根目录，而共有界面文件位于 common/ 子目录，需要使用正确的相对路径。

**修复方案**：
将所有共有界面链接从相对路径改为带 common/ 前缀的路径：

| 修复前 | 修复后 |
|--------|--------|
| `href="home.html"` | `href="common/home.html"` |
| `href="profile.html"` | `href="common/profile.html"` |
| `href="assistant.html"` | `href="common/assistant.html"` |
| `href="communication.html"` | `href="common/communication.html"` |
| `href="past.html"` | `href="common/past.html"` |
| `href="navigation.html"` | `href="common/navigation.html"` |
| `href="voice-recognition.html"` | `href="common/voice-recognition.html"` |
| `href="system-intro.html"` | `href="common/system-intro.html"` |
| `href="index.html"` | `href="common/index.html"` |

**验证结果**：✅ 所有9个共有界面链接已修复
