#### ❌ 网关服务 (8080/8082)
**问题发现**:
```
端口    进程             说明                   状态
8080    node.exe(29436)  占用网关配置端口      ❌ 阻塞网关启动
8081    java.exe(14320)  auth-service          ✅ 运行正常
8082    java.exe(14208)  registration-service  ⚠️ 被误认为网关
8084    java.exe(60952)  meeting-service       ✅ 运行正常
```

**根本原因**:
1. 网关在 `application.yml` 中配置为 `server.port: 8080`
2. 8080 端口被 Node.js 占用（可能是前端开发服务器）
3. 前端代码硬编码 45 处 `http://localhost:8082`（期望为网关）
4. 8082 实际运行的是 **报名服务** (conference-registration)
5. 网关服务实际未启动

**影响范围**: 30+ 前端页面无法通过网关访问后端接口

## ❌ 阻塞问题详情

### P0 级问题：网关服务架构错配

#### 问题描述
前端期望网关在 8082，实际 8082 是报名服务，网关配置为 8080 但被 Node.js 占用，导致网关无法启动。

#### 技术细节
**网关配置** (conference-gateway/src/main/resources/application.yml):
```yaml
server:
  port: 8080  # 配置端口
  
spring:
  cloud:
    gateway:
      routes:
        - id: auth-service
          uri: lb://auth-service
          predicates:
            - Path=/api/v1/auth/**
        - id: registration-service
          uri: lb://registration-service
          predicates:
            - Path=/api/v1/registration/**
        - id: meeting-service
          uri: lb://meeting-service
          predicates:
            - Path=/api/v1/meeting/**
```

**端口占用分析**:

## 🔧 解决方案

### 方案 A：网关使用备用端口 8083（最快）

#### 优点
- ✅ 无端口冲突，可立即启动
- ✅ 保留 Node.js 和报名服务不动
- ✅ 前端统一修改配置文件一次即可

#### 缺点
- ⚠️ 需重新编译网关（5 分钟）
- ⚠️ 前端需统一修改 API 地址

#### 执行步骤
```powershell
# 1. 修改网关配置
cd G:\huiwutong新版合集\backend\conference-backend\conference-gateway
# 编辑 src/main/resources/application.yml
# 将 server.port: 8080 改为 server.port: 8083

# 2. 重新编译
mvn clean package -DskipTests

# 3. 启动网关
cd target
java -Dspring.profiles.active=dev -jar conference-gateway-1.0.0.jar

# 4. 验证启动
# 等待日志显示: Started GatewayApplication in X.XXX seconds
curl http://localhost:8083/actuator/health

# 5. 测试路由
curl -H "Authorization: Bearer <token>" http://localhost:8083/api/v1/meeting/list
```

#### 前端配置修改
```javascript
// app-uniapp/common/config.js (新建)
export const API_CONFIG = {
  // 开发环境
  development: {
    gateway: 'http://localhost:8083',
    auth: 'http://localhost:8081',
    meeting: 'http://localhost:8084'
  },
  // 生产环境
  production: {
    gateway: 'https://api.example.com',
    auth: 'https://api.example.com',
    meeting: 'https://api.example.com'
  }
}

// 获取当前环境配置
const env = process.env.NODE_ENV || 'development'
export const API_BASE_URL = API_CONFIG[env].gateway
```

然后在各页面替换硬编码URL:
```javascript
// 原代码
// const url = 'http://localhost:8082/api/meeting/list'

// 改为
import { API_BASE_URL } from '@/common/config.js'
const url = `${API_BASE_URL}/api/v1/meeting/list`
```

### 方案 C：Nginx 反向代理（生产级方案）

#### 优点
- ✅ 前端代码完全不改
- ✅ 生产环境标准架构
- ✅ 支持负载均衡、SSL、缓存等高级功能

#### 缺点
- ⚠️ 需安装配置 Nginx（30 分钟）
- ⚠️ 增加部署复杂度

#### Nginx 配置示例
```nginx
# nginx.conf
upstream gateway {
    server 127.0.0.1:8080;
}

server {
    listen 8082;
    server_name localhost;
    
    location /api/ {
        proxy_pass http://gateway;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

#### 执行步骤
```powershell
# 1. 安装 Nginx (Windows)
# 下载 nginx-1.24.0.zip → 解压到 C:\nginx

# 2. 修改配置
# 编辑 C:\nginx\conf\nginx.conf (添加上述配置)

# 3. 启动 Nginx
cd C:\nginx
start nginx

# 4. 启动网关（端口 8080）
# (同方案 B)

# 5. 验证代理
curl http://localhost:8082/api/v1/meeting/list
```

#### 1. 创建配置文件
```javascript
// app-uniapp/common/config.js
export const API_CONFIG = {
  gateway: 'http://localhost:8083',  // 网关地址
  timeout: 10000,                     // 请求超时
  retryCount: 3                       // 重试次数
}

// 封装请求函数
export function request(options) {
  const { url, method = 'GET', data, header = {} } = options
  
  return uni.request({
    url: `${API_CONFIG.gateway}${url}`,
    method,
    data,
    header: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${uni.getStorageSync('token')}`,
      ...header
    },
    timeout: API_CONFIG.timeout
  })
}
```
